# vloop 承認系単語 読み替え表

> Issue #20。Issue 本文の「承認」「判断」「選ぶ」等で vloop/vrun が**停止・確認しないようにする**ための
> **読み替え（解釈）ルール**。
> ※ ファイルや Issue 本文を破壊的に書き換えるのではなく、**Claude が読むときの解釈を固定**する規約。
>   （[[../Issue記述共通ルール]] / [[../../05_monetization/ChatGPT承認ゲート標準]] と一体運用）

## 読み替え表（Issue 本文の語 → Claude の動作）

| Issue 本文の語 | Claude の解釈・動作 | やってはいけない |
|---|---|---|
| 承認 / approve | **推奨案を出す**（比較・根拠つき）。status は変えない | approved 化・承認確認 |
| 判断 / 決める | **仮説を置いて文書化して進む**（停止しない） | ユーザーに「どうしますか」 |
| 選ぶ / 選定 | **候補提示＋推奨順位**を出す（決定はしない） | 1件に勝手に確定し着手 |
| 確認して | **調査して結果を記録**（確認質問を投げない） | 「確認しますか？」 |
| 評価して | スコア化・妥当性評価を**記録**（承認に使わない） | スコアで自動着手 |
| 進めて / 着手 | candidate/pending は**判断材料作成まで**。approved のみ実作業 | 未承認で progress 投入・本番実装 |
| 公開 / リリース | **設計・ドラフトまで**。公開操作はしない（停止条件） | 外部公開・課金・広告 |

## 適用ルール

1. Issue 本文に上記語があっても **停止理由にしない**（停止条件は別途定義済：外部公開/課金/広告/秘密情報/データ削除/破壊的変更/Issueとルール明確矛盾/git失敗のみ）
2. 「承認」系は **推奨/比較/仮説** に読み替えて成果物を出す（承認自体は ChatGPT＋人間）
3. 読み替えた場合、最終報告の「仮説」に「Issue の『承認』を『推奨』と解釈」等を明記
4. 例外: 停止条件に該当する語（公開実行・課金・広告・削除）は読み替えず**停止**
