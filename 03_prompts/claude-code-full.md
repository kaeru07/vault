# Claude Code プロンプト — フル版

新規アプリ・DB 設計・認証・全体設計・複数アプリ横断など大規模作業向け。

## テンプレ

```
対象:
<リポジトリ / アプリ全体>

背景:
- <なぜこのプロジェクトが必要か>
- <既存の構造との関係>

目的:
- <最終的に達成したいこと（中長期）>

スコープ:
- 今回のフェーズで対応する範囲
- 次フェーズに送る範囲

設計方針:
- アーキテクチャの主な選択肢と採用案
- 既存技術スタックとの整合
- データ構造の方針

依存:
- 他アプリ / 外部サービスの前提
- 必要な環境変数（値はここに書かない）

セキュリティ:
- 機密情報の取り扱い
- 認証 / 認可の方針

禁止:
- 既存本番データの破壊
- 環境変数のハードコード
- 認証スキップ実装
- スコープ外への波及

段階:
1. <フェーズ1>
2. <フェーズ2>
3. <フェーズ3>

各フェーズの完了条件:
- フェーズ1: <build/test/動作確認>
- ...

検証:
- 各フェーズ後に typecheck + build + 主要動作確認
- 完了時に手動 QA 項目を列挙

報告形式:
【設計概要】
【実施内容】
【変更ファイル】
【データ構造変更】
【動作確認】
【未対応 / 次フェーズ】
【セキュリティ確認】
```

## 使うシーン

- 新規アプリの初期構築
- 既存アプリの大規模リファクタ
- DB スキーマ変更を伴う改修
- 認証導入・複数アプリ横断機能

## 注意

- 「フェーズ分割」を必ず明示してフェーズごとに区切って進める
- フェーズ完了時に必ず検証＆レビュー
- 機密情報・本番データに触れる場合はユーザー判断待ちで止める
- 大規模変更は ultrareview を併用検討
