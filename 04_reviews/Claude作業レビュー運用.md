# Claude 作業レビュー運用

> Claude Code（および Codex / ChatGPT）の作業を**1 件ずつレビュー**する運用ページ。
> 個別レビューファイルの雛形は `90_templates/claude-review-template.md`。

---

## レビュー方針

- **1 タスク = 1 レビュー = 1 execution-run**（progress アプリの runId と紐付ける）
- 採用したものより、**差し戻し / 棄却したもの**を丁寧に残す（次に同じミスを防ぐ）
- レビュー結論は **採用 / 修正後採用 / 差し戻し / 棚上げ** のいずれかを明記
- 軽量タスクでも 3 行レビューは残す（「OK」だけにしない）

---

## レビュー観点（必ず通す）

| 観点 | チェック内容 |
|---|---|
| 着手前提 5 項目 | 何を直す / どこ触る / どこ触らない / 完了条件 / 検証方法 が事前に揃っていたか |
| スコープ遵守 | 「触らない」と決めた範囲に手を入れていないか |
| 不要な抽象化 | 不要なヘルパー・引数・設定が混ざっていないか |
| 不要なコメント | 「何をしているか」だけのコメントが増えていないか |
| 命名 | 内容を表す名前になっているか |
| 既存機能への影響 | 関連箇所・既存テスト・既存導線への波及 |
| 検証 | typecheck / build / 動作確認の結果 |
| エラー握りつぶし | try-catch で黙って飲み込んでいないか |

---

## 抜け漏れ確認

- [ ] 5 項目が事前に提示されたか
- [ ] 変更ファイル一覧が報告にあるか
- [ ] build / typecheck の結果が報告にあるか
- [ ] 未対応 / 次にやるべきことが書かれているか
- [ ] progress アプリへの POST が完了したか（runId が記載されているか）
- [ ] 同時に複数タスクを混ぜていないか（1 task = 1 review = 1 run）

---

## 危険変更の確認

特に注意する変更:

- DB スキーマ / マイグレーション
- 認証 / セッション
- 環境変数 / `.env` の読み書き
- 本番デプロイ系のスクリプト
- パッケージ依存の大幅変更（メジャー上げ・削除）
- `git push --force` / `git reset --hard` 等の破壊的操作
- 外部 API（決済 / メール / SMS）の呼び出し

危険変更があれば:
- `engineering/reports/dangerous-work-YYYYMMDD-HHMM.md` に詳細を残す
- レビューの結論欄に「危険変更あり」を明記
- 超危険作業は「**ユーザー判断待ち**」で保留

---

## 次アクション

レビューの最後に**必ず**書く:

- 採用後の追加タスク（リファクタ・ドキュメント）
- 差し戻し時の再着手条件
- 棚上げ時の再評価条件・期限

「次アクションなし」も明記。空欄は禁止。

---

## progress レビューとの関係

- progress アプリ側にも ExecutionRun のレビュー欄がある（reviewStatus）
- 役割分担:
  - **progress 側**: 構造化された短評（採用 / 差し戻し / メモ）と reviewer 名
  - **Obsidian 側**（このフォルダ）: **長文・判断根拠・将来再開条件**を残す
- 重複は避け、Obsidian 側に「runId: XXXX」を必ず書いて相互参照する
- 大規模変更は ultrareview の結果リンクをここに貼る

---

## ファイル命名

`04_reviews/YYYY-MM-DD-<short-title>.md` 形式。例:

- `2026-05-13-progress-execution-runs-api.md`
- `2026-05-13-nanikiru-shorts-quota-fix.md`

同日複数件は連番ではなく**タイトルで区別**する。

---

## 蓄積ルール

- 既存ファイルは**削除しない**
- 差し戻し・棄却の記録ほど**後で価値が出る**
- 同じ失敗が 2 回出たら CLAUDE.local.md の「蓄積ルール」へ昇格させる
- 月次でフォルダを眺めて、繰り返し出てくるテーマを抽出する

---

## 成果物紹介（必須・Issue #21）

レビュー時は**作成物の場所と使い方**を必ず含める。以下7項目を省略しない:

| 項目 | 書く内容 |
|---|---|
| 何ができたか | 作成/変更した成果物の1行サマリー |
| どこで見れるか | 実ファイルパス（Vault 相対 or リポジトリパス） |
| 何に使うか | その成果物の用途・どの運用に効くか |
| どう使うか | 具体的な使い方（テンプレならコピー先、ルールなら適用タイミング） |
| 次に見るファイル | 関連して次に開くべきファイル（`[[link]]`） |
| 次にやること | フォローアップ／人間判断待ち項目 |
| 注意点 | 破壊禁止・未確認・承認待ち等の警戒事項 |

- vloop/vrun の作業ログ・最終報告・`20_reviews/` 生成物にこの7項目を反映する
- session-review-template（[[../90_templates/session-review-template]]）にも「成果物紹介」セクションを追加済
- 「未確認」を「確認済み」と書かない（CLAUDE.local.md ガード）
