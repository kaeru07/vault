# 調査手段の妥当性評価

> すべての調査結果に **根拠メタ + スコア** を必ず付ける。根拠が弱い調査は candidate 止まりとし、承認に回さない。
> スコア合計が高くても**自動着手しない。必ず pending_approval に置く。**

## 1. 調査結果に必ず付ける根拠メタ（必須）

| 項目 | 記入内容 |
|---|---|
| どこを調べたか | 具体的URL/ストア/媒体名（曖昧不可） |
| いつ調べたか | YYYY-MM-DD（鮮度判定用） |
| 何件見たか | 確認した件数（n=○） |
| 情報源種別 | 一次情報 / 公式 / SNS / 推測 のいずれか明記 |
| 収益性の根拠 | 価格・DL数・課金有無等の具体根拠 |
| 競合の根拠 | 競合数・強さ・空き市場の根拠 |
| 市場 | 日本市場向け / 海外市場向け / 両方 |
| 再現性 | 同手順で再調査して同結論が出るか |
| 情報鮮度 | 情報の最終更新時期（古い場合は要再調査） |

> 「情報源種別=推測」または「何件見たか=不明」のものは **承認候補にしない**（candidate のまま再調査）。

## 2. 評価スコア（各1〜5）

| 軸 | 1（低） | 5（高） |
|---|---|---|
| 収益期待 | ほぼ無収益 | 明確に収益見込み大 |
| 実装速度 | 数ヶ月 | 数日でMVP |
| 既存資産流用 | ゼロから | 既存資産ほぼ流用 |
| 広告相性 | 不向き | 広告と高相性 |
| 動画化相性 | 不向き | Shorts等で拡散しやすい |
| 継続性 | 単発 | 継続的に積み上がる |
| 競合回避しやすさ | レッドオーシャン | 空き市場 |
| 調査根拠の強さ | 推測中心 | 一次情報・件数十分 |

合計 8〜40点。並べ替えの参考にするが、**合計点での自動着手は禁止**。

## 3. スコアの使い方（重要）

- 合計点は **優先順位の参考のみ**
- **収益化インパクトが最優先判断軸**（収益期待 + 調査根拠の強さ を重視）
- 合計点が高くても `pending_approval` に置き、人間承認を待つ
- 「調査根拠の強さ ≤ 2」は合計が高くても承認候補から外す（要再調査）

## 4. 記入例（フォーマット）

```
手段: <収益化手段台帳の行>
調査日: 2026-05-18 / 情報源: 公式ストア(一次) / n=20 / 市場: 日本
収益性根拠: 上位アプリ課金あり・類似で広告収益報告あり
競合根拠: 同カテゴリ上位5本確認、ニッチ軸は空き
スコア: 収益4 速度4 流用3 広告4 動画3 継続4 競合回避3 根拠4 = 29/40
判定: pending_approval（承認待ち。自動着手しない）
```
