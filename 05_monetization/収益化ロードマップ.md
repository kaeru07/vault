> 候補別の具体テーブルは [[収益化候補一覧]] へ。

---

## 現在の最優先方針

- **既存資産（麻雀・将棋・progress）を最短距離で収益化に接続する**
- 新規アプリの立ち上げよりも、**今あるアプリを App Store / Google Play / Shorts に「公開する」**ことを優先
- Claude Code は「実装の高速化」ではなく「公開・告知・運用までの距離を縮める」用途で使う

---

## 重点テーマ（2026 Q2 時点）

- **麻雀「何切る？」Shorts 量産** → 認知獲得 → アプリ DL 誘導
- **shogi-kakoi-trainer の App Store / Google Play 公開**
- **progress アプリのナレッジ資産化**（テンプレ販売・note 化の素材として）

## 時間軸別の方針

### 短期（〜 1 ヶ月）

- nanikiru-shorts: 動画テンプレを 3 パターン量産 → 投稿運用に乗せる
- shogi-kakoi-trainer: App Store 公開タスクを完了
- mahjong-trainer: Vercel デプロイ判断（公開するか / 寝かせるか）
- 収益チャネルの**口座・SSO・申請**だけ先に通しておく（AdSense / App Store Connect / Google Play Console）

### 中期（1〜3 ヶ月）

- Shorts の月間視聴回数を観測して、広告 / アフィリエイト / 自アプリ送客のどれが効くか判断
- 麻雀系を統合した「学習ポータル」構想の検証（mahjong-trainer / quiz / analyzer / nanikiru-shorts）
- Obsidian + Claude Code ワークフローを note 記事化（有料 / 無料の切り分け）

### 長期（3〜12 ヶ月）

- 麻雀・将棋以外への展開（ポーカー学習・AI ニュース・市場調査 SaaS など）
- 自分の作業履歴（Git / progress / Claude Code）から「未収益化資産」を抽出する仕組み（[[../06_research/activity-mining]]）
- 教材販売・コミュニティ運営など、ストック型収益への移行

## チャネル別メモ

### App Store

- 申請に時間がかかる前提で**早めに枠を取りに行く**
- shogi-kakoi-trainer を先行ケースに、Capacitor 化のテンプレを横展開
- 課金: 初期は無料 → 後で機能解放型の課金を載せる

### Google Play

- 競合分析は [[../06_research/GooglePlay市場調査]] を参照
- App Store より審査が緩いので、まずここで反応を見る判断もあり
- 広告モデル（AdMob）との相性を見る

### SNS（X / TikTok）

- X: 開発過程と気付きを淡々と流す。フォロワー数より「検索流入経路」を意識
- TikTok: nanikiru-shorts と相性が良い。コメント数 / 滞在時間を見る

### YouTube Shorts

- 主戦場の 1 つ。nanikiru-shorts の動画はここに最適化
- 「断定しない」「議論を促す」UI 言語で**コメント数を最大化**
- チャンネル登録誘導 → アプリ DL の導線を準備

### AI 活用

- Claude Code は**公開作業の距離を縮める**用途で使う（実装 + 検証 + レビュー + 投稿原稿）
- ChatGPT / Codex はセカンドオピニオン
- progress アプリは作業履歴の正本 → 収益化判断の素材として蓄積

## 優先順位ルール

判断に迷ったら以下の順で評価する。

1. **公開済み or 公開直前** のものを優先（在庫を捌く感覚）
2. **既存ユーザー / 視聴者がいるチャネル** を優先（ゼロ → イチより、イチ → 一〇）
3. **撤退コストが低い** ものから（やってみてダメなら止めやすいか）
4. **収益化までの距離が短い** ものから（マネタイズ手段が既に存在するか）
5. **自分の知識が深い領域** から（麻雀・将棋・AI ワークフロー）

逆に**後回しにすべきもの**:

- 新規アプリの 0 → 1 立ち上げ（既存が完了するまで保留）
- 完璧な仕様検討（公開しないと学習が始まらない）
- 競合のいない市場（だいたい需要が薄い）

## 撤退ライン

- アプリ: 公開後 3 ヶ月で DAU が 一桁 / レビュー反応ゼロなら新規開発を停止
- Shorts: 30 本投稿して平均再生 1,000 未満なら戦略を見直す
- 上記いずれも**ノートとデータは残す**（[[../02_apps/途中プロジェクト一覧]] に移動）
