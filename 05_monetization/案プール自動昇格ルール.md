# 案プール → candidate 自動昇格ルール

> Issue #27。idea_pool（[[idea_pool/_README]] #26）の大量案から**上位だけを candidate 化**するルール。
> **昇格境界の厳守**: `idea → candidate` は AI 可（起票）。`candidate → pending_approval` は妥当性評価完了が必要。
> **`pending_approval → approved` は人間のみ**（[[収益化シナリオ承認フロー]] / [[ChatGPT承認ゲート標準]]）。AI は approved に上げない。

## 昇格段階と主体（厳守）

```
idea（idea_pool）
  │  本ルールのスコア＋安全弁を満たす上位のみ（AI 可）
  ▼
candidate（05_monetization/scenarios/candidate-XXX.md 起票）
  │  妥当性評価8軸完了（[[調査手段の妥当性評価]]）— AI 可
  ▼
pending_approval（ChatGPT 承認待ちに登録）
  │  ChatGPT 方向性承認＋人間 status 確定（[[ChatGPT承認コマンド標準]]）
  ▼
approved（人間のみ）
```

## スコアリング（idea → candidate 選定用）

idea の `粗score`（#26）を妥当性評価8軸の暫定値へ写像し、収益化インパクト重み付き（[[ranking-rule]] と同係数）:

`sortKey = 収益期待×3 + 調査根拠×2 + 既存資産流用×2 + (実装速度+動画化+広告+継続+競合回避)×1`

- 調査根拠は idea 段階では低めに見積もる（一次情報なし＝根拠≤2 が基本）
- 根拠≤2 の案は **candidate 化しても pending_approval に上げない**（要追加調査・[[調査手段の妥当性評価]] §3 準拠）

## 重複除去（昇格前）

- idea_pool の dedupKey（#26）で日内重複除去済み前提
- 昇格時さらに **既存 candidate（scenarios/）と核ワード突合** → 重複は昇格せず「既出」記録
- 既存ファイルは削除しない（重複側を昇格対象から外す方向で解決）

## 上位5件選定ルール

- sortKey 降順で **上位5件まで**を candidate 化対象（5 は上限・ノイズ抑制）
- ただし以下は除外（安全弁）:
  - 根拠≤2 かつ 収益期待≤2（弱すぎ → idea のまま）
  - 外部公開/課金/広告の即時実行が前提の案（candidate 記録は可、実行はしない）
  - 既存 candidate と重複
- 5件に満たなければ満たすだけ（無理に5件にしない＝水増ししない）

## candidate 化の実処理（AI 可・ただし起票まで）

1. 選定された idea を [[../90_templates/収益化シナリオテンプレート]] で `scenarios/candidate-XXX.md` 起票
2. `status: candidate`（**pending_approval にしない**。妥当性評価8軸が一次情報で揃って初めて pending_approval 提案）
3. [[ranking-rule]] で順位再計算 → [[../20_reviews/収益化候補レビュー会議]] に反映
4. **ChatGPT 承認待ち更新**: pending_approval まで達したもののみ [[../20_reviews/ChatGPT承認待ち]] に追記（[[ChatGPT承認コマンド標準]] のコマンド例付き）。idea/candidate 段階は承認待ちに載せない

> 「承認待ち更新」は **pending_approval 到達分のみ**。candidate 起票だけで承認待ちに載せると承認ゲートが形骸化するため。

## 完了条件への対応
- 「上位のみ candidate 化」: ルール定義済。実昇格は idea_pool 生成後（#25→#26 稼働が前提・現時点データなし＝未実行）
- 「承認待ち更新」: pending_approval 到達分のみ更新する規約を定義
- commit/push: 本ルールを commit/push（✅）

## 安全弁（最重要・再掲）
- AI は **approved に絶対しない**（人間のみ）
- AI は candidate を妥当性評価未完了で pending_approval に上げない
- 根拠≤2 は pending_approval 非対象
- スコア上位でも自動着手・自動 progress 投入しない（[[progress連携基準]]）

## 未対応点 / 仮説
- 実昇格は idea_pool（#26）にデータが出てから（順序依存・現状未実行）。本タスクはルール定義まで
- 粗score→8軸写像の係数は初期仮説（運用で decision-log と乖離検証）
- idea_pool 寿命管理（古い idea 間引き）は本ルールと #26 で別途定義予定
