# 05_monetization — 収益化アイデア・検討

収益化に関するアイデア・検討・判断を集約する。**「思いつき」と「検討済み」と「実行中」を分けて記録する**のが目的。

## 既存ノート

- [[ideas]] — 進行中 / 検討中 / 仕込み / 雑記の総合インデックス

## ファイル構成

| 種類 | 命名 | 用途 |
|---|---|---|
| インデックス | `ideas.md` | 進行中・検討中・仕込みの一覧 |
| アイデア単票 | `<idea-slug>.md` | 1 アイデアを掘り下げる場合（テンプレを使う） |

アイデアが育って単独 md に切り出すときは `90_templates/monetization-idea-template.md` をコピーする。

## 書く内容

| セクション | 内容 |
|---|---|
| 概要 | 1 行サマリー / どのアプリ・領域に紐づくか |
| 仮説 | 誰の・どんな課題を・どう解決し・なぜ払うか |
| 収益モデル | 広告 / 買い切り / サブスク / フリーミアム / 教材販売 / アフィリエイト 等 |
| 想定規模 | 単価 × 想定 DL or 訪問数 のラフ試算 |
| 競合 | 既存サービスと差別化ポイント（`06_research/` と相互リンク） |
| リスク | 法令 / 規約 / 競合の参入障壁 / 自分のリソース |
| 検証ステップ | 最小コストで仮説を検証する順序 |
| 状態 | idea / researching / building / live / dropped |

## 書かない内容

- 機密情報（売上実額・契約金額・取引先固有名詞で NDA に該当するもの）
- 仮の数値を断定形で書かない（「想定」「仮置き」と明記）
- 他社の非公開情報

## 運用ルール

- アイデアは「思いついた時点」で書く。完成度は気にしない
- 状態が `dropped` になったものも削除しない。理由を残す（次に似た案が出たときに参照）
- 実行に移ったアイデアは `02_apps/<app>.md` に切り出して、こちらは「実行中」と参照リンクを残す
- 数字を扱うときは出典・日付を必ず添える
- 「税金」「契約」「規約」に関わる判断が出たら `pm/decision-log.md` 側にも 1 行残す
