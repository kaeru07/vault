# 定期調査運用

> いきなり完全自動化しない。**まずは週1回の手動定期調査から**。段階的に半自動→自動へ。

## 段階設計

### 第1段階: 手動定期（現在ここから開始）
- 頻度: **週1回**（曜日固定推奨。例: 日曜）
- 実施: 人間がトリガ → Claude/Codex/ChatGPTで調査 → [[調査手段の妥当性評価]] 付きレポート作成
- 保存: `06_research/YYYY-MM-DD-<topic>.md` + 候補は [[収益化候補一覧]] / シナリオ化
- 開始方法: 「収益化定期調査して（週次）」と依頼 → 収益化手段台帳の未調査/再調査対象を上から1〜3件
- 完了: 妥当性評価付きレポート + candidate/pending_approval 起票（**approvedはしない**）

### 第2段階: 半自動（第1段階が4週continして安定後）
- Claude/Codexが手段台帳を読み、未調査/鮮度切れを自動抽出 → 調査ドラフト生成
- 人間はレビューと承認のみ
- 移行条件: 第1段階を4回以上欠かさず回せた / レポート品質が安定

### 第3段階: 自動（第2段階が安定後）
- cron / GitHub Actions 等で定期トリガ（調査ドラフト生成まで）
- **承認は引き続き人間のみ**（自動化するのは調査生成までで、approved化は自動化しない）
- 移行条件: 第2段階のレポートが連続でレビュー指摘ほぼ無し

## 鮮度・再調査ルール
- 各手段の最終調査日を [[収益化手段台帳]] / 候補一覧で管理
- 90日以上前の調査は「鮮度切れ」→ 再調査対象
- 市場変化が速い手段（GPTs/Chrome拡張等）は30〜60日で再調査

## いきなり自動化しない理由
- 調査ノイズ過多・誤った candidate 量産を防ぐ
- 「AIが勝手に案件を増やして着手」を構造的に禁止（[[収益化シナリオ承認フロー]]）
- まず手動で粒度と品質基準を確立してから自動化する
