# 承認判断テンプレート

> 収益化シナリオを承認 / 却下 / 保留するときの**判断理由記録**用。人間が埋める。
> AI は判断しない（[[../05_monetization/収益化シナリオ承認フロー]]）。記入後 [[../05_monetization/decision-log]] に1行サマリーを追記。

```markdown
---
type: monetization-decision
scenario: candidate-XXX
date: YYYY-MM-DD
decidedBy:            # 役割（owner / pm 等）
transition:           # pending_approval→approved / →rejected / →hold / hold→pending_approval
---

## 判断結果
- 遷移: <例 pending_approval → approved>
- 一言理由: <1行>

## 判断観点（[[../05_monetization/収益化シナリオ承認フロー]] の最小確認項目）
- 収益化インパクト（最優先軸）: high / medium / low / none — 根拠:
- 調査根拠の強さ（一次情報か）: 強 / 中 / 弱（弱なら原則 hold）
- 対象アプリ/手段が明確か: はい / いいえ
- 最初の作業が1つに分解できるか: はい / いいえ
- 完了条件があるか: はい / いいえ

## 却下/保留の場合の理由（該当時）
- 却下理由 / 保留理由:
- 再評価の条件（hold の場合 何が揃えば pending_approval へ戻すか）:

## 付帯メモ（任意）
- 関連調査: [[../06_research/...]]
- 影響する既存資産: [[../05_monetization/既存資産_収益転用候補]]
- 機密は値を書かない（種別のみ）
```

## 使い方

1. 承認会議 / 判断時にこのテンプレをコピー → `05_monetization/decisions/YYYY-MM-DD_candidate-XXX.md` 等に保存
2. シナリオファイルの `status:` / `approvedBy:` を**人間が**書き換え（[[../05_monetization/収益化シナリオ承認フロー]]）
3. [[../05_monetization/decision-log]] の履歴テーブル先頭に1行追記
4. ob sync / commit / push（[[../03_prompts/GitHub反映ルール]]）
