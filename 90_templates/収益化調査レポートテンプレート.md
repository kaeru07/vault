# 収益化調査レポートテンプレート

> 定期調査（[[../05_monetization/定期調査運用]]）の出力。コピーして `06_research/YYYY-MM-DD_<topic>.md` に使う。
> **妥当性評価メタを必ず埋める。埋まらない項目があるものは candidate 止まり（承認に回さない）。**

```markdown
---
title: <調査トピック> 収益化調査
type: monetization-research
date: YYYY-MM-DD
手段: <収益化手段台帳の行（複数可）>
市場: 日本 / 海外 / 両方
status: candidate   # 承認状態はシナリオ側で管理
---

## 調査メタ（必須・1つでも不明なら承認に回さない）
- どこを調べたか: <具体URL/ストア/媒体>
- いつ調べたか: YYYY-MM-DD
- 何件見たか: n=<件>
- 情報源種別: 一次情報 / 公式 / SNS / 推測
- 情報鮮度: <情報の最終更新時期>
- 再現性: <同手順で同結論か>

## わかったこと
- 収益性の根拠: 
- 競合の根拠: 
- 日本/海外: 

## スコア（各1〜5）
| 収益期待 | 実装速度 | 既存資産流用 | 広告相性 | 動画化相性 | 継続性 | 競合回避 | 調査根拠 | 合計 |
|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  | /40 |

## 判定
- 収益化インパクト: high / medium / low（最優先軸）
- 次状態: candidate / pending_approval（**approvedにはしない**）
- 理由: <根拠の強さ・収益期待で判断。根拠≤2なら要再調査>

## 次の一手（承認された場合に備えた1作業案）
- <progress送り用に1つに分解した作業案（承認前は実行しない）>
```

## 注意
- このレポート単体で progress へ送らない（[[../05_monetization/progress連携基準]]）
- 「情報源種別=推測」「件数不明」は承認候補から除外し再調査
