# 月次収益レビュー テンプレート

> 毎月の収益化活動を振り返り方向修正する。コピーして `20_reviews/月次収益レビュー.md` の月次セクションに使う。
> 数値は事実のみ（推測は「推測」と明記）。**方針変更の確定は人間**（[[../05_monetization/収益化シナリオ承認フロー]]）。

```markdown
## YYYY-MM 月次収益レビュー

### 1. 今月の収益（事実）
- 実収益: <金額 / ゼロなら 0 と明記>（情報源: <ストア/広告管理画面 等>）
- 未確認の数値: <あれば「未確認」と明記。確認済みと書かない>

### 2. 実施施策
- <candidate-XXX / Issue#XX で何をしたか・commit>

### 3. 効いたこと / 効かなかったこと
- 効いた: <根拠とともに>
- 効かなかった: <根拠とともに → idea-graveyard / 失敗レビュー 候補>

### 4. パイプライン状況（[[../05_monetization/収益化ステップ一覧]] と突合）
- candidate / pending_approval / approved / in_progress / published の件数
- 律速（どこで止まっているか）

### 5. 次月方針（案・確定は人間）
- 収益化インパクト最優先で次月に進める候補:
- hold / 撤退を提案する候補:
- ルール化したい学び（→ CLAUDE.local.md 提案）:

### 6. 次の一手（1つ）
- <ユーザーが次に取る具体行動>
```

## 運用
- 毎月初（または月末）に1セクション追記。過去月は消さない
- approved 変更・撤退の確定は人間。AI は数値集計と案提示まで
- 完了後 ob sync / commit / push（[[../03_prompts/GitHub反映ルール]]）
