# 市場調査ログ 2026-05-28

## 今日の結論
次に見るべきは「AIで作られた小型アプリが、ストア内発見・課金・継続率で勝てる余地がどこに残っているか」。
Google Play は AI 検索・Play Shorts・AI Studio 連携で、アプリ発見と試作の入口が変わりつつある。
収益化では汎用AIチャットより、議事録、業務自動化、動画編集、学習補助のような「明確な利用頻度」がある領域が有望。
MCP / エージェント連携は個人開発でも小規模SaaS向けのプラグイン型収益導線になり得る。

## 競合アプリ
- ChatGPT / OpenAI / 汎用AIアシスタント、App Store生産性カテゴリ上位 / 強み: 圧倒的認知と課金転換 / 弱み: 個別業務に特化しないと差別化困難
- Otter / Otter.ai / 音声文字起こし・議事録 / 強み: 明確な業務用途と継続利用 / 弱み: 日本語・業種特化・後処理テンプレに隙間あり
- Gemini / Google / Android・検索・Play連携のAI導線 / 強み: OS・ストア・検索面の配布力 / 弱み: 個人開発者が同じ土俵で汎用AI競争をするのは不利
- Claude / Anthropic / プロ用途、コード、調査、業務支援 / 強み: 有料ユーザー成長と業務文脈 / 弱み: 消費者向けの軽量ユースケースではChatGPTより弱い可能性
- CapCut / ByteDance / AI動画編集・SNS向け制作 / 強み: 動画投稿者の高頻度利用 / 弱み: 大型編集スイートのため、特定テンプレ・自動化に細分化余地

## 収益化ヒント
- 議事録・音声メモ系は無料枠＋月額課金、文字起こし時間制限、業種別テンプレ課金が合う。
- Google AI StudioでAndroid試作を量産し、Play Storeでは広告よりも軽量サブスクか買い切り追加機能を優先する。
- SMB向け自動化は「1業務1エージェント」に絞り、初期設定代行＋月額保守で収益化する。
- MCP対応ミニツールはChatGPT / Claude内から使える外部業務アクションとして、成功報酬・手数料・SaaS課金を設計する。
- Shorts / TikTok向けには「調査ログから動画台本生成」「アプリ比較台本生成」など、制作補助ツールをnote・テンプレ販売に接続する。
