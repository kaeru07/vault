# 市場調査ログ 2026-06-27

## 今日の結論
次に見るべきは「既存作業面に入り込む小型AI」と「AI動画の収益化テンプレ」の組み合わせ。
単体AIアプリより、ブラウザ、IDE、動画編集、広告管理、音声入力など既存導線へ差し込む形が強い。
MCP / AI Agent は企業向けでは権限管理と監査、個人開発では定型業務の自動化に寄っている。
Shorts / TikTok / YouTube 系はAI動画生成の導線が増えているが、収益化可否や品質判定は未確認要素が大きい。

## 注目ジャンル
- AI音声入力 / 入力速度改善 / Wispr Flow、iOS、Android、macOS
- AIコーディングエージェント / 開発時間短縮と高単価SaaS化 / Claude Code、Codex、Cursor
- MCP・エージェント権限管理 / 企業導入時の監査・安全性ニーズ / Zendesk、Arcade.dev、Anthropic MCP

- 既存ツール埋め込み型AI / 新規アプリの集客難を回避 / Product Hunt、ブラウザ拡張、Slack、Notion、GitHub

## 競合アプリ
- Wispr Flow / Wispr AI / 全アプリ横断のAI音声入力 / 強み: 入力頻度が高く継続課金しやすい / 弱み: OS連携・認識精度・プライバシーへの不満が出やすい
- Claude Code / Anthropic / コード編集・テスト・レビューを行うエージェント / 強み: 開発者向け高単価課金に向く / 弱み: トークン費用、品質ばらつき、複雑タスクの信頼性は未確認
- ChatGPT / OpenAI / 汎用AIチャット・音声・画像・業務補助 / 強み: 認知度とモバイル導線 / 弱み: 個人開発が正面競争すると差別化が難しい
- CapCut / ByteDance / AI動画編集・テンプレ・字幕・アバター / 強み: SNS投稿までの導線が短い / 弱み: 汎用化が進み、単純な動画生成アプリは埋もれやすい
- Product Hunt上のAI Browser系 / 提供元は複数 / ブラウザ操作・スクレイピング・予約実行 / 強み: 業務自動化に直結 / 弱み: CAPTCHA、利用規約、安定性、権限管理が課題

## 収益化ヒント
- 月額課金: 音声入力、議事録、コードレビュー、ブラウザ自動化など「毎日使う入力・確認作業」に課金。
- 従量課金: AIコードレビュー、AI動画生成、広告クリエイティブ生成など、計算コストに比例する機能。
- 広告・アフィリエイト: AI動画テンプレ、Shorts台本、商品紹介動画を生成し、YouTube / TikTok / noteへ展開。
- B2B小額SaaS: 個人事業者向けに「問い合わせ返信」「予約調整」「請求書下書き」など限定業務のエージェント化。
- note / 動画教材: 「Claude Code / Codexで1日で作るAI業務ツール」型の制作過程を販売・動画化。
