## 各案 詳細（必須 18 項目）

### 1. 湯巡りログ -onsen-

- 一言: 訪問した温泉と泉質をローカル記録、47 都道府県スタンプラリー
- 想定ユーザー: 30-60 代の温泉巡り好き / 車で日帰り温泉に行くシニア層
- なぜ今: 円安で国内旅行回帰 / シニアのスマホ慣れ進行 / Untappd 的「リアル巡り×記録」が再注目
- API 不要理由: 全国温泉約 3,000 件は静的 JSON で内蔵可能、地図はオフラインベクタータイル
- 保存データ: 訪問日 / 場所 / 泉質 / 効能 / 写真 / 評価 / メモ
- 蓄積: あり（年間訪問数、県制覇率、泉質コレクション）
- 毎日開く仕組み: 帰宅後すぐ記録、月間スタンプカード、月初リマインダー
- 収益化: 広告バナー / 月 300 円で写真無制限 / 図鑑追加コンテンツ買い切り
- MVP 機能: 訪問記録 + 都道府県マップ + スタンプ表示
- 差別化: 泉質コレクション、シニア向け大文字 UI
- 競合リスク: ニフティ温泉（記録機能弱い）、サウナイキタイ（サウナ特化）
- タイトル案: 「湯巡りログ - 温泉スタンプ帳」「温泉手帳 -onsen log-」「47 都道府県温泉制覇」
- ASO: 温泉, スタンプ, 湯巡り, 泉質, 日帰り温泉, 旅行記録
- SNS 投稿: 月まとめ画像 / 県制覇達成 / 泉質コレクション
- 実装難易度: 2 / 収益期待: 4 / 継続率: 4 / 海外展開: 2

### 2. 打牌セルフ採点 -tehai-

- 一言: 自分の麻雀対局を手入力、ルールベースで「危険牌」「裏目」を採点
- 想定ユーザー: 麻雀好き 20-50 代 / 雀魂・天鳳プレイヤー / 何切る勢
- なぜ今: 麻雀人気再燃（Mリーグ視聴増）/ AI なしのルール採点でストア審査と相性良
- API 不要理由: 麻雀ルール・シャンテン数・危険牌判定はすべて純ロジック（PRNG・牌譜パースのみ）
- 保存データ: 対局日 / 半荘 / 4 局分の自分の打牌 / シャンテン / 採点
- 蓄積: あり（誤打パターン傾向、月間採点平均）
- 毎日開く仕組み: 1 日 3 局入力で連続スタンプ、誤打 TOP10 復習問題化
- 収益化: 広告 / 月 480 円で詳細統計・月別グラフ / 何切る問題集追加課金
- MVP 機能: 牌入力 UI + シャンテン計算 + 危険牌スコア
- 差別化: AI なしのため誤判定 0（ルール明確）、Mリーグ実戦の写経モード
- 競合リスク: 雀魂アプリ本体に分析機能あり、ただし「自分の打牌の振り返り」専用は無い
- タイトル案: 「打牌セルフ採点 -tehai-」「麻雀振り返り帳」「何切る記録帳」
- ASO: 麻雀, 牌譜, 何切る, 雀魂, 天鳳, 振り返り
- SNS 投稿: 月間誤打 TOP3 / 採点グラフ / 何切る投票
- 実装難易度: 3 / 収益期待: 4 / 継続率: 5 / 海外展開: 4

### 3. 盆栽育成カレンダー

- MVP 機能: 樹種登録 + 写真 + 作業カレンダー + 通知
- 差別化: 海外バイリンガル UI（英 / 日）、経年タイムラプス機能
- 競合リスク: 既存「Bonsai Empire」アプリ、ただし日本式樹種データは弱い
- タイトル案: 「盆栽カレンダー」「Bonsai Calendar -盆栽日記-」「樹歴ノート」
- ASO: 盆栽, bonsai, 園芸, 剪定, 水やり, 趣味

## 最初に作るべき本命アプリ 3 つ

### 本命 1: 打牌セルフ採点 -tehai-

- スコア合計: 難 3 / 収 4 / 継 5 / 海 4 = 16
- なぜ本命:
  - **既存運用との親和性**: 既に `02_apps/mahjong.md` `02_apps/nanikiru-shorts.md` がある。麻雀ドメインを深掘りすれば Shorts チャネルとの相互送客が可能
  - **API なしで勝てる**: 麻雀の打牌評価は AI なしのルールベースで成立する稀少ジャンル。むしろ「断定しない / 説明できる」UI が強み
  - **継続率最強**: 1 日 1 半荘でも入力動機が生まれる。蓄積で振り返り価値が増す
  - **収益化動線**: 詳細統計サブスク + 何切る問題集追加課金 + Shorts 流入で広告
- 着手の最初の壁: 牌入力 UI の使い勝手と、シャンテン数判定の純粋ロジック実装
- MVP 範囲: 1 半荘 4 局分の自分の打牌入力 → シャンテン推移 → 危険牌スコア
- リスク: 雀魂本体が分析機能を強化すると差別化が薄まる → 「自分の打牌だけを長期蓄積」に絞る

### 本命 2: 朝散歩カレンダー

- スコア合計: 難 1 / 収 4 / 継 5 / 海 5 = 15
- なぜ本命:
  - **1 週間で MVP 可能**: 1 タップ記録＋連続日数のみで実装最軽量
  - **海外展開即可**: 言語非依存、海外健康市場巨大
  - **継続率最強**: 朝の習慣化アプリは Streak（連続日数）と相性最良
  - **シニア層**: 既存 nanikiru 等で持つアセットとは別チャネルを開拓できる
- 着手の最初の壁: 通知設計と「サボった日」の UI（罪悪感を負わせず再開動機を作る）
- MVP 範囲: 1 タップ記録 + 連続日数 + 月カレンダー + 朝 6:00 通知
- リスク: 歩数計系の競合多い → 「朝の起床＋散歩」習慣化に絞り、歩数計と差別化

### 本命 3: 庭タイムラプス

- スコア合計: 難 3 / 収 3 / 継 5 / 海 5 = 16

## 共通の戦略ヒント

- **3 つすべて Streak（連続日数）モデル**: 朝散歩・庭タイムラプス・打牌セルフ採点はすべて 1 日 1 回開く動機がある
- **3 つともシンプル MVP**: 1〜2 週間で β 版が出せる規模、収益化検証を高速で回せる
- **共通基盤を作れる**: 写真記録 + 連続日数 + 月別ビュー + 通知 + ローカル DB（SQLite）は共通化可能
- **Shorts/TikTok との相互送客**: 庭タイムラプス → ガーデニング系チャネル / 打牌セルフ採点 → 麻雀チャネル
