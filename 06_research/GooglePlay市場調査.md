# Google Play 市場調査

> Google Play での公開・運用に向けた**調査の集約ページ**。麻雀・将棋・問題集系の仮説を中心に置き、調査結果を継ぎ足していく。
> 個別調査は `90_templates/market-research-template.md` から `06_research/<topic>.md` に切り出す。

---

## 調査観点

| 観点 | 何を見るか |
|---|---|
| **ダウンロード数** | カテゴリ Top アプリの DL レンジ / 自分の現実的な目標値 |
| **レビュー** | 件数・評価・上位の不満と要望 |
| **広告** | バナー / インタースティシャル / リワードの混在度・違和感 |
| **ASO** | アプリ名キーワード / 説明文 / アイコン / スクショ |
| **競合** | 上位 5 アプリの強みと弱み |
| **更新頻度** | 直近 6 ヶ月の更新回数（活性度） |
| **価格** | 無料 / 買い切り / サブスク / 広告 / フリーミアム |
| **対応 OS / 言語** | 日本語のみ / 多言語 / Android 最低バージョン |

---

## ダウンロード数の目安

- 初期目標: 数百 DL（自分の SNS / Shorts 経由）
- 損益分岐: 広告依存なら 数千 〜 数万 DL、課金なら 数百ユーザー × 課金率
- ASO だけで Top 10 入りは難しい前提 → **外部チャネル（Shorts / X）**から流入を作る

---

## レビューから読み取るべきこと

- 競合のレビューに「**こうだったら使うのに**」という不満が出ていれば差別化余地
- 評価 1〜2 のレビューは**外せない地雷**の発見に最適
- 評価 4〜5 のレビューは**ヘビーユーザーの動機**の発見に有用
- 「広告がうざい」が多いカテゴリ → 課金型 or 控えめ広告で差別化可能性

---

## 広告

- AdMob のフォーマット相性:
  - バナー: 邪魔だが安定
  - インタースティシャル: 問題 N 問ごと等の自然タイミングで
  - リワード: ヒント解放など能動的選択に紐付ける
- カテゴリ別広告単価は将棋 < 麻雀 < ビジネス系の傾向（一般論。要検証）

---

## ASO

- アプリ名: メインキーワードを先頭近くに
- 短い説明: 1 行で「**誰の何を解決するか**」を言い切る
- スクショ: 1 枚目で機能でなく**ベネフィット**を見せる
- アイコン: 視認性 + カテゴリの中で**異質に見える色**を選ぶ
- 多言語化: 短期は日本語のみで OK

---

## 競合（カテゴリ別の仮説）

### 麻雀「何切る」系

- 競合は**「正解はこれ！」型**が多い
- → 差別化軸: 「断定しない」「議論を促す」「派閥を立てる」（[[../02_apps/nanikiru-shorts]]）
- 仮説: Shorts と連動した認知拡大が効くカテゴリ

### 将棋囲い・定跡系

- 既存アプリは初心者 〜 中級者向けが多い → 「**囲い**」だけに絞る差別化
- 中高年向けの UX（大きめ文字・音声）で**未開拓**を狙う仮説（[[../02_apps/shogi-trainer]]）

### 問題集系（一般）

- 「学習継続」を支える UX が勝負（ストリーク・コレクション）
- 通知設計が雑なアプリは離脱が大きい → 「**邪魔しない通知**」で差別化可能

---

## 麻雀 / 将棋 / 問題集系の仮説（まとめ）

- **麻雀**: Shorts 連携 + コミュニティ感で勝負。アプリ単体より**ポータル統合**で評価する
- **将棋**: 中高年向け縦軸を切り取る。Android で先に出して反応を見る
- **問題集**: コンテンツ量（JSON ベース）と継続 UX が肝。AI 量産との相性が良い
