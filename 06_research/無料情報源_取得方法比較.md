# 無料情報源 取得方法・難易度 比較

> Issue #29。情報収集自動化（#25 / #28）の着手前に、**無料で取得できる 8 情報源について取得方法・難易度を整理**する。
> **本ファイルは机上調査整理のみ**。実取得・スクレイパ実装はしない（実装は承認後・別アプリリポジトリ）。
> 規約適合は実装着手時に**各源 1 つずつ最新規約を再確認する**（規約は時期で変わるため）。

## 1. 比較表（8 源 × 6 観点）

| # | 源 | 取得方法（想定） | スクレイピング可否（公開部分） | ログイン要否 | レート制限（想定） | 難易度 | 無料性 |
|---|---|---|---|---|---|---|---|
| 1 | Google Trends | 公開トピック RSS / Daily Trends RSS / 公開ページ | 一部 RSS あり = 可 / HTML スクレイプは要規約確認 | 不要 | 公開 RSS は緩い / HTML は要配慮 | 低〜中 | 完全無料 |
| 2 | Google Play | ストア公開ページ HTML / カテゴリランキングページ | 公開ページ = 可（ただし利用規約要確認） | 不要（公開閲覧） | 不明（実装時に要観測） | 中 | 完全無料 |
| 3 | App Store | ストア公開ページ HTML / iTunes Search API（公式・無料） | iTunes Search API 推奨 / HTML はサブ | 不要（公式 API） | 公式 API のレート公表あり | 中 | 完全無料（公式 API） |
| 4 | Reddit | 公開 JSON エンドポイント（`/r/<sub>/top.json`） | 可（公式公開 JSON） | UA 必須・トークンなしでも可 | 公式表記あり（要 UA） | 低 | 完全無料 |

| 8 | AI ニュース | RSS 集約（Hacker News / arXiv / 主要ブログ RSS） | RSS = 可（規約順守） | 不要 | RSS は緩い | 低 | 完全無料 |

> **未確認は「未確認」と明記**（規約変動・取得可否はライブ確認が必要なため）。

## 2. 取得方法の解像度（実装時の選択肢）

### 公式 API ありを優先する源

- App Store: **iTunes Search API**（公式・無料・キー不要）— 第 1 優先
- YouTube: **YouTube Data API v3**（公式・無料枠あり・キー必要）— 第 2 優先
- 公式 API はレート・利用規約が明確で実装/監視が容易

### 公開 RSS / JSON があるものを次に優先

- Google Trends: Daily Trends RSS（地域別）
- Reddit: `/r/<sub>/top.json` / `/r/<sub>/new.json`（UA 必須）
- AI ニュース: 各種 RSS フィード集約

### HTML スクレイピングが必要な源

- Google Play / Product Hunt / X（公開ページ）
- 実装時の注意:
  - User-Agent を明示（個人/プロジェクト名 + 連絡先）
  - レート制限を保守的に設定（1 req / 2〜5 秒目安）
  - robots.txt と利用規約の最新版を**実装着手時に必ず再確認**
  - 取得失敗の連続（例: 3 日連続）は規約変更や IP ブロックの兆候 → 停止して人間判断

## 3. 難易度と優先順位（実装着手順の推奨）

| 順 | 源 | 推奨理由 | Phase |
|---|---|---|---|
| 1 | Reddit（公開 JSON） | 公式公開・UA だけで動く・低リスク | Phase 1（最初の 1 源） |
| 2 | App Store（iTunes Search API） | 公式・無料・キー不要 | Phase 1 |
| 3 | AI ニュース（RSS 集約） | RSS で枯れている・低リスク | Phase 1 |
| 4 | Google Trends（Daily Trends RSS） | RSS で取得可・規約緩め | Phase 2 |
| 5 | YouTube（Data API v3） | 公式 API・無料枠あり・キー管理必要 | Phase 2 |
| 6 | Product Hunt（HTML） | HTML スクレイプ要配慮 | Phase 3 |
| 7 | Google Play（HTML） | HTML スクレイプ要配慮・カテゴリ毎の構造変動 | Phase 3 |
| 8 | X（公開トレンド） | 規約変動激しい・優先低 | Phase 4（要再評価） |

> Phase 1（Reddit / iTunes / RSS）から開始すれば**実装難易度・規約リスクが最も低い 3 源で最初の daily/ 出力を作れる**。

## 4. 機密・規約ガード（実装着手時の必須チェック）

実装直前に必ず確認:

- [ ] 各源の利用規約（最新版）を 1 つずつ目視
- [ ] robots.txt（HTML 源）を取得側 IP で確認
- [ ] API キーが必要な源（YouTube）はキー管理ルール（[[../03_prompts/GitHub反映ルール]] 機密ガード）
- [ ] 取得 raw に個人情報・認証情報・トークンが混入しないか保存前スキャン
- [ ] レート制限を保守的に（公表値の半分以下を目安）

これらが満たせない源は**当該 Phase をスキップして次の Phase へ**進む。
