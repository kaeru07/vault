# 市場調査方法レビュー 2026-06-06

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいかを最優先に見た。  
対象は App Store / Google Play、AIニュース、AIツール、MCP、AI Agent、AI自動化を中心にした。  
動画化しやすさでは AI動画生成、Shorts / TikTok、クリエイター支援AIを重く見た。  
日本 / 海外両方の需要は意識したが、実際の取得は海外ソース寄りだった。

## 良かった点
- AI動画生成、meal planner、メッセージングAIエージェントなど、課金導線に近いテーマを拾えていた。
- App Store / Google Play のレビュー不満を、課金・解約・生成失敗という収益化に直結する観点で整理できていた。
- Codex、GitHub Copilot、Buzzy Builder MCP など、個人開発の制作速度に影響するツールを拾えていた。
- 公式情報と二次情報、推測、未取得情報を分ける品質チェックが機能していた。
- 「作れそうなアプリ案」が低〜中難度に寄っており、個人開発の次アクションに落としやすかった。

## 弱かった点
- 昨日ログ未取得により、差分・重複・新規性の判定が弱かった。
- TikTok / Shorts の実投稿が未取得で、動画化しやすさの根拠が不足していた。
- App Store / Google Play の当日ランキング実順位がなく、競合選定がやや恣意的だった。
- 日本語圏の需要確認が薄く、海外で見えた不満が日本市場でも成立するか未検証だった。
- 価格、CVR、市場規模、実購入意欲までは確認できておらず、収益化判断が仮説止まりだった。

## 情報源の偏り
今日カバーした情報源は、App Store、Google Play、OpenAI、Microsoft、Google Developers Blog、Anthropic、GitHub、Workday、NetFoundry、TechCrunch、Product Hunt、PR配信、TikTok Business レポートだった。  
不足していた情報源は、TikTok / Shorts の実投稿、App Store / Google Play のランキング実データ、日本語レビュー、X / Reddit / YouTube コメント、実際の料金表・導入条件、前日レポートだった。  
特に AI ニュースと AI ツールは公式発表寄りで、ユーザー側の利用実感や不満の一次情報が少なかった。

## 収益化判断の精度
「AI動画の課金透明性」「meal planner の家族共有・買い物導線」「小規模店舗向けDM返信AI」は、実際の支払い理由に近く、収益化仮説としては比較的強い。  
一方で、Buzzy Builder MCP、NetFoundry、Workday、Microsoft Web IQ は開発・運用基盤としては重要だが、個人開発で直接売上に変えるには距離がある。  
作れそうなアプリ案は実装難度の見積もりは妥当だが、集客チャネル、価格帯、初期ユーザー像が不足していたため、収益化判断の精度は中程度に留まる。

## 見落とした可能性
- 日本語圏の AI動画アプリ、献立アプリ、LINE連携AIのレビュー不満。
- AI動画生成の返金・クレジット消費ルールを比較する具体的な価格表。
- TikTok / Shorts で実際に伸びている AI動画・AIレシピ投稿の型。
- MCP / Agent 系ツールの OSS 代替と、個人が月額で使える価格帯。

## 次回改善すること

- Google Play / App Store のレビューを最低3アプリ横断で、課金・品質・解約・UIに分類する。
