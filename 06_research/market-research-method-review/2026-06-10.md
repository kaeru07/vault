# 市場調査方法レビュー 2026-06-10

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、短尺動画、App Store / Google Play 周辺を中心に見た。  
汎用AI本体との競争を避け、出力整形、配信、広告化、業務ログ化など周辺市場に重きを置いた。  
動画化しやすい題材として、AI動画広告、Codex / Claude Code / MCP活用、AI開発ツール比較を意識した。

## 良かった点
- AI本体ではなく、成果物の再利用・配信・業務接続という個人開発向けの隙間に寄せられた。
- MCP / Agent / Remote MCP / Claude Code Agent SDK など、Codex活用に直結する調査ができた。
- 各ファイルで「公式」「二次情報」「推測」「未取得」を分け、根拠の強弱を明記できた。
- 収益化ヒントが、サブスク、クレジット課金、広告収益、教材、B2B小額課金に分散していた。
- 「取得状況: partial」と不足点を明示し、過信しない構成にできた。

## 弱かった点
- App Store / Google Play の公式ランキング確認が不足し、実際に伸びているアプリとの差分が弱い。
- SNS一次投稿、レビュー、Product Hunt コメントなど、ユーザー不満の一次情報が少ない。
- Claude Fable 5、Apple-Gemini連携、ChatGPT MAU 10億人など重要情報が二次情報依存だった。
- 収益化案に対して、想定価格、MVP画面、集客導線、初期検証方法がまだ浅い。
- AIツール調査が開発者向けMCP/Agentに寄り、一般消費者向けAIアプリの観察が薄かった。

## 情報源の偏り
今日カバーした情報源は、The Verge、Axios、TechRadar、Business Insider、Economic Times、Android Central、arXiv、OpenAI / Claude Code / GitHub / MCP 公式ドキュメントが中心だった。  
不足していた情報源は、App Store / Google Play の国別ランキング、公式価格ページ、TikTok / Reddit / X / Product Hunt の一次投稿、実ユーザーレビュー、開発者フォーラム。  
ニュース媒体と公式ドキュメントには寄れたが、実需要を示すランキング・レビュー・投稿データが不足していた。

## 収益化判断の精度
収益化ヒントは方向性としては近い。特に「AI成果物リパッケージャー」「商品URL to 縦動画広告ブリーフ」「Agent実行ログ・権限管理」は、短期MVPに落としやすい。  
一方で、実際に誰がいくら払うかの検証が不足している。競合価格、検索需要、既存ツールの不満、配布チャネルを確認できていないため、収益化確度は中程度に留まる。  
今日の判断は「作れそう」には近いが、「売れそう」の裏取りはまだ弱い。

## 見落とした可能性
- AI動画広告ツールの具体的な料金、商用利用条件、出力制限、解約理由。
- App Store / Google Play のAIノート、AI学習、AI写真、AI動画カテゴリの国別上位変化。
- 個人開発者がMCPサーバーを販売・配布する場合のマーケット、課金方法、セキュリティ不安。
- 日本語圏のnote、X、YouTube Shortsで実際に売れているAI教材・テンプレ商品の観察。

## 次回改善すること
- App Store / Google Play のAIチャット、AI動画、AIノート、AI学習アプリ上位を国別に最低10件確認する。
- Reddit、Product Hunt、X、アプリレビューから、AI動画広告またはAI Agent運用の不満を最低2系統で拾う。
- 収益化案ごとに、想定価格、初期MVP画面、初回集客チャネルを1つずつ書く。
- OpenAI / Anthropic / Google / Apple / GitHub は、二次報道の前に公式リリースまたは公式ドキュメントを確認する。
- AIツール調査に、開発者向けツールだけでなく一般ユーザー向け新規AIアプリを最低1枠入れる。
