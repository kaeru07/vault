# 市場調査方法レビュー 2026-07-01

## 今日使った調査方針
収益化しやすく、短期リリース可能で、Claude Code / Codex による量産が効くテーマを優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、App Store / Google Play、Shorts / TikTok、学習・麻雀・将棋系を中心に見た。  
特に「小型 AI アプリ」「AI Agent 実行基盤」「MCP 安全性」「ショート動画導線」に重きを置いた。  
一方で、実ランキング・実レビュー・国内 SNS の一次確認は不足した。

## 良かった点
- AI ツール調査は GitHub / Cloudflare / Vercel など公式情報中心で、根拠の質が比較的高かった。
- Codex / Claude / Gemini などのモデルニュースを、個人開発のワークフロー改善に接続できていた。
- MCP / Agent 系を「作る側」だけでなく「権限チェック・安全性診断」という収益化切り口で見られた。
- 市場調査では、Google Play の会話型アプリ発見と Play Shorts を小規模アプリ露出の観点で拾えた。
- 各ファイルに品質チェックがあり、公式未確認・推測・不足データを明示できていた。

## 弱かった点
- 市場調査と AI ニュースは二次情報依存が多く、公式発表・一次資料の確認が不足した。
- App Store / Google Play の当日ランキング、価格、レビュー不満を直接確認できていない。
- 日本市場の観測が弱く、海外メディア・GitHub 系・英語圏 AI 開発者向け情報に偏った。
- 収益化案は妥当だが、価格帯・競合課金・継続率の見立てが薄く、実際の売上距離はまだ粗い。
- 昨日ログ未取得により、新規性と重複率の判定が弱かった。

## 情報源の偏り
今日カバーした情報源は、Android Central、The Verge、Axios、Business Insider、FT、TechRadar、GitHub Changelog、Cloudflare Docs、Vercel AI SDK docs、arXiv が中心だった。  
公式情報は AI ツール調査では比較的取れていたが、市場調査と AI ニュースでは OpenAI / Anthropic / Google / Apple / Google Play 公式が不足した。  
不足していた情報源は、App Store / Google Play の実ランキング、国内 App Store レビュー、TikTok / YouTube Shorts の実投稿、X / Reddit / Product Hunt、国内個人開発者の一次発信。  
結果として、開発者向け Agent / MCP 情報に強く、一般ユーザー向けアプリ需要の確認が弱かった。

## 収益化判断の精度
「収益化ヒント」は、AI 予定分解、Shorts 台本生成、MCP 権限チェッカー、復習カード生成など、小さく作れる案に寄っており方向性は良い。  
ただし、競合の価格・無料枠・レビュー不満・検索流入・動画導線を直接比較していないため、収益化に近いかの精度は中程度。  
「作れそうなアプリ案」は実装難度の見積もりは現実的だが、誰が最初に課金するか、どの導線で獲得するかの具体性が不足した。  
今日の判断は「開発可能性」は高め、「需要検証」と「課金検証」は不足という評価。

## 見落とした可能性
- App Store / Google Play の日本ランキング内で伸びている非 AI 名義の実用アプリ。
- TikTok / YouTube Shorts で実際に再生されている AI アプリ紹介、麻雀・将棋・学習系の投稿導線。
- Stripe、Notion、Supabase、Firebase など、個人開発 SaaS に直結する MCP / Agent 連携。
- レビュー低評価から拾える「既存 AI プランナー・動画編集アプリへの具体的不満」。

## 次回改善すること
- Google Play / App Store の日米ランキングを最低 1 回確認し、競合 3 件の価格・無料枠・レビュー不満を分けて記録する。
- OpenAI / Anthropic / Google / Apple / Google Play は、二次記事の前に公式ブログ・公式 docs・changelog を確認する。
- TikTok または YouTube Shorts で、AI アプリ系キーワードを 3 つ検索し、再生数・投稿頻度・誘導先を記録する。
- GitHub 系に偏らないよう、MCP / Agent は Stripe、Notion、Supabase、Firebase、Cloudflare から最低 2 系統を見る。
- 昨日分 Markdown を参照し、各テーマに「新規」「継続」「重複疑い」を付ける。
