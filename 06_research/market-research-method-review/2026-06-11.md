# 市場調査方法レビュー 2026-06-11

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI 動画編集、AI エージェント、MCP、学習アプリ、AI 自動化に重きを置いた。  
動画化しやすいテーマとして、CapCut / Edits / Pippit、AI ミニドラマ、Gemini / World Cup 文脈を拾った。  
日本 / 海外両方の需要を見る方針は意識したが、実際は海外二次情報と公式ドキュメント寄りだった。

## 良かった点
- 収益化に近い「動画制作ワークフロー」「MCP 運用」「小型 SaaS 管理画面」に論点を寄せられた。
- AI ツール調査では公式ドキュメント中心に確認でき、Figma MCP、Codex、Cloudflare Agents MCP など実装に近い情報が多かった。
- 市場調査、AI ニュース、AI ツールの間で「Agent / MCP / 小規模業務自動化」という接続が見えた。
- 収益化ヒントが、生成 AI そのものではなくテンプレ、分析、権限管理、導入支援に寄っていた点は現実的だった。
- 品質チェックで未確認情報、二次情報、推測を分けて書けていた。

## 弱かった点
- App Store / Google Play の当日ランキングを直接確認できておらず、需要の鮮度が弱い。
- TikTok / YouTube Shorts の実投稿、ハッシュタグ、再生数の直接観察がなく、動画領域の根拠が二次情報に寄った。
- 昨日との差分が未取得のため、継続テーマと新規テーマの切り分けができていない。
- AI ニュースは IPO 観測など大きな話題に寄り、短期で作れる収益化ネタへの変換がやや粗い。
- 収益化案に月額単価、顧客像、初期販売チャネルの具体性が不足していた。

## 情報源の偏り
今日カバーした情報源は、公式サイト・公式ドキュメント、OpenAI / Figma / GitHub / Cloudflare / MCP 公式、Axios、The Verge、Business Insider、WSJ、Wired、arXiv が中心だった。  
不足していた情報源は、App Store / Google Play の国別ランキング実データ、TikTok / YouTube Shorts / Instagram Reels の実投稿、Product Hunt、GitHub Trending、Hacker News、日本語圏の個人開発者・クリエイター投稿。  
特に市場調査側は、公式ランキングと SNS 実地観察が弱く、ニュース側は二次報道依存が目立った。  
AI ツール調査は一次情報が多く比較的堅いが、新規ツールの網羅性は不足していた。

## 収益化判断の精度
収益化ヒントは、AI 動画台本、字幕テンプレ、投稿分析、MCP 導入支援、目的特化型語学アプリなど、個人開発で小さく検証しやすい方向に寄っており精度は中程度。  
一方で「誰が最初に払うか」「いくらなら払うか」「既存ツールではなぜ足りないか」の確認が弱く、売上に近い判断までは届いていない。  
作れそうなアプリ案は実装可能性はあるが、競合との差別化と集客導線の検証が不足している。  
今日の判断は「候補抽出」としては有効だが、「収益化優先順位の決定」としては追加調査が必要。

## 見落とした可能性
- 日本語圏の CapCut テンプレ販売、ショート動画代行、AI 動画教材の実際の価格帯。
- App Store / Google Play で急上昇している小規模 AI アプリ、特に日本ランキング内のニッチアプリ。
- Product Hunt / GitHub Trending 由来の新規 Agent / MCP ツールで、個人開発者がすぐ模倣できるもの。
- AI ツール導入に困っている中小企業・個人事業主の具体業務、請求・予約・投稿・問い合わせ対応などの現場課題。

## 次回改善すること
- App Store と Google Play を日本・米国で確認し、AI 動画編集、学習、自動化カテゴリから実アプリ名を最低 5 件ずつ拾う。
- TikTok / YouTube Shorts / Instagram Reels で AI ミニドラマ、CapCut テンプレ、AI 広告動画の実投稿を確認し、再生数・投稿形式・CTA を記録する。
- 前日 Markdown を読み込み、各テーマを「継続」「新規」「重複疑い」に分類してから本文を書く。
- Product Hunt、GitHub Trending、Hacker News、公式 Changelog を分けて確認し、AI ツールの新規性を判定する。
- 収益化案ごとに、想定顧客、初回販売チャネル、想定単価を 1 行ずつ追加する。
