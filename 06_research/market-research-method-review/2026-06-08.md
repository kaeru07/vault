# 市場調査方法レビュー 2026-06-08

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいテーマを優先した。  
対象は App Store / Google Play、AIニュース、AIツール、MCP、AI Agent、Shorts / TikTok、AI自動化に重きを置いた。  
特に「汎用AI本体」ではなく、会議メモ、音声入力、業務データ接続、Shorts自動化など周辺の用途特化領域を見た。  
一方で、日本市場、公式ポリシー、価格・無料枠の確認は十分ではなかった。  

## 良かった点
- OpenAI、Anthropic、Google、Microsoft、GitHub など公式一次情報を多めに使い、AIニュースの根拠は比較的強かった。
- App Store / Google Play の収益上位と Product Hunt を組み合わせ、実需と新規ツールの両方を見られた。
- Codex / Claude Code / Gemini / Copilot を「開発支援」だけでなく、個人開発の量産手段として評価できた。
- 収益化ヒントが月額課金、従量課金、B2B小型SaaS、動画・note導線に分かれており、出口の形は整理できていた。
- 各ファイルに品質チェックがあり、未取得・推測・重複可能性を明示できていた。

## 弱かった点
- 日本の App Store / Google Play ランキングを取得しておらず、日本向け需要の判断が弱い。
- Shorts / TikTok / YouTube のAI生成コンテンツ収益化は Reddit 依存が残り、公式ポリシー確認が不足した。
- Product Hunt 上位ツールの価格、無料枠、継続利用、口コミまで追えておらず、収益化判断が浅い。
- 昨日分との比較が未取得で、同じテーマを繰り返している可能性を定量化できていない。
- AIツール調査は Codex / GitHub Copilot 系に寄り、非開発者向けAIツールや消費者向けアプリの掘り下げが薄かった。

## 情報源の偏り
今日カバーした情報源は、OpenAI公式、Anthropic公式、Google公式、Microsoft公式、GitHub公式、AppBrain、Product Hunt、TechCrunch、Axios、Reddit が中心だった。  
公式一次情報はAIニュースでは強かったが、市場調査では AppBrain と Product Hunt への依存が大きかった。  
不足していた情報源は、日本のApp Store / Google Play、YouTube公式ヘルプ、TikTok公式ポリシー、Apple / App Store側の公式発表、実ユーザーレビュー、料金ページ、導入事例。  
特に収益化判断に必要な「誰がいくら払っているか」を示す情報源が弱かった。  

## 収益化判断の精度
「会議メモ改善チェッカー」「AI音声メモからタスク化」「業務データQ&Aテンプレ集」は、短期実装・月額課金・Codex活用の条件に合っており、収益化に比較的近い。  
「Shorts収益化リスク診断」は需要仮説は良いが、公式ポリシー確認が不足しているため、現時点ではリスク診断そのものの正確性を担保しにくい。  
「Product Hunt監視ダッシュボード」は作りやすいが、無料情報の再整理に留まると課金理由が弱い。価格監視、導入事例、模倣難度スコアまで入れる必要がある。  
全体として、アプリ案は実装可能性に寄っており、支払い意思・集客導線・競合価格の検証が不足していた。  

## 見落とした可能性
- 日本市場の学習アプリ、麻雀、将棋、写真・動画カテゴリで、AI機能が課金に結びついている事例。
- App Store 側のAI発見導線、Apple Intelligence、WWDC関連のアプリ配布・課金変更。
- YouTube / TikTok / Instagram Reels の公式収益化ルールと、AI生成コンテンツの実際の審査基準。
- Product Hunt以外の個人開発者向け情報源、特に Indie Hackers、Hacker News、X、Reddit の具体的な課金報告。
