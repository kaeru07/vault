# 市場調査方法レビュー 2026-06-20

## 今日使った調査方針
収益化しやすく、短期リリース可能で、Claude Code / Codex で量産しやすいテーマを優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、個人開発市場、AI 自動化を中心に見た。  
特に「AI本体」ではなく、利用量管理、支出管理、モデル移行、MCP権限管理など周辺課題に重きを置いた。  
一方で App Store / Google Play、Shorts / TikTok、麻雀・将棋・学習アプリの確認は弱かった。  

## 良かった点
- OpenAI / Gemini / Anthropic について、一部は公式ニュースや changelog を確認できた。
- Gemini のモデル廃止、TTS ストリーミング、OpenAI の支出管理など、実装タスクに落としやすい情報を拾えた。
- MCP / Agent 認可を Arcade.dev、Zendesk、研究情報と接続し、収益化テーマとして整理できた。
- 「AIサブスク管理」「MCP設定リスクチェッカー」「モデル移行対応」など、個人開発で試せる案に落とせた。
- 各ファイルで未取得・推測・二次情報を明記し、過信を避ける構成になっていた。

## 弱かった点
- App Store / Google Play の当日ランキング直URLを取得できず、実際のモバイル課金需要との接続が弱かった。
- Microsoft Intelligent Terminal、Copilot Cowork、Cursor買収報道など、AIツール調査で公式一次情報が不足した。
- Gemini不満は報道ベースが中心で、Google公式ステータス、Playレビュー、Redditなど一次・準一次情報の分解が不十分だった。
- AIニュースとAIツールで MCP / Agent 認可テーマが重複し、別角度の市場探索が少なかった。
- 日本市場向けの学習アプリ、広告収益アプリ、Shorts/TikTok導線の具体化が浅かった。

## 情報源の偏り
今日カバーした情報源は、OpenAI News、Anthropic News、Gemini API changelog、Business Insider、WSJ、TechRadar、Android Central、Windows Central、ITPro、Investors.com、arXiv が中心だった。  
不足していた情報源は、App Storeランキング、Google Playランキング・レビュー、Product Hunt、GitHub Trending / OSSリポジトリ、Hacker News、Reddit、公式ステータスページ、Microsoft / GitHub 公式ドキュメント。  
全体として、英語圏の二次メディアと大手AI企業ニュースに寄り、日本の消費者向け需要や実際のアプリ課金データが弱かった。  

## 収益化判断の精度
「収益化ヒント」「作れそうなアプリ案」は、AIサブスク管理、Gemini利用制限ログ、MCP設定リスクチェッカー、AIアシスタント乗り換え診断など、短期試作には近かった。  
特に支出管理、モデル移行、認可・監査ログは、実際の運用痛点に近く、個人開発者や小規模チーム向けに売れる可能性がある。  
ただし、ユーザーが本当に支払うかは未検証で、検索需要、既存競合、価格帯、レビュー不満の量を確認できていない。  
今日の判断精度は「方向性は良いが、需要検証が不足」。収益化仮説としては中程度の確度に留まる。  

## 見落とした可能性
- App Store / Google Play の上昇中AIアプリから、実際に課金されているニッチを拾う視点。
- 日本語圏の学習、資格、語学、占い、家計、業務効率化アプリに AI 機能を足す収益化余地。
- Product Hunt / GitHub / Hacker News 上の MCP セキュリティ、Agent監査、AI支出管理ツールの既存競合。
- Shorts / TikTok 向けに、TTSストリーミングやAI動画編集をどう広告収益・教材販売へつなげるか。

## 次回改善すること
- App Store と Google Play のAI関連上位アプリを最低5件ずつ確認し、順位・価格・レビュー不満を市場調査ログに入れる。
- Gemini不満は、公式ステータス、Playレビュー、Redditまたはフォーラム、二次報道に分けて根拠を整理する。
- AIツール調査では、Microsoft / GitHub / Anysphere / Arcade.dev の公式ページを先に確認し、二次情報は補助扱いにする。
- MCP権限管理ツールの競合を Product Hunt、GitHub、Hacker News から最低3件探し、価格・対象ユーザー・差別化を表にする。
- 日本市場向け転用欄を追加で意識し、学習アプリ・ショート動画・広告収益アプリへの具体例を各1件以上書く。

軽微な反映案: AIツール調査では公式ドキュメント未確認の項目に「二次情報ベース」と明記する。
