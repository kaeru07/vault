# 市場調査方法レビュー 2026-07-16

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は Google Play / AIニュース / AIツール / MCP / AI Agent / Shorts / AI自動化に重きが寄った。  
麻雀・将棋・学習アプリは収益化案には出たが、実データ確認は薄かった。  
広告収益系と動画化余地は、Play Shorts・YouTube Shorts・アプリ紹介動画として扱った。  

## 良かった点
- Android 代替ストア、Play Shorts、AIコーディングエージェントを「配布・集客・開発効率」に接続できた。
- AIニュースとAIツール調査が、単なる新機能紹介ではなく収益化導線まで落ちていた。
- Lovable MCP、Base44、GitHub Security Review など、個人開発で試せる具体ツールを拾えた。
- 各ファイルで status: partial と不足根拠を明示でき、未確認情報を断定しなかった。
- 「作れそうなアプリ案」が低〜中難度中心で、短期検証に寄っていた。

## 弱かった点
- App Store / Google Play の当日ランキング一次データを取得できず、アプリ市場調査として弱い。
- OpenAI / Google の公式発表URL未取得が残り、AIニュースの一部が二次情報依存になった。
- 昨日ログ未取得のため、重複判定と差分分析が実質できていない。
- 麻雀・将棋・学習アプリの市場データが薄く、方針上の対象に対して観測不足だった。
- 収益化案は多いが、単価・顧客属性・販売チャネルの検証が不足していた。

## 情報源の偏り
今日カバーした情報源は、The Verge、Axios、Business Insider、TechRadar、Android Central、WSJ、Times of India、GitHub公式、Google公式、Lovable公式、Base44公式、MCP公式、AI SDK公式が中心だった。  
不足していた情報源は、App Store / Google Play の当日ランキング、Sensor Tower 等の市場データ、TikTok / YouTube Shorts の実例、Reddit / X / Hacker News などのユーザー反応、OpenAI公式発表、前日ログである。  
全体として、開発者向けAIツールと米国テックメディアに偏り、消費者向けアプリ市場と日本語圏需要の確認が弱かった。  

## 収益化判断の精度
収益化ヒントは、Android 代替ストア対応、アプリ紹介動画テンプレ、AIツール料金比較、レビュー返信・Data Safety 支援など、個人開発者向けに売りやすい形へ寄せられていた。  
ただし、実際に支払い意思がある顧客像、既存競合の価格、検索需要、販売導線の確認が不足しているため、収益化精度は中程度に留まる。  
「作れそうなアプリ案」は実装難度の見積もりは現実的だが、需要の裏取りが弱く、今すぐ作る優先順位までは確定できない。  

## 見落とした可能性
- App Store / Google Play の学習、麻雀、将棋、AIユーティリティ上位アプリの課金形態。
- Shorts / TikTok で実際に伸びているアプリ紹介動画の台本・画面構成・CTA。
- 日本の個人開発者、教育系クリエイター、小規模事業者が実際に困っているAI運用コスト問題。
- AIセキュリティレビュー系ツールを小規模SaaS向けに商品化する場合の競合と価格帯。

## 次回改善すること
- App Store / Google Play の当日ランキングを、AI、学習、動画、ユーティリティで各5件以上確認する。
- OpenAI / Anthropic / Google / GitHub は、二次記事より先に公式ブログ・changelog・価格ページを確認する。
- 前日Markdownを読み、重複テーマと新規差分を明示してから本文を書く。
- Shorts / TikTok / YouTube でアプリ紹介動画を最低5本分解し、訴求型を記録する。
- 麻雀・将棋・学習アプリを最低1カテゴリは毎日見る対象として実データに含める。
