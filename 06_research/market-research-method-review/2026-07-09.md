# 市場調査方法レビュー 2026-07-09

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、Shorts / TikTok、学習・麻雀・将棋周辺を重視した。  
巨大AI本体ではなく、既存ワークフローに差し込む補助ツールや課金導線を中心に見た。  
一方で App Store / Google Play の当日ランキング確認は不十分だった。

## 良かった点
- MCP paidTool、Codex、Claude Code、モデルルーティングなど、個人開発の実装・収益化に近い材料を拾えた。
- 「AI生成そのもの」より「投稿前チェック」「権利確認」「実行ログ」「テンプレ管理」に寄せた判断は現実的だった。
- 公式情報と二次情報、推測を品質チェック内である程度分離できていた。
- 各ファイルで収益化ヒントと作れそうなアプリ案まで接続できていた。

## 弱かった点
- App Store / Google Play の当日ランキングが未取得で、消費者需要の強弱判断が弱い。
- 昨日ログ未参照のため、重複テーマと新規性を具体的に判定できていない。
- Anthropic、OpenAI、Cloudflare、Product Hunt 寄りで、日本語圏の実需要や課金事例が薄い。
- Google Spark、GitHub Agent HQ、Microsoft Agent Framework など一部は公式一次情報が不足したまま評価している。
- 「作れそう」な案は多いが、誰がいくら払うかの検証がまだ粗い。

## 情報源の偏り
今日カバーした情報源は、Anthropic公式、OpenAI公式、Google Gemini API notes、Cloudflare公式、Product Hunt、The Verge、Wired、TechRadar、Business Insider、arXiv が中心だった。  
不足していた情報源は、App Store / Google Play の当日ランキング、TikTok 公式・実トレンド、YouTube Studio / 公式ヘルプ、Reddit・X・YouTubeコメント、日本語圏の個人開発・学習・ゲーム系ユーザー不満である。  
一次情報はAI基盤側に寄り、実際の購入者・利用者の声は少なかった。

## 収益化判断の精度
収益化ヒントは、Shorts投稿前チェッカー、Codexタスクテンプレ管理、MCP接続ヘルスチェック、x402 paidTool など、短期試作と課金導線に近いものが多かった。  
ただし、月額500円から2,000円を払う具体的なユーザー像、既存代替手段、集客経路、継続利用理由の検証は不足している。  
現時点の精度は「着想としては良いが、需要確認前」であり、明日はランキング・不満・価格を同時に見る必要がある。

## 見落とした可能性
- 日本語圏の学習、資格、将棋、麻雀アプリで実際に課金されている機能。
- YouTube / TikTok クリエイターが困っている権利確認、投稿管理、収益停止まわりの生の不満。
- App Store / Google Play のレビュー欄から見えるAIアプリの離脱理由。
- MCP paidTool の実装難度、決済対応地域、税務・返金・不正利用リスク。

## 次回改善すること
- App Store / Google Play の日本・米国ランキングで、AIチャット、動画編集、学習、将棋、麻雀を各5件ずつ確認する。
- 前日Markdownと照合し、重複テーマ、新規テーマ、継続観察テーマを明示する。
- Reddit、X、YouTubeコメント、アプリレビューのいずれか1系統からユーザー不満を最低3件拾う。
- 収益化ヒントごとに「想定ユーザー」「支払い理由」「月額/買い切り/従量」の仮説を1行で付ける。
- 二次情報で見つけたAIツールは、公式価格ページまたは公式ドキュメントを優先して確認する。
