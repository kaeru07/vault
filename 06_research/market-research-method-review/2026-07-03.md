# 市場調査方法レビュー 2026-07-03

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいかを主軸に見た。  
対象は AI ニュース、AI ツール、MCP、AI Agent、App Store / Google Play、Shorts / TikTok 寄りに偏った。  
麻雀・将棋・学習アプリは、短尺連続コンテンツや AI Agent 化の転用先として扱った。  
広告収益系と外部課金導線は見たが、実ランキングや実売上の確認は弱かった。

## 良かった点
- AI ニュースでは公式情報を多く使い、Claude / OpenAI / Google の agent workflow 化を収益化観点に接続できた。
- AI ツール調査では NLWeb、MCP、Codex Skills、Claude Code Agent SDK など、個人開発の生産性に直結する基盤を拾えた。
- 市場調査では Google Play の AI 検索導線、外部決済、短尺コンテンツ課金を同じ収益化文脈で整理できた。
- 「作れそうなアプリ案」が抽象論だけでなく、ASO自然文診断、麻雀ミニ連載、比較カードメーカーなど実装単位に近かった。
- 各ファイルで未取得情報を明示しており、結論の確度を過大に見せていない。

## 弱かった点
- App Store / Google Play の当日ランキング、カテゴリ順位、レビュー実例を直接確認できていない。
- TikTok / YouTube Shorts の実例やコメント不満が未取得で、動画化しやすさの判断が推測に寄った。
- AI ニュースに存在しない、または要確認のモデル名・発表名が混ざっている可能性があり、公式確認の厳密さが不足した。
- 価格、CVR、LTV、売上単価など、収益化判断に必要な数値がほぼ入っていない。
- 昨日との差分確認ができておらず、継続テーマと新規テーマの切り分けが弱かった。

## 情報源の偏り
今日カバーした情報源は、Anthropic / OpenAI / Google 公式、GitHub README、MCP 関連リポジトリ、Guardian、Business Insider、Android Central、Times of India、MarketWatch など。  
不足していた情報源は、App Store / Google Play の実ランキング、Sensor Tower 等の市場データ、SNSコメント、Reddit、開発者フォーラム、日本語圏の個人開発者事例、価格ページ、公式 changelog。  
全体として、AI 企業公式と開発者向け基盤情報に偏り、消費者側の不満・購買行動・競合アプリ実績が薄かった。

## 収益化判断の精度
「収益化ヒント」は方向性としては妥当だったが、実際の収益に近い順序づけはまだ弱い。外部課金、短尺動画流入、AI Agent の従量課金は収益化に近いが、具体的な価格、集客経路、初期CVR、運用コストが未検証である。  
「作れそうなアプリ案」は実装難度が低いものも含まれており有用だが、どれを先に作るべきかの根拠が不足した。次回は最低1案について、想定ユーザー、販売価格、最小機能、集客動画、検証指標まで落とす必要がある。

## 見落とした可能性
- AI アプリのレビュー欄に出ている課金不満、解約理由、広告過多への反応。
- 日本語圏の学習・麻雀・将棋アプリで、短尺動画と相性の良い既存ヒット例。
- 個人開発者がすぐ使える無料枠、startup credit、教育・研究向け助成金。
- MCP / Agent 系ツールのセキュリティ事故、権限設計、導入失敗例。

## 次回改善すること
- App Store / Google Play で AI、学習、動画、麻雀・将棋カテゴリの上位アプリを各3件確認し、順位と課金形態を記録する。
- TikTok / YouTube Shorts / Reddit から、AI生成コンテンツまたはAIアプリへの不満を最低2件、要約で取得する。
- AI ツールは公式 changelog または release ページを優先し、更新日・価格・無料枠・セルフホスト可否を表にする。
- アプリ案を1つだけ選び、初期機能、価格、流入動画案、検証KPIまで具体化する。
- 前日ファイルとの差分を確認し、重複テーマには「新規性あり / 継続観察のみ」を明記する。
