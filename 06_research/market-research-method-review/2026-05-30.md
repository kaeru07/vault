# 市場調査方法レビュー 2026-05-30

## 今日使った調査方針
収益化しやすく、短期リリース可能で、Codex / Claude Code で量産しやすい領域を優先した。  
特に AI サブスク監査、MCP 設定支援、AI ツール比較、学習コンテンツ変換を重く見た。  
App Store / Google Play / AI ニュース / AI ツール / MCP / AI Agent を横断したが、当日ランキングの一次取得は弱かった。  
動画化しやすいテーマとして、Shorts / TikTok 向け比較・紹介台本も観察対象に入れた。

## 良かった点
- 大手AIアプリと正面競争せず、比較・監査・導入支援・変換ツールに寄せた点は収益化方針に合っていた。
- Claude / Codex / Gemini / GitHub Copilot / MCP の変化を、個人開発の実装速度やコスト管理に接続できた。
- Google Play の Ask Play / Play Shorts を、集客導線やASO改善の観点で扱えた。
- 各レポートに品質チェック、重複可能性、未取得情報を明記できた。
- 収益化ヒントが「支出削減」「工数削減」「比較流入」など課金理由に近かった。

## 弱かった点
- App Store / Google Play の本日順位、レビュー、価格、課金導線を直接比較できていない。
- Reddit 由来の不満を多く使ったが、実売上・継続率・収益公開での裏取りが不足した。
- AIニュースとAIツール調査が Claude / Codex / Gemini / MCP に寄り、消費者向けアプリ市場の一次観察が薄くなった。
- 「作れそうなアプリ案」は出たが、競合価格・獲得チャネル・初期MVP範囲の具体性がまだ不足している。
- 日本語圏の個人開発、学習、AI副業、広告収益アプリの需要確認が弱かった。

## 情報源の偏り
今日カバーした情報源は、公式発表、企業ブログ、GitHub Changelog、Product Hunt、AppBrain、Sensor Tower、Forbes、Android Central、Reddit が中心だった。  
公式情報はAIモデル・エージェント・MCP周辺では強かったが、アプリ市場の実需要や売上実績は二次情報と推測に寄った。  
不足していた情報源は、App Store / Google Play の個別アプリレビュー、国内ストアランキング、TikTok / Shorts の実トレンド、開発者の収益公開、広告CPAやASO実験データ。  
特に日本市場、非エンジニア利用者、課金レビューの声が不足していた。

## 収益化判断の精度
今日の「収益化ヒント」は、AIサブスク監査、MCP設定チェッカー、AIツール比較メディア、学習コンテンツ再利用など、課金理由が比較的明確なものに寄っていた。  
支出削減や設定工数削減に直結する案は収益化に近いが、実際の支払い意思や既存競合の価格確認が不足している。  
「作れそうなアプリ案」は実装難度の見積もりは現実的だった一方、初期顧客、流入経路、月額単価、無料枠設計までは詰め切れていない。  
自己評価としては、着眼点は収益化寄りだが、検証データの厚みはまだ中程度。

## 見落とした可能性
- 日本語圏のAI副業・学習アプリ・資格学習アプリで、実際に課金されている小型カテゴリ。
- App Store / Google Play のレビュー低評価から見える、既存AIノート・AI学習・AIチャットアプリの具体的不満。
- AIツール比較メディアのSEO難度、アフィリエイト単価、検索ボリューム。
- MCP設定支援ツールの対象顧客が個人開発者なのか、B2B開発チームなのかの切り分け。

## 次回改善すること
- App Store / Google Play で「AI subscription」「AI cost」「AI notes」「AI quiz」を検索し、各3本以上の価格・レビュー不満・更新頻度を表にする。
- Reddit由来の仮説は、公式ドキュメント、開発者ブログ、収益公開、GitHub issue のいずれかで最低1つ裏取りする。
- 日本語圏の X / note / Qiita / Zenn / App Store 日本ランキングから、国内需要の有無を別枠で確認する。
- 収益化ヒントごとに、想定顧客、課金単価、初期MVP、最初の集客チャネルを1行ずつ付ける。
- AIニュースは大手モデル更新だけでなく、個人開発アプリの販売・流入・レビューに接続する情報を優先する。
