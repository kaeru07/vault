# 市場調査方法レビュー 2026-07-04

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいかを優先して評価した。  
対象は AI ニュース、AI ツール、MCP / Agent、App Store / Google Play、短尺動画、学習アプリ、広告収益系に重きを置いた。  
特に「AI生成物を作る」より「配布・発見・課金までつなげる」流れを中心に読んだ。  
一方で、App Store / Google Play の実ランキング確認と公式一次情報の確認は不足した。  

## 良かった点
- 市場調査、AIニュース、AIツール調査の3本で、Google Play 会話型検索、NotebookLM 短尺動画、Meta Pocket、Codex / Claude Code / MCP を同じ「生成後の配布・運用」文脈で整理できた。
- 収益化ヒントが、ストア説明文診断、Shorts台本生成、学習ノート再利用、麻雀・将棋の短尺解説など、短期試作に落とせる粒度になっていた。
- 各ファイルで「公式」「二次情報」「推測」「未取得」を分けており、確度の低い情報を断定しすぎなかった。
- AIツール調査では、モデル性能比較だけでなく、開発エージェントを日次運用・量産体制に組み込む観点まで見られた。

## 弱かった点
- Google Play、NotebookLM、Meta Pocket、ZCode、Claude Sonnet 5 など重要項目で二次情報依存が多く、公式ページ・開発者向け仕様・価格条件の確認が不足した。
- App Store / Google Play の実際の上位アプリ、価格、レビュー不満、スクリーンショット訴求を直接比較できていない。
- 「AI短尺動画」「AIエージェント」「ASO / AI検索最適化」に話題が寄り、麻雀・将棋・学習アプリなど既存方針の個別市場検証が浅かった。
- Codex Micro など話題性のあるニュースを拾ったが、個人開発の収益化に直結するかの評価はまだ弱い。
- 「作れそうなアプリ案」は出せているが、想定価格、販売先、初期顧客、検証方法まで踏み込めていない。

## 情報源の偏り
今日カバーした情報源は、The Verge、Business Insider、TechRadar、Times of India、Financial Times、Axios、arXiv、OpenAI公式、Claude Code公式、MCP公式が中心だった。  
不足していた情報源は、Google / Android Developers 公式、NotebookLM公式ヘルプ、Meta公式、App Store実ランキング、Google Play実ランキング、各アプリのレビュー欄、開発者ブログ、料金ページ、利用規約。  
特にストア系の話題を扱ったにもかかわらず、実ストア掲載ページとレビュー本文の確認が足りなかった。  
日本市場の一次観測として、国内App Store、Google Play日本ランキング、X / note / YouTube上の個人開発者反応も不足していた。  

## 収益化判断の精度
「収益化ヒント」と「作れそうなアプリ案」は、AI生成後の投稿、販売、説明文改善、学習再利用に寄っており、個人開発の短期試作には比較的近かった。  
特に「ストア説明文AI診断ツール」と「調査メモからShorts台本メーカー」は、実装難度が低く、既存ワークフローに組み込みやすい。  
一方で、実際に課金されるかの根拠は弱く、競合価格、検索需要、レビュー不満、広告単価、継続率の確認が不足している。  
収益化判断の精度は中程度。アイデア化までは良いが、販売可能性の検証は次回以降に持ち越し。  

## 見落とした可能性
- AI検索時代のASO支援について、既存ASOツールやGEO / AEO支援SaaSの価格、顧客層、差別化を比較できていない。
- AI短尺動画の収益性について、YouTube Shorts、TikTok、Instagram Reels の広告単価や外部送客成功例を確認していない。
- 麻雀・将棋・学習アプリで、実際にユーザーが課金している機能、レビュー不満、継続理由を掘れていない。
- MCP / Agent 系は技術的可能性に寄り、本番運用リスク、権限分離、個人開発での小さな有料機能化まで見られていない。

## 次回改善すること
- Google Play と App Store で、AI動画、AI学習、AI生産性カテゴリの上位10件を確認し、価格、レビュー不満、スクリーンショット訴求を表で比較する。
- Google / Android Developers、NotebookLM公式、Meta公式、Anthropic公式、ZCode公式を優先し、二次情報だけの項目は「未確認」と明示する。
- 収益化ヒントごとに、想定顧客、初期価格、販売先、1日で作れる検証版を1行ずつ付ける。
- 麻雀、将棋、学習のうち最低1ジャンルは毎日必ず実アプリまたは実ユーザー不満を確認する。
- AIエージェント系ニュースは、導入価値だけでなく「今日の個人開発フローを何分短縮するか」で評価する。
