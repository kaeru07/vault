# 市場調査方法レビュー 2026-06-23

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP / AI Agent、App Store / Google Play、Shorts / TikTok 導線に重きを置いた。  
特に「AI アプリの発見導線」「モデル分散」「MCP セキュリティ」「個人開発向け支援ツール」を中心に見た。  
一方で、麻雀・将棋・学習アプリ・広告収益系の個別市場確認は弱かった。

## 良かった点
- 収益化に近いテーマとして、AI アプリ紹介動画、ストア掲載文最適化、モデル比較ツールまで落とし込めた。
- AI ニュースと AI ツール調査で、MCP セキュリティと agent 認可を共通テーマとして拾えた。
- 「公式 / 二次情報 / 研究 / 推測」を品質チェック内である程度分離できた。
- ChatGPT 一強から Gemini / Claude / Meta AI へ分散する流れを、個人開発の訴求・課金設計に接続できた。
- 作れそうなアプリ案は実装難度まで付けられており、次の試作候補にしやすい。

## 弱かった点
- App Store / Google Play の当日実ランキングを取得できず、市場調査として一次データが不足した。
- Microsoft Build 2026、Gemini Spark、Arcade.dev など重要項目が二次情報中心で、公式確認が弱かった。
- 収益化ヒントは多いが、価格帯、競合課金、レビュー不満、獲得チャネルの比較が不足した。
- 麻雀・将棋・学習アプリなど、方針に含まれる日本向け高需要領域をほぼ見られていない。
- 昨日との差分が未確認のため、継続テーマと新規テーマの切り分けが曖昧だった。

## 情報源の偏り
今日カバーした情報源は、TechRadar、The Verge、Android Central、Business Insider、MarketWatch、Axios、The Guardian、Tom's Guide、WSJ、arXiv などの英語圏メディアと研究論文が中心だった。  
公式情報は IBM Bob など一部に限られ、OpenAI / Anthropic / Google / Microsoft / GitHub の公式 changelog や pricing は不足した。  
不足していた情報源は、App Store / Google Play 実ランキング、Product Hunt、X、Reddit、日本語圏レビュー、国内アプリランキング、個人開発者の実運用ログ。  
そのため、AI 業界動向の把握はできたが、実ユーザー需要と課金転換の根拠は弱い。

## 収益化判断の精度
「AI アプリ紹介動画台本」「ストア掲載文ローカライズ」「AI アシスタント比較」は短期リリースしやすく、収益化に比較的近い。  
一方、「AI エージェント監視」「MCP 認可」「Agent 基盤」は需要はありそうだが、個人開発で短期収益化するには顧客解像度と販売導線がまだ不足している。  
今日の判断は、アイデア抽出としては有用だが、実際の課金可能性を測るには競合価格、検索需要、レビュー不満、SNS 反応の確認が必要。  
自己評価としては、収益化への近さは中程度。市場テーマ選定は良いが、購買行動の根拠がまだ薄い。

## 見落とした可能性
- 日本の App Store / Google Play における AI 学習アプリ、麻雀・将棋 AI、実用ツールのランキング変化。
- AI アプリ利用者のレビュー不満、解約理由、価格への反応。
- Shorts / TikTok で実際に伸びている AI ツール紹介動画の構成、尺、CTA。
- Product Hunt や Reddit で反応が強い個人開発 AI ツールの価格・ポジショニング。

## 次回改善すること
- App Store / Google Play の日本・米国ランキングで AI 関連上位 10 件を確認し、価格・課金導線・レビュー不満を表にする。
- OpenAI / Anthropic / Google / Microsoft / GitHub の公式 changelog と pricing を先に確認してから二次報道を読む。
- Product Hunt、X、Reddit から個人開発 AI ツールの反応を最低 3 件拾い、実ユーザー需要として分離して書く。
- 麻雀・将棋・学習アプリのうち 1 ジャンルを毎日必ず 1 枠確認する。
- 昨日分 Markdown との差分を確認し、重複テーマと新規テーマを明示する。
