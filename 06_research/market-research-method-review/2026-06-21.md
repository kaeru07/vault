# 市場調査方法レビュー 2026-06-21

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいテーマを優先した。  
対象は App Store / Google Play、AI ニュース、AI ツール、MCP、AI Agent、Shorts / TikTok、学習アプリ、AI 自動化を中心に見た。  
特に「小型AIアプリ」「短尺動画による集客」「MCP / Agent の権限管理」を収益化候補として重く扱った。  
一方で、App Store / Google Play の当日ランキング直接確認は不足した。

## 良かった点
- 市場調査・AIニュース・AIツール調査の3本で、MCP / Agent 権限管理という共通した収益化テーマを抽出できた。
- AIチャット正面競争ではなく、学習、動画制作、権限ログ、コスト管理など狭い用途に落とせた。
- AIツール調査では公式URL中心に確認し、Claude Code changelog、OpenAI release notes、GitHub、Vercel docs など根拠が比較的強かった。
- Play Shorts / YouTube Shorts とアプリ発見導線を結びつけ、配信・動画化まで含めた収益化仮説を出せた。
- 各ファイルで取得状況を partial と明記し、未確認情報と推測を分けられた。

## 弱かった点
- App Store / Google Play の当日ライブランキングを直接取得できず、競合アプリ評価が定点観測として弱い。
- 市場調査で低評価レビューやユーザー不満の一次情報が少なく、収益化ヒントの一部が推測寄りになった。
- AIニュースは重要報道が二次情報中心で、OpenAI / Anthropic / Google の公式一次情報確認が不足した。
- AIツール調査が Vercel 系に偏り、非Vercel系の個人開発ツール、OSS、モバイル向けAIツールの比較が薄い。
- 昨日分との照合がなく、継続テーマと新規テーマの切り分けが不十分だった。

## 情報源の偏り
今日カバーした情報源は、Google Play / App Store 公式アプリURL、OpenAI release notes、Claude Code changelog、GitHub、Vercel docs、arXiv、The Verge、Business Insider、Axios、FT、Guardian、MarketWatch など。  
公式情報は AI ツール調査では比較的多かったが、市場調査とAIニュースでは二次情報の比率が高かった。  
不足していた情報源は、App Store / Google Play の当日ランキング、各アプリの低評価レビュー、TikTok / YouTube Shorts の実動画URL、Reddit / X / Product Hunt / Hacker News の利用者反応、国内ストア・国内SNSの観測。  
特に収益化判断に必要な「実際に払っているユーザー」「低評価レビュー」「売上順位」の情報が足りなかった。

## 収益化判断の精度
「収益化ヒント」は方向性としては近いが、実売上・検索需要・課金導線の裏取りが不足している。  
小型学習アプリ、Shorts台本チェッカー、AI動画制作コスト管理、MCP権限棚卸しは、個人開発で短期試作しやすい点は妥当。  
ただし、課金される強さで見ると、学習クイズや台本チェッカーは無料競合が多く、単体課金よりテンプレ販売・広告・業務向け機能との組み合わせが必要。  
最も収益化に近いのは、MCP / Agent の権限ログ、モデル切替、AI利用コスト管理のような「失敗・漏えい・赤字化を防ぐ」B2B寄りの小型ツール。

## 見落とした可能性
- App Store / Google Play の低評価レビューから見える、AIアプリ利用者の具体的な不満。
- 日本国内の学習、資格、将棋、麻雀アプリでAI機能が刺さるニッチ領域。
- Product Hunt / GitHub Trending / Hacker News で伸びているAI開発者ツール。
- AI動画制作の実収益だけでなく、テンプレ、教材、素材販売、運用代行への派生導線。

## 次回改善すること
- App Store / Google Play の当日ランキングを直接確認し、AI、学習、動画、仕事効率化カテゴリから各3件を記録する。
- 競合アプリごとに低評価レビューを最低2件拾い、不満をそのままアプリ案に変換する。
- OpenAI / Anthropic / Google / Vercel / GitHub の公式 changelog を先に確認し、その後に二次報道を見る。
- TikTok / YouTube Shorts でAIアプリ紹介動画を最低3件確認し、再生数、訴求、導線を記録する。
- Vercel系以外のAIツールを最低2件入れ、情報源の偏りを補正する。

## market-research-method.md への反映案
軽微な反映案: 次回から低評価レビューを各競合2件以上確認し、収益化ヒントの根拠に含める。  
大きな変更案: 日次調査に「ストアランキング直接取得」と「SNS動画実例確認」を必須ゲートとして追加する。  
大きな変更案: AIツール調査を「開発者向け」「消費者向け」「収益化支援向け」に分け、偏りを毎回可視化する。
