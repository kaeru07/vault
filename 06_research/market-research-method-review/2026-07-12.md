# 市場調査方法レビュー 2026-07-12

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいかを優先して見た。  
対象は AI ニュース、AI ツール、MCP、AI Agent、Shorts / TikTok、広告収益系に寄った。  
App Store / Google Play も対象に入っていたが、本日ライブ順位は取得できず、方針に対して未達だった。  
「常駐ワーク代行」「MCP」「AI動画制作補助」を個人開発の収益導線に変換できるかを中心に評価した。

## 良かった点
- OpenAI、Anthropic、Google、MCP、Gemini CLI など、AI エージェント化の流れを複数面から接続できた。
- ChatGPT Work / Claude Cowork を「大手トレンド」ではなく、軽量代替ツールの余地として読めた。
- 収益化ヒントが、広告、課金、note、動画、アフィリエイトに分解されていた。
- 作れそうなアプリ案が、議事録チェッカー、Shorts 台本スコアラー、MCP 棚卸しなど MVP 化しやすい粒度だった。
- 未取得情報を「未確認」「partial」と明記できており、根拠の強弱は分けられていた。

## 弱かった点
- App Store / Google Play の当日ライブ順位を取れず、需要の実測が弱かった。
- OpenAI GPT-5.6 / ChatGPT Work 周辺で公式一次情報を取れないまま二次情報への依存が残った。
- SNS 一次反応、YouTube コメント、Reddit、X など、ユーザー不満の現場確認が不足した。
- 麻雀、将棋、学習アプリなど、方針にある既存個人開発テーマとの接続が薄かった。
- アプリ案に想定価格、初期 MVP、集客チャネル、検証指標まで入っておらず、実行判断には一段足りない。

## 情報源の偏り
今日カバーした情報源は、OpenAI / Anthropic / Google 周辺の公式・準公式ページ、The Verge、Axios、TechRadar、AP、Business Insider、Times of India、arXiv、GitHub、MCP 公式が中心だった。  
不足していた情報源は、App Store / Google Play の当日ランキング、Sensor Tower / data.ai / Similarweb 系の市場データ、Reddit / X / YouTube コメントなどの一次反応。  
日本市場の情報源も弱く、日本語レビュー、国内アプリランキング、note / Qiita / Zenn / YouTube 日本語圏の実利用例をほぼ拾えていない。  
結果として、AI 業界ニュースには強いが、個人が明日売れる小型アプリの需要検証は弱かった。

## 収益化判断の精度
収益化ヒントは方向性としては近い。特に「大手 AI 統合への不満から軽量・用途特化ツールを作る」「常時稼働エージェントを月額課金理由にする」という判断は妥当だった。  
一方で、実際の収益化に必要な購入者像、価格、初回導線、競合価格、既存代替手段の比較が不足していた。  
「作れそうなアプリ案」は実装可能性は高いが、誰がなぜ払うかの検証が弱い。  
今日の精度は中程度。市場の方向は読めているが、売上に近い順番付けはまだ粗い。

## 見落とした可能性
- AI 学習アプリ、麻雀、将棋カテゴリで、既存ユーザーに追加課金されやすい具体機能。
- ChatGPT / Claude / Gemini のレビュー欄に出ている不満から作れる小型代替ツール。
- MCP Registry や GitHub 新着 MCP サーバーから見える、企業が実際に接続したいデータ種別。
- Shorts / TikTok 制作支援で、生成よりも分析、再編集、投稿後改善に課金余地がある領域。

## 次回改善すること
- App Store / Google Play の当日順位を、公式ストア、Similarweb、ランキング取得サイトのいずれかで最低 1 系統確認する。
- OpenAI、Anthropic、Google は最初に公式ブログ、価格ページ、リリースノートを確認し、未取得なら二次情報と明記する。
- Reddit、X、YouTube コメント、ストアレビューのうち最低 1 系統を見て、ユーザー不満を 3 件以上拾う。
- アプリ案ごとに「想定ユーザー」「月額価格」「最小 MVP」「最初の集客動画タイトル」を 1 行で付ける。
- 麻雀、将棋、学習アプリ、広告収益系の既存テーマへ最低 1 つ接続して、汎用 AI ニュースで終わらせない。
