# 市場調査方法レビュー 2026-05-29

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI 学習アプリ、AI 会議ノート、MCP、AI Agent、AI 自動化、広告収益系に重きを置いた。  
汎用 AI チャットそのものより、用途特化・成果物生成・投稿支援・業務手続き化に寄せて評価した。  
動画化しやすい TikTok / Shorts 流入と、月額課金に接続しやすい学習・会議・音声入力を重点的に見た。

## 良かった点
- 市場調査、AI ニュース、AI ツール調査の三者が「AI 学習」「MCP」「監督型エージェント」に接続しており、テーマの一貫性はあった。
- 公式情報と二次情報、推測、未取得情報を分けて書けており、根拠の強弱が比較的明確だった。
- Claude Opus 4.8、Codex 遠隔操作、Firecrawl、Base MCP など、個人開発の実装・運用に関係する材料を拾えた。
- 「業務ログを eval 化する」「MCP 接続を課金軸にする」など、単なるニュース要約ではなく商品設計に近い示唆が出せた。
- 取得状況を partial と明記し、未確認項目を次回再調査対象に残せた。

## 弱かった点
- App Store / Google Play の個別アプリ比較が浅く、価格、レビュー不満、オンボーディング、課金導線まで踏み込めていない。
- Reddit や二次情報由来の不満を使ったが、開発者ブログ、収益公開記事、広告運用実績での裏取りが不足した。
- AI 学習アプリと TikTok 流入の仮説が継続テーマ化しており、昨日以前との差分確認なしでは重複リスクが高い。
- AI ツール調査では「新規ツール」と「既存ツールの更新」が一部混在し、分類の精度が弱かった。
- 収益化ヒントは多いが、想定単価、原価、獲得チャネル、継続率の仮説がまだ粗い。

## 情報源の偏り
今日カバーした情報源は、公式発表、TechCrunch、Reuters 二次掲載、Vercel / GitHub / AWS / Anthropic / OpenAI 公式、AppBrain、AppDossier、AppHunter、AppMagic、Apptopia、Product Hunt、Reddit、Metricool が中心だった。  
不足していた情報源は、実際の App Store 検索結果、Google Play の個別レビュー、TikTok の実投稿・広告ライブラリ、開発者の収益公開、Sensor Tower 系の定量データ、国内個人開発者の事例。  
AI ニュースは公式寄りで良かった一方、市場調査は Reddit とランキング二次情報への依存が残った。  
日本市場の需要確認が弱く、海外トレンドをそのまま国内収益化へ接続している可能性がある。

## 収益化判断の精度
「AI 学習アプリ」「会議ノート」「Firecrawl を使った監視・調査基盤」「MCP 接続課金」は収益化に比較的近い。短期試作、月額課金、動画化、Codex / Claude Code 活用の条件が揃っているためである。  
一方、Gemini Spark、Base MCP、Alteryx One は話題性はあるが、個人開発で短期収益化するには提供条件、規制、価格、導入対象が未確認で距離がある。  
作れそうなアプリ案は実装難度の見積もりまでは出せたが、誰が何円で買うか、既存アプリから乗り換える理由、初回流入チャネルの検証が不足している。  
総合すると、収益化判断は方向性としては妥当だが、実際の売上に近い一次データは足りない。

## 見落とした可能性
- AI 学習アプリの国内需要。受験、資格、英語、大学講義など、日本語圏で課金されやすい具体カテゴリを見ていない。
- 会議ノートの法人導入条件。セキュリティ、録音同意、議事録共有、既存 SaaS 連携の不満を深掘りできていない。
- AI ツールの原価構造。Claude Opus 4.8 や Firecrawl 利用時の API 原価と月額料金の採算を計算していない。
- TikTok / Shorts 流入後の継続率。DL 獲得ではなく、翌日・7日後・課金転換の事例を探せていない。

## 次回改善すること
- App Store / Google Play で「AI study」「AI notes」「meeting notes」を検索し、上位 3 本ずつの価格、レビュー不満、無料枠、有料導線を表で比較する。
- Reddit / TikTok 由来の未確認情報は、最低 1 件は開発者ブログ、収益公開記事、公式事例で裏取りしてから収益化判断に使う。
- AI ツールは「新規サービス」「既存サービスの大型更新」「連携追加」に分類して、導入価値評価の混線を減らす。
- 収益化ヒントごとに、想定課金額、主な原価、初期獲得チャネルを 1 行ずつ付ける。
- 日本市場向けに、学習、資格、議事録、個人事業主バックオフィスのどれか 1 領域を毎日必ず確認する。
