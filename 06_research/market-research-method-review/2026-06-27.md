# 市場調査方法レビュー 2026-06-27

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI Agent、MCP、AI ツール、AI ニュース、Shorts / TikTok、AI 自動化、広告収益系に重きが置かれた。  
単体アプリより、GitHub、ブラウザ、動画編集、広告運用など既存ワークフローへ差し込む小型 AI を重視した。  
一方で App Store / Google Play、麻雀、将棋、学習アプリの実ランキング確認は弱かった。

## 良かった点
- GitHub Changelog、MCP 公式、Arcade Docs、arXiv など、AI Agent / MCP 周辺は公式・一次情報を比較的多く使えた。
- 収益化観点を「AI ツール紹介」だけで終わらせず、広告計測、AI 実行コスト、GitHub 連携 SaaS まで落とし込めた。
- 「既存作業面への埋め込み型 AI」という今日の軸が、市場調査・AI ニュース・AI ツール調査で一貫していた。
- 各ファイルで未確認情報、二次情報、推測を明示しており、根拠の強弱はある程度分離できていた。
- 次回見るべき対象が App Store、Google Play、公式価格、MCP Apps、Arcade pricing など具体的だった。

## 弱かった点
- App Store / Google Play の実ランキングを確認できず、消費者向けアプリ市場の収益化判断が推測寄りになった。
- AI ニュースで GPT-5.6、Gemini 3.5 Pro など公式未確認の話題が混ざり、実装判断に使うには危うい情報が残った。
- ユーザー不満の多くが二次情報・未確認扱いで、レビュー本文やフォーラム投稿など一次の不満データが不足した。
- GitHub / MCP / Agent 系に寄りすぎ、麻雀、将棋、学習アプリ、広告収益アプリの具体競合比較が薄かった。
- 作れそうなアプリ案は出ているが、想定価格、獲得チャネル、初回 MVP の検証手順まで踏み込めていない。

## 情報源の偏り
今日カバーした情報源は、Product Hunt、arXiv、GitHub Changelog、MCP 公式、Arcade Docs、TechRadar、Axios、Business Insider、The Verge、Android Central、WSJ が中心だった。  
強かったのは AI Agent、MCP、GitHub Copilot、AI 広告、AI コーディング周辺。  
不足していたのは App Store / Google Play のランキング、アプリレビュー本文、YouTube / TikTok 公式ポリシー、実際の Shorts 収益事例、国内個人開発者の販売導線。  
結果として、開発者向け B2B / B2D の解像度は高いが、一般ユーザー向けアプリ市場の検証が弱い。

## 収益化判断の精度
今日の「収益化ヒント」は、音声入力、コードレビュー、ブラウザ自動化、広告クリエイティブ生成など、継続利用や業務 ROI に近いものが多く、方向性は比較的良い。  
ただし、AI 動画・Shorts 系は収益化ポリシー、実収益、アカウント制限、品質基準を未確認のまま扱っており、収益化に近いと言い切るには根拠が弱い。  
作れそうなアプリ案は短期試作しやすいが、競合優位性はまだ「便利そう」止まりで、誰が月額を払うかの検証が不足している。  
総合評価は中程度。B2B 小型 SaaS と開発者向け補助ツールは精度が高め、消費者向け AI 動画・学習アプリは精度が低め。

## 見落とした可能性
- App Store / Google Play の売上上位 AI アプリから、実際に課金されているカテゴリを逆引きする視点。
- 日本語圏の X、note、Qiita、Zenn、YouTube での Claude Code / Codex 実運用不満や教材需要。
- AI ツールの価格改定・無料枠縮小によって生まれる「費用監視」「代替モデル選択」系の小型 SaaS 需要。
- 麻雀、将棋、学習アプリなど、既存の方針対象に AI Agent / 動画化を掛け合わせる具体案。

## 次回改善すること
- App Store / Google Play の当日ランキングを確認し、AI 音声入力、AI 動画、AI 学習アプリを最低 3 件ずつ記録する。
- OpenAI、Anthropic、Google、GitHub、YouTube、TikTok は公式ブログ・価格ページ・ポリシーページを二次情報より先に確認する。
- ユーザー不満はレビュー本文、GitHub Issue、Product Hunt コメント、Reddit などから最低 3 件の一次発言を拾う。
- 作れそうなアプリ案には、想定ユーザー、初期価格、MVP 機能 1 つ、最初の集客チャネルを併記する。
- 前日ファイルと照合し、同じテーマは「新情報あり」「継続観察のみ」「今回は除外」に分ける。

## market-research-method.md への反映案
軽微な反映案: 毎日見る対象の確認時に、App Store / Google Play は実ランキングまたはレビュー一次情報を優先する。  
大きな変更案: 調査対象を「開発者向け」「一般消費者向け」「動画・広告収益向け」の 3 枠に分け、日替わりで重点配分する。  
大きな変更案: 作れそうなアプリ案に、想定価格・初回 MVP・獲得チャネルを必須項目として追加する。
