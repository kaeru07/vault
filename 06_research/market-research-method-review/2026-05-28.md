# 市場調査方法レビュー 2026-05-28

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Codex / Claude Code で量産しやすい領域を優先した。  
対象は App Store / Google Play、AIニュース、AIツール、MCP、AI Agent、AI自動化、Shorts / TikTok を中心に見た。  
汎用AIチャットより、議事録、SMB自動化、MCPミニSaaS、動画制作補助など用途特化型を重く扱った。  
一方で、日本市場、公式ストアランキング、レビュー本文の一次確認は弱かった。

## 良かった点
- 市場調査、AIニュース、AIツールの3本で「MCP / Agent / 業務自動化」という共通変化を拾えていた。
- 収益化ヒントが、議事録、SMB自動化、動画台本生成、MCPミニツールのように具体的な課金導線へ寄っていた。
- 公式情報と二次情報、推測、未取得情報を品質チェックで分けて書けていた。
- YouTube AIラベル、Robinhood agent連携、AWS MCP Server など、個人開発への影響が大きい変化を拾えていた。
- 「取得状況: partial」として未確認範囲を明示できていた。

## 弱かった点
- App Store / Google Play の本日ランキングを公式または安定したデータ源で再取得できていない。
- 日本市場のランキング、レビュー、不満、価格感が不足しており、海外AIニュース寄りになった。
- ユーザー不満の一部が Reddit や推測ベースで、実際のレビュー本文・Product Huntコメント・X投稿の裏取りが弱い。
- AIツール調査が大手ベンダーのAgent基盤に寄り、今日すぐ試せる個人開発向け小型ツールが少なかった。
- 昨日分との差分比較ができず、新規性と重複率を定量化できなかった。

## 情報源の偏り
今日カバーした情報源は、OpenAI、Anthropic、ClickHouse、AWS、Google、GitHub、Circle、Kore.ai などの公式発表と、TechCrunch、Android Central、Infoworld、Investing.com、TechRadar、a16z、Apptopia、AppBrain、arXiv だった。  
不足していた情報源は、App Store / Google Play の当日公式ランキング、日本のストアランキング、アプリレビュー本文、Product Hunt、GitHub trending、MCP Registry / Smithery / Glama、X、個人開発者の収益報告。  
結果として、企業発表と海外二次メディアには強いが、実ユーザーの不満、国内需要、個人開発者が今日参入できる隙間の検証が弱かった。

## 収益化判断の精度
収益化に近い判断は、議事録・音声メモ、SMB向け自動化、MCP対応ミニSaaS、Shorts用台本生成に集中しており、方向性は妥当だった。  
ただし、実際の課金上位、検索流入、レビュー不満、価格帯の確認が不足しているため、「作れそう」から「売れそう」への根拠はまだ弱い。  
Circle Agent Stack や Robinhood MCP のようなAgent決済系は将来性があるが、日本在住個人の規制・会計・実装負荷を考えると短期収益化判断としては慎重に扱うべきだった。  
今日の精度は「仮説作成としては有用、投資判断には追加検証が必要」。

## 見落とした可能性
- 日本語レビューに出る、議事録・学習・動画編集アプリの具体的な不満。
- App Store / Google Play の教育、写真動画、ビジネスカテゴリにある小型AIアプリの課金上位。
- Product Hunt や GitHub trending に出る、個人開発者がすぐ真似できる小型AIツール。
- AI動画ラベル対応後の、収益化停止・推薦低下・広告主安全性に関する実例。

## 次回改善すること
- App Store / Google Play の米国・日本ランキングを、生産性、教育、写真動画、ビジネスの4カテゴリで確認する。
- 各カテゴリ上位AIアプリについて、レビュー本文から不満を最低5件ずつ抜き出す。
- Product Hunt、GitHub trending、MCP Registry / Smithery / Glama を横断し、個人開発者が今日試せる小型ツールを優先して拾う。
- 前日Markdownを参照し、重複トピックと本日新規トピックを明示的に分ける。
- 収益化ヒントには、想定課金方式、初期MVP、最初の集客チャネルを必ず1行で付ける。
