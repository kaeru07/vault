# 市場調査方法レビュー 2026-06-15

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい周辺ツールを優先して見た。  
対象は AI ニュース、AI ツール、MCP / AI Agent、動画編集、AI 自動化、広告収益系に重きがあった。  
一方で App Store / Google Play / SNS 実投稿は方針上の対象だが、実取得できず弱かった。  
巨大AI本体ではなく、比較・節約・ルーティング・雛形生成・動画制作補助のような個人開発向け切り出しを重視した。

## 良かった点
- 各ファイルで「未取得」「未確認」「二次情報ベース」を明示し、断定を避けられていた。
- AIサブスク高騰、モデルルーティング、MCP Apps、AI動画制作補助など、収益化に近い周辺領域へ寄せられていた。
- 市場調査、AIニュース、AIツールの間で「個人開発では本体競争ではなく周辺ツール化」という結論が一貫していた。
- 作れそうなアプリ案が、低〜中難度の実装単位に落ちていた。
- 品質チェックで、公式一次情報不足と実ランキング未取得を明示できていた。

## 弱かった点
- App Store / Google Play の実ランキングとレビュー不満を取得できず、アプリ需要の強さを検証できていない。
- OpenAI / Anthropic / Google / Microsoft の公式一次URLが不足し、重要ニュースの一部が二次情報依存になった。
- AIニュースに「Claude Fable 5」など公式未確認の固有名が含まれ、誤情報混入リスクが高い。
- 収益化案は多いが、価格、競合数、流入経路、想定ユーザーの支払い理由まで詰め切れていない。
- Microsoft Build 2026 系、Codex / Copilot / MCP 系の話題が重なり、同一イベント由来の情報を別件として膨らませた可能性がある。

## 情報源の偏り
今日カバーした情報源は、TechRadar、Tom’s Hardware、The Verge、Android Central、Windows Central、Tom’s Guide、Vox、Business Insider / NY Post 系報道、arXiv が中心だった。  
不足していた情報源は、OpenAI / Anthropic / Google / Microsoft / GitHub の公式ブログ・価格ページ・開発者ドキュメント、App Store / Google Play の実ランキングとレビュー、SNS検索結果、Reddit / X / YouTube コメントなどの利用者不満ソース。  
特に収益化判断に必要な「実ユーザーの不満」と「課金導線」は、二次ニュースよりストアレビューと公式料金表を見るべきだった。

## 収益化判断の精度
収益化ヒントは方向性としては近い。AIサブスク節約、モデル切替、AIツール比較、動画制作補助、MCP雛形生成は、個人開発で小さく出せる余地がある。  
ただし精度は中程度に留まる。理由は、需要の強さを示すランキング、レビュー件数、検索量、競合価格、実際の不満文が不足しているため。  
「作れそうなアプリ案」は実装可能性に寄っており、売れる確度の検証が弱い。次回は各案に対して、想定課金、競合、流入チャネル、検証に使う一次情報を最低1つずつ付ける必要がある。

## 見落とした可能性
- ストアレビュー上の具体的な不満: 高額課金、無料枠縮小、広告過多、解約しにくさ、生成品質への不満。
- 日本語圏の需要: note、Qiita、Zenn、YouTube、X での AI ツール比較・動画編集・業務自動化ニーズ。
- 既存ツールの価格改定履歴: Copilot、ChatGPT、Claude、CapCut、Pippit などの料金変化とユーザー反応。
- 広告収益系アプリ: AIニュース・AIツール比較だけでなく、麻雀、将棋、学習アプリなど方針対象の非AI寄り収益ジャンル。

## 次回改善すること
- App Store / Google Play で「AI」「動画編集」「学習」「将棋」「麻雀」の上位アプリを各2件以上見て、レビュー不満を引用ではなく要約で記録する。
- OpenAI / Anthropic / Google / Microsoft / GitHub の公式ページを先に確認し、二次情報の固有名・価格・提供条件は公式確認済みと未確認に分ける。
- 収益化アプリ案ごとに「想定ユーザー」「支払い理由」「競合」「最初のMVP機能」を1行で付ける。
- MCP / Agent 系は、ニュース、仕様更新、実装例、収益化案を分離して記録する。
- 昨日分のファイルを参照し、重複テーマと本日新規テーマを明示する。
