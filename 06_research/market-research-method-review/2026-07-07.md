# 市場調査方法レビュー 2026-07-07

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすいテーマを優先した。  
対象は AI アプリ市場、AI ニュース、AI ツール、MCP / Agent、短尺動画、ストア発見導線に寄せた。  
特に「大手AI本体と競わず、周辺ツール・運用支援・特化ミニアプリで収益化する」観点を強く使った。  
一方で、麻雀・将棋・学習アプリ・広告収益系の具体調査は薄かった。

## 良かった点
- Google Play の Ask Play / Play Shorts を、アプリ発見・短尺動画・ストア最適化の収益化テーマに接続できた。
- Claude Code / Codex / Copilot / Gemini CLI を、単なる新機能紹介ではなく、個人開発の検証・QA・保守自動化に落とし込めた。
- 公式情報と二次情報を分け、未取得・推測・partial の扱いを明記できた。
- 「エージェント安全管理」「ストア文言AI診断」「AI動画ネタ帳」など、短期試作しやすいアプリ案に整理できた。
- 収益化ヒントが月額SaaS、買い切り、テンプレート販売、動画コンテンツ化に分散していた。

## 弱かった点
- App Store / Google Play の当日ライブランキングを直接確認できず、市場調査の根拠が二次情報に寄った。
- TikTok / YouTube Shorts の実アカウント、再生数、販売導線を確認しておらず、AI動画収益化の実証が弱い。
- AIニュースは 2026-07-07 初出と直近継続ネタの分離が不十分だった。
- AIツール調査が GitHub Copilot 周辺に偏り、動画生成・広告生成・学習系ツールの比較が不足した。
- 麻雀・将棋・学習アプリなど、方針にある個別ジャンルの収益化観察がほぼ入らなかった。

## 情報源の偏り
今日カバーした情報源は、OpenAI / Anthropic / GitHub / Gemini CLI / MCP などの公式情報と、Business Insider、Android Central、WSJ、TechRadar、Axios などの二次情報が中心だった。  
不足していた情報源は、App Store / Google Play の当日ランキング、実レビュー、TikTok / YouTube Shorts の一次投稿、個人開発者の収益報告、広告出稿事例、国内ユーザー向けの日本語圏情報だった。  
特に「実際に誰が払っているか」を見る情報源が不足し、技術トレンドから収益化を推測する比率が高かった。

## 収益化判断の精度
収益化ヒントは方向性としては近い。ストア文言最適化、AI動画制作支援、エージェント安全管理、レビュー不満抽出は、小規模SaaSやテンプレート販売に接続しやすい。  
ただし、実ランキング、実レビュー、価格、競合の有料プラン、販売実績を確認できていないため、課金確度は中程度に留まる。  
「作れそうなアプリ案」は実装難度の見積もりは現実的だが、顧客獲得チャネルと初回課金ポイントの検証が不足している。

## 見落とした可能性
- 学習アプリ領域で、AIチューター、試験対策、暗記、親子向け学習のストア上位・課金導線を見ていない。
- 麻雀・将棋のAI解析、棋譜レビュー、初心者向け動画教材など、既存方針にある趣味特化市場を確認していない。
- AI動画ツールの商用利用規約、透かし、生成上限、テンプレート販売可否を比較していない。
- エージェント安全管理ツールの既存競合、価格、OSS代替、企業向け導入障壁を確認していない。

## 次回改善すること
- App Store / Google Play の当日AI関連ランキングを直接確認し、順位・価格・レビュー数・課金導線を表にする。
- TikTok / YouTube Shorts のAI動画アカウントを最低5件見て、再生数、投稿頻度、販売先、テンプレート導線を記録する。
- AIニュースは「今日初出」「直近継続」「未確認報道」を分けて、今日初出だけを冒頭に置く。
- AIツール調査に動画生成・広告生成・学習支援ツールを最低1枠入れ、Copilot系への偏りを減らす。
- 収益化案ごとに、想定顧客、初回価格、既存競合、1週間で作るMVP範囲を1行で付ける。
