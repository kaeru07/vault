# 市場調査方法レビュー 2026-06-12

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP / Agent、短尺動画、AI 自動化、広告収益系を中心に見た。  
App Store / Google Play も観察対象に入れたが、当日実チャートの一次確認は不足した。  
動画化しやすいテーマとして、AI短尺動画、AI広告生成、開発エージェント系を重めに扱った。

## 良かった点
- 収益化に近い「商品URL→広告動画」「長尺→Shorts」「LLM発見性診断」へ絞れた。
- AI ニュースを API 価格、Codex 強化、Gemini 障害など個人開発の原価・可用性へ接続できた。
- AI ツール調査で GitHub Spark、Agent Framework、MCP Apps など開発量産に関係する候補を拾えた。
- 各レポートに「公式未確認」「二次情報中心」「推測」を明記し、根拠の弱さを残せた。
- 品質チェック欄で重複可能性と次回再調査対象を明示できた。

## 弱かった点
- App Store / Google Play の実順位を確認できず、競合アプリ分析が二次情報寄りになった。
- SNS 上の実ユーザー不満を十分に取れず、収益化仮説が推測に寄った。
- OpenAI / Anthropic / Google / GitHub / Microsoft の公式一次 URL が不足した。
- 価格、無料枠、商用利用条件の比較表がなく、導入価値評価の精度が弱い。
- 昨日分との照合ができず、継続テーマと新規テーマの切り分けが曖昧だった。

## 情報源の偏り
今日カバーした情報源は、Business Insider、The Verge、TechRadar、Axios、AP、Android Central、Creative Bloq、Wikipedia、arXiv などの二次情報・研究情報が中心だった。  
不足していた情報源は、App Store / Google Play の当日チャート、各社公式ブログ・価格表・ステータスページ、Product Hunt 当日カテゴリ、GitHub Changelog、Microsoft Learn / Build 公式、X / Reddit / TikTok の一次ユーザー投稿。  
特に Microsoft / GitHub 系ツールは二次媒体に偏り、公式仕様・価格・提供範囲の確認が弱かった。

## 収益化判断の精度
収益化ヒントは、短尺動画広告、AI広告生成、LLM発見性、マイクロドラマ課金に寄せており、方向性としては比較的近い。  
ただし、実需要の裏付けに必要な検索ボリューム、SNS不満、アプリ順位、競合価格、無料枠制限が不足している。  
「商品URL→縦型広告台本ジェネレーター」と「マイクロドラマ課金シミュレーター」は低難度で検証しやすいが、課金されるかは未検証。  
「Product Hunt露出診断」は個人開発者向けに刺さる可能性がある一方、支払い意思の確認が必要。

## 見落とした可能性
- Shopify / Amazon / TikTok Shop など EC 実務者が実際に困っている広告制作ワークフロー。
- CapCut、Instagram Edits、TikTok Studio の価格・透かし・商用利用条件の比較。
- AI ツールのサブスク疲れに対する「複数ツール統合」「請求可視化」「利用量最適化」系ニーズ。
- 日本市場の一次情報、特に X、note、App Store 日本ランキング、国内個人開発者の課金事例。

## 次回改善すること
- App Store / Google Play の当日ランキングを最低 1 回確認し、AIアプリと動画編集アプリの順位を記録する。
- OpenAI、Anthropic、Google、GitHub、Microsoft は公式ブログ・価格表・ステータスページを先に確認する。
- X / Reddit / TikTok / Product Hunt のいずれかから、実ユーザー不満を最低 3 件拾う。
- AI動画広告ツールを 3 件に絞り、価格、無料枠、透かし、商用利用、出力形式を表で比較する。
- 昨日分 Markdown と照合し、新規・継続・重複の区分を明記する。
