# 市場調査方法レビュー 2026-06-25

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、App Store / Google Play、Shorts / TikTok、AI 自動化を中心に見た。  
特に「AI生成物がアプリ・PR・動画になる」流れと、個人開発者が周辺ツール化できる領域に重きを置いた。  
一方で、App Store / Google Play の当日ランキングやユーザーレビュー取得は不足した。  

## 良かった点
- 市場調査、AIニュース、AIツールの3本で、AIエージェント、AIコーディング、AI動画、MCPを横断して見られた。
- OpenAI changelog、Claude Code changelog など、一部は公式一次情報を確認できた。
- 収益化ヒントが「テンプレ販売」「運用ツール」「品質チェック」「週次レポート」など個人開発向けに落ちていた。
- 各ファイルに品質チェックと取得状況があり、未確認・推測・二次情報の区別を残せた。
- Codex / Claude Code 活用余地と agent 権限管理のリスクを収益化判断に接続できた。

## 弱かった点
- App Store / Google Play のライブランキング、レビュー本文、課金導線を直接確認できていない。
- AIツール調査で Copilot desktop app、Antigravity 2.0、Project Solara など二次情報依存の項目が多かった。
- SNS一次投稿URLがなく、Shorts / TikTok の伸びている形式を具体的に検証できていない。
- 「作れそうなアプリ案」は多いが、競合価格、無料枠、課金開始ポイントの比較が弱い。
- 昨日分との照合ができず、重複テーマの削減や新規性評価が甘くなった。

## 情報源の偏り
今日カバーした情報源は、Business Insider、Axios、Financial Times、The Verge、TechRadar などのニュース媒体、OpenAI / Claude Code / GitHub / CapCut / MCP / OpenClaw 公式系、arXiv 論文が中心だった。  
不足していた情報源は、App Store / Google Play の当日ランキング、アプリレビュー本文、TikTok / YouTube Shorts / X の一次投稿、Reddit / Hacker News / GitHub Issues など実利用者の不満が見える場。  
公式一次情報は OpenAI と Claude Code では取れていたが、Google Antigravity、Copilot desktop app、Gemini 3.5 Pro 延期、Project Solara は二次情報に寄った。  
収益化判断に必要な価格ページ、課金導線、無料枠、解約・不満レビューの確認が不足した。  

## 収益化判断の精度
「収益化ヒント」は、AIミニアプリ生成テンプレ、Codex / Claude Code 開発キット、ショート動画台本管理、AI生成物レビューなど、個人開発で試しやすい方向に寄っており概ね妥当。  
ただし、需要の強さをランキング、レビュー数、課金プラン、SNS投稿の伸びで裏取りできていないため、現時点では仮説の精度は中程度。  
「作れそうなアプリ案」は実装難度の見積もりは現実的だが、誰がいくら払うか、既存代替より何が強いかの検証が薄い。  
短期で収益化に近いのは、汎用AIアプリより「MCP対応の小型業務ツール」「AI動画運用補助」「Codex作業ログ・費用対効果可視化」に絞るべき。  

## 見落とした可能性
- AIアプリのレビュー低評価に出る具体的な不満、返金理由、課金直前の離脱ポイント。
- 日本市場の App Store / Google Play 上位アプリ、特に学習、麻雀、将棋、広告収益系の当日変化。
- TikTok / YouTube Shorts で実際に伸びている AIアプリ制作動画、AIドラマ、Codex作業動画のフォーマット。
- MCP / agent 系ツールの実装者向け課金機会だけでなく、非エンジニア向け運用代行・設定代行の需要。

## 次回改善すること
- App Store と Google Play の日本・米国ランキングを最低各1回確認し、AIアプリ、動画編集、学習系の順位と課金導線を記録する。
- 各調査で最低3件、ユーザーレビューまたはSNS一次投稿を取得し、不満を原文ベースで分類する。
- 競合ツールごとに価格、無料枠、課金開始ポイント、セルフホスト可否を固定表で埋める。
- 二次情報だけのニュースは見出しに「二次情報」と明記し、公式未確認なら収益化判断を一段下げる。
- 昨日分 Markdown と照合し、重複テーマは「継続監視」に回し、新規性がある項目だけ主本文に残す。
