# 市場調査方法レビュー 2026-06-26

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP / AI Agent、AI 自動化、Shorts / TikTok 系に重きを置いた。  
市場調査では AI 動画、AI チャット広告、AI サブスク管理、MCP 安全確認を中心に見た。  
一方で App Store / Google Play の当日ライブランキング確認は不足した。

## 良かった点
- AI ニュースでは Codex / Claude Code の委任型エージェント化を収益化観点に接続できた。
- AI ツール調査では GitHub MCP Server、Cursor、Claude Code など公式情報中心に確認できた。
- 市場調査では AI 動画、広告、テンプレ販売、アフィリエイトまで複数の収益導線を整理できた。
- MCP / Agent 系を「開発効率」だけでなく、監査、権限、コスト管理の商材候補として扱えた。
- 各ファイルで取得状況と未取得情報を明記でき、根拠の弱さを隠さなかった。

## 弱かった点
- App Store / Google Play の当日ランキングを直接確認できず、アプリ市場の実需検証が弱い。
- AI ニュースは二次情報依存が多く、OpenAI / Anthropic / Google の一次情報確認が不足した。
- 市場調査が AI 動画、Codex / Claude、MCP に寄り、麻雀・将棋・学習アプリの確認が薄かった。
- 収益化ヒントは妥当だが、価格帯、想定ユーザー数、広告単価などの数字が不足した。
- 前日ログとの比較ができず、重複テーマを定量的に減らせなかった。

## 情報源の偏り
今日カバーした情報源は、The Verge、Axios、Business Insider、TechRadar、Vanity Fair、arXiv、OpenAI News、GitHub Changelog、Cursor Changelog、Claude Code changelog、GitHub MCP Server releases が中心だった。  
不足していた情報源は、App Store / Google Play のランキング、公式価格ページの最新確認、日本語圏 TikTok / YouTube Shorts、国内個人開発者の販売事例、note / Gumroad / BOOTH などの販売面データ。  
特に AI ツール調査は公式情報中心で良かったが、市場調査と AI ニュースは二次メディアに偏った。  
日本語圏の需要確認が弱く、海外 AI 開発者向け情報に寄った。

## 収益化判断の精度
「AI 動画制作支援」「AI サブスク管理」「MCP 安全チェック」「AI エージェント監査ログ」は、短期実装可能性と Codex / Claude Code 活用余地があり、収益化には比較的近い。  
ただし、実際に買われるかの検証は不足しており、競合価格、検索需要、既存アプリ売上、SNS 投稿の伸びを確認できていない。  
「縦型動画台本ジェネレーター」や「投稿パッケージ生成」は作りやすいが差別化が弱く、単体 SaaS よりテンプレ販売や既存ワークフロー連携の方が現実的。  
今日の判断精度は中程度。方向性は良いが、需要の定量確認が不足している。

## 見落とした可能性
- AI 動画以外の広告収益系アプリ、特に壁紙、診断、学習、習慣化アプリの売上上位確認。
- 麻雀・将棋など日本語圏で強い趣味領域に AI 解説・学習機能を載せる小型アプリ案。
- 企業向けではなく個人クリエイター向けの「月額 500-1,500 円」で売れる軽量ツール。
- AI エージェント悪用報道を受けた、個人開発者向けセキュリティ教育コンテンツ販売。

## 次回改善すること
- App Store / Google Play で AI チャット、AI 動画、学習、広告収益系の当日上位を各 3 件ずつ確認する。
- OpenAI / Anthropic / Google / GitHub / Cursor は、二次記事より先に公式ブログ、changelog、pricing を確認する。
- 日本語圏 TikTok / YouTube Shorts で「AI 副業」「AI 動画」「AI 学習」の伸びている投稿形式を最低 5 件見る。
- 作れそうなアプリ案には、想定課金、初期 MVP、集客チャネルを 1 行ずつ付ける。
- 前日ファイルとの差分を確認し、同じテーマが続く場合は「継続理由」か「調査打ち切り理由」を書く。
