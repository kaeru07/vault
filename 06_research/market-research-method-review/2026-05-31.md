# 市場調査方法レビュー 2026-05-31

## 今日使った調査方針
収益化しやすい、短期リリース可能、Claude Code / Codex で量産しやすい対象を優先した。  
特に App Store / Google Play、AIニュース、AIツール、MCP、AI Agent、Shorts / TikTok 導線を重視した。  
市場調査ではアプリ発見導線、課金分析、AIツール比較に寄せた。  
AIニュースとAIツール調査では、coding agent、MCP/API課金、開発自動化の収益化余地を中心に見た。

## 良かった点
- Google Play の Ask Play / Play Shorts と App Store Connect Analytics を、個人開発の流入・課金改善に接続できていた。
- 公式情報源を比較的多く使い、Google、Apple、OpenAI、Anthropic、AWS、GitHub、Salesforce など一次情報の比率が高かった。
- AIニュースとAIツール調査が MCP / agent / coding automation でつながっており、日次テーマの一貫性があった。
- 「作れそうなアプリ案」が Play Shorts 台本生成、課金KPIメモ帳、AIツール比較ページ生成など短期試作に寄っていた。
- 品質チェックで partial 理由、未取得情報、重複可能性を明記できていた。

## 弱かった点
- App Store / Google Play の実ランキング、レビュー低評価、価格、課金CTAの定量確認が不足していた。
- Reddit やSNS由来の不満は代表性が弱く、複数投稿・複数レビューでの裏取りが足りなかった。
- Product Hunt はカテゴリ件数や上位傾向を見たが、個別ツールの価格、登録導線、課金ページまでは十分に掘れていない。
- 昨日分との比較が未取得で、重複判定がほぼ推測になっていた。
- AIニュース側は大手モデル・大手SaaSの話題が多く、個人開発者が今週作るべき最小プロダクトへの落とし込みがやや広かった。

## 情報源の偏り
今日カバーした情報源は、Google / Apple / OpenAI / Anthropic / AWS / GitHub / Salesforce / Vercel などの公式発表、Forbes / TechCrunch / CIO / Reuters転載などの二次情報、Product Hunt 系集計、Reddit 投稿だった。  
不足していた情報源は、App Store / Google Play の実レビュー、TikTok / YouTube Shorts の実広告・投稿、Sensor Tower 系の市場データ、日本語圏の個人開発者事例、GitHub の実装リポジトリ、価格ページの直接確認。  
全体として、一次情報は強いが、ユーザー不満と収益実態の現場データが弱かった。

## 収益化判断の精度
収益化判断は中程度に近かった。Play Shorts 台本生成、App Store課金KPI分析、AIツール比較ページ生成、MCP対応SaaSは、個人開発で短期試作しやすく収益導線も想像しやすい。  
一方で、実際に売れるかの判断に必要な競合価格、レビュー不満の量、検索需要、広告出稿状況、既存テンプレ販売の実績が不足していた。  
今日の「収益化ヒント」は方向性としては有効だが、実装着手前には 1 テーマにつき競合 5 件の価格・低評価理由・集客導線を確認する必要がある。

## 見落とした可能性
- App Store / Google Play 上位AIアプリの低評価レビューから、具体的な有料機能ニーズを抽出する作業。
- TikTok / YouTube Shorts で実際に伸びているAIアプリ広告の訴求、尺、CTA、コメント欄の不満。
- Product Hunt 上位AIツールの料金ページ、無料枠制限、オンボーディング、初回課金までの導線。
- 日本語圏のAIツール利用者、個人開発者、業務自動化ニーズの観測。

## 次回改善すること
- App Store / Google Play の上位AIアプリを各5件選び、価格、無料枠、課金CTA、低評価レビュー理由を表にする。
- Reddit やSNSの不満は、同種の不満が最低3件以上ある場合だけ「ユーザー不満」として採用する。
- Product Hunt 上位ツールは、紹介文だけでなく pricing / signup / docs まで確認し、収益導線を1行で記録する。
- 昨日分Markdownを必ず参照し、重複テーマと新規テーマを分けて書く。
- TikTok / Shorts は「AIアプリ名 + ad / demo / review」で実投稿の訴求を確認し、動画化しやすさを評価する。
