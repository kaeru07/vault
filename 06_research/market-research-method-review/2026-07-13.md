# 市場調査方法レビュー 2026-07-13

## 今日使った調査方針
収益化しやすさ、短期リリース可能性、Claude Code / Codex で量産しやすい領域を優先した。  
対象は AI ニュース、AI ツール、MCP、AI Agent、App Store / Google Play、Shorts / TikTok、学習アプリを中心に見た。  
特に「小型業務エージェント」「AI 動画制作支援」「MCP ツール課金」を収益化候補として重く扱った。  
一方で、麻雀・将棋・広告収益系は方針上の対象だが、本日はほぼ深掘りできていない。

## 良かった点
- Cloudflare x402 paid MCP tools を、個人開発者が小機能を販売する収益化導線として拾えた。
- 汎用 AI アプリではなく、承認・監査・実行証跡を持つ小型エージェントに寄せた判断は方針と合っていた。
- AI ツール調査では公式 URL が多く、Gemini CLI、Codex CLI、Cloudflare MCP、GitHub Copilot cloud agent の根拠が比較的明確だった。
- 市場調査・AI ニュース・AI ツールの 3 本で、MCP / Agent / Codex 活用という共通軸を作れた。
- 取得状況を partial と明記し、未確認情報を断定しすぎない姿勢は保てた。

## 弱かった点
- App Store / Google Play の公式または準公式チャートを直接取得できず、需要判断が二次情報寄りになった。
- AI ニュース内で公式確認済みと二次情報の境界が一部強く見えすぎる箇所があり、根拠の温度差をもっと本文中に反映すべきだった。
- 各ツールの最新料金、無料枠、制限、商用利用条件が未確認のまま収益化評価に進んでいる。
- ユーザー不満の根拠がレビュー欄・Reddit・X などの一次的な声ではなく、推測やニュース記事中心だった。
- 麻雀、将棋、広告収益系など、方針にある個別市場の調査が抜けた。

## 情報源の偏り
今日カバーした情報源は、OpenAI / Anthropic / GitHub / Cloudflare / Gemini CLI / MCP Registry などの公式情報、Business Insider / The Verge / TechRadar / Times of India などの二次情報、arXiv、Similarweb 周辺だった。  
不足していたのは、App Store 公式 RSS、Google Play 当日ランキング、Appfigures 等のチャート系、Product Hunt 当日上位、実際のアプリレビュー、Reddit / X / YouTube コメントなどのユーザー不満ソースである。  
特に OpenAI / Claude / Codex / MCP に寄り、消費者向けアプリ、ゲーム、学習、麻雀・将棋領域の一次観測が弱かった。

## 収益化判断の精度
収益化に近かった案は、Cloudflare x402 を使った有料 MCP ツール、AI 動画投稿前チェック、Codex タスク承認ボード、小規模事業者向け日次オート秘書である。  
ただし、実際の支払い意欲を示すレビュー、検索需要、価格比較、競合の課金状況が不足しているため、現時点では「作れそう」止まりで「売れそう」の根拠は弱い。  
MCP ツール課金は収益化導線として有望だが、買い手が人間ではなく Agent になる市場がどの程度早く立ち上がるかは未検証である。  
次回はアプリ案ごとに、既存競合、価格、低評価レビュー、初回販売チャネルを最低 1 つずつ確認する必要がある。

## 見落とした可能性
- 麻雀・将棋の棋譜解析、学習支援、対局後レビューなど、方針にあるニッチ市場の AI 化余地。
- App Store / Google Play の低評価レビューから拾える、課金直前の不満や解約理由。
- Product Hunt や Indie Hackers 周辺の、すでに売れている小型 AI ツールの価格・導線。
- YouTube / TikTok の収益化ポリシー原文と、AI 生成コンテンツへの実際の制限。

## 次回改善すること
- Apple RSS、Google Play、Appfigures または Similarweb のうち最低 2 系統で、当日ランキングを直接確認する。
- 競合アプリを 3 件に絞り、価格、無料枠、低評価レビュー理由、直近アップデートを表で記録する。
- Product Hunt の当日 AI ツール上位から、MCP / Agent / 動画 / 学習に関係するものを最低 3 件見る。
- Reddit / X / アプリレビューから、ユーザー不満を最低 2 件は一次情報として拾う。
- 麻雀または将棋のどちらかを 1 日 1 件だけでも固定観測し、AI 応用と収益化余地を確認する。
