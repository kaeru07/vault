## AI業界の変化

- 短期: モデル単体性能より「エージェント運用コスト・安全性・権限管理」が差別化軸になっている。OpenAI の container session 分単位課金、Claude Code の sandbox / credentials 保護、DeepMind の control roadmap が同じ方向を示す。
- 短期: ChatGPT 広告のように、AI チャット面そのものが配信・広告在庫になりつつある。個人開発者は SEO だけでなく「AI 内導線」「会話内コマース」「広告計測」を意識する必要がある。
- 中期: Gemini 3.5 Pro 延期報道から、 frontier model は発表日より実運用の安定性、token 消費、agent 長期タスク性能が重視される流れ。
- 中期: Codex / Claude Code / Gemini 系 coding agent の競争は IDE 補助から、PR 作成、レビュー、セキュリティ修正、広告・業務自動化まで拡張している。

## 個人開発への影響

- 今週、AI agent に渡す権限を棚卸しする。`.env`、SSH 鍵、production DB、課金 API、GitHub token は agent から読めない構成を標準にする。
- OpenAI / Claude / Gemini のモデル名をハードコードしている箇所を確認し、検証用と本番用を分ける。`chat-latest` のような可変 snapshot は評価環境向けに限定する。
- coding agent を「新規機能丸投げ」より、バグ修正、テスト追加、PR レビュー、LP 差分生成、広告クリエイティブ量産に使う。失敗時の被害が小さく、収益化に近い。
- AI チャット内広告・commerce 導線を想定し、プロダクト説明、FAQ、比較表、価格表を LLM が引用しやすい構造化コンテンツにする。
- Gemini 3.5 Pro 前提の実装予定は 7 月以降にずらし、今週は Claude Code / Codex / 現行 Gemini API で動く抽象化レイヤーを作る。

## 収益化への影響

- 広告: ChatGPT 広告報道により、検索広告・SNS 広告に加えて「AI 会話面の広告」が新しい獲得チャネルになる可能性がある。公式仕様は未確認なので、今は計測設計と商品説明の整備が先。

- SaaS: Safety Dashboard や agent containment の流れから、AI 利用ログ、権限監査、プロンプト監査、社内 agent 管理の小規模 SaaS 需要が伸びる。
- 配信: Codex / Claude Code の改善を題材に、「agent に任せてよい作業・任せてはいけない作業」の検証動画は Shorts / TikTok 向き。
