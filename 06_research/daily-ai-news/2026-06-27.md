## AI業界の変化

- **短期: エージェント開発が「実験」から「課金管理対象」へ移行**
  - Codex / Claude Managed Agents / Gemini Computer Use / Copilot cloud agent が並び、個人開発でも「何回呼ぶか」「1 セッションいくらか」の設計が必要。

- **短期: API 制限緩和とモデル廃止が同時進行**
  - Claude はレート制限を引き上げる一方、Opus 4.7 fast mode や旧 Sonnet / Opus の廃止が進む。Gemini も Veo / Imagen / Gemini 2.0 系の shutdown が続く。

- **中期: チャット UI よりワークフロー UI が主戦場**
  - Codex 論文では複数エージェント並列管理や Skills 利用が伸びている。個人開発でも単発チャットより、定型タスク・レビュー・生成・修正の一連の流れを商品化しやすい。

- **中期: 企業 AI 支出は冷え込みより再配分**
  - RBC 調査を報じた Business Insider では、企業の AI 予算が新規枠として確保され、トークン費用は多くの企業で管理可能とされる。ただし一次資料未取得のため詳細数値は未確認。
  - 根拠: https://www.businessinsider.com/enterprise-ai-spending-grows-openai-leads-rbc-reveals-2026-6

## Claude/Codex/OpenAI/Gemini関連

**Claude**

- 2026-06-26、Claude API の rate limits が引き上げ。個人開発ではバッチ処理、長文生成、エージェント実行の同時数を増やしやすい。
- 2026-06-25、Claude Opus 4.7 fast mode は 2026-07-24 削除予定。移行先は Opus 4.8 fast mode。

**Codex**

- Codex 利用者は 2026 年前半に 5 倍超。10% 超のユーザーが週内に 3 つ以上の Codex agent を並列管理、26.6% が Skills を利用したとの論文報告。
- GitHub Docs 上で OpenAI Codex Visual Studio Code integration が premium request 対象として明記。preview 期間中もコスト監視が必要。

**OpenAI**

- 2026-06-24、`chat-latest` snapshot 更新。ただし公式は本番 API では GPT-5.5 の利用を推奨。
- 2026-06-23、Safety Usage Dashboard 公開。`safety_identifier` を付ける設計にすると、拒否・ブロック状況の分析がしやすい。

**Gemini**

- 2026-06-24、Gemini 3.5 Flash の Computer Use tool が public preview。ブラウザ操作型の個人向け自動化 SaaS に使いやすい。
- 2026-06-15、Veo 2.0 / 3.0 系は 2026-06-30 shutdown、Imagen 4 / Gemini 3 Image 系は 2026-08-17 shutdown 予定。動画・画像生成アプリはモデル ID 移行が必要。

## 個人開発への影響

1. **今週、AI API のモデル ID と廃止予定を棚卸しする**
   - Gemini の Veo / Imagen、Claude の旧 Opus / Sonnet、OpenAI の snapshot 依存を確認し、固定 ID と fallback を整理する。

2. **エージェント系機能は「1 実行あたり原価」を先に出す**
   - Copilot / Codex / Claude / Gemini Computer Use は便利だが、セッション単位・リクエスト単位・ツール実行単位で原価が変わる。

3. **AI コーディング補助は Claude Code / Codex の二系統で検証する**
   - Codex 利用拡大は明確。Claude 側も rate limit 緩和があり、個人開発では「要件定義 Claude、実装修正 Codex」など役割分担を試す価値がある。

4. **安全性ログを最初から保存する**
   - OpenAI Safety Usage Dashboard の流れを見ると、商用 AI アプリは拒否・ブロック・ユーザー識別子の運用が収益化後に重要になる。

5. **Computer Use は小さな自動化商品から試す**
   - Gemini の public preview により、ブラウザ入力、管理画面操作、定期チェックなどの小型 SaaS 化余地が増えた。ただし本番利用可否は利用規約と安定性を要確認。

## 収益化への影響

- **課金**
  - Copilot / Codex / Claude / Gemini 系の agent 実行は従量コストが増えやすい。月額固定だけでなく、実行回数制限・追加クレジット・高負荷機能の上位プラン化が必要。

- **SaaS**
  - Claude rate limit 緩和と Gemini Computer Use preview により、個人開発でも「社内作業代行」「調査代行」「管理画面操作代行」型の小規模 SaaS を作りやすい。

- **配信 / 動画化**
  - Codex の利用拡大、複数 agent 管理、Computer Use はデモ映えする。Shorts / TikTok では「AI に実作業を任せる」比較企画が作りやすい。

- **広告**
  - 今日の主要ニュースに直接の広告単価改善はない。ただし OpenAI docs 上では ChatGPT Apps / Commerce / Ads 関連導線が確認でき、今後 ChatGPT 内配信・計測が個人開発の販売チャネルになる可能性は継続監視。
