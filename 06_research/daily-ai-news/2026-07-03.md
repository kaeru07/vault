## 記事本文
今日の AI 業界で最も重要だった変化は、単体チャットの性能競争ではなく、エージェントをどれだけ安く、長く、検証可能に動かせるかへ競争軸が移ったことだ。Anthropic は Claude Sonnet 5 を公開し、計画、ブラウザ、ターミナル、コーディング、知識作業をこなす「最も agentic な Sonnet」と位置づけた。特に個人開発者にとって大きいのは、Opus 級に近い作業能力をより安い Sonnet 系で使える点で、導入価格は短期の試作量産に向いている。

OpenAI 側も同じ方向に動いている。GPT-5.6 Sol は限定プレビュー段階で、一般利用者がすぐ自由に使える状態ではないが、Codex と API から先に展開される点が重要だ。OpenAI の公開分析では、Codex はすでに OpenAI 社内の非エンジニア部門でも主要な AI 作業手段になっており、AI コーディングツールが「コード補完」ではなく、調査、分析、業務自動化、社内ツール作成まで吸収し始めている。

Google はモデル単体より、開発者がエージェントを安全に製品化するための土台を押し出している。ADK 2.0 は LLM に全判断を任せるのではなく、決定的な workflow graph と LLM node を組み合わせる設計を明確にした。Genkit Agents API も、会話履歴、tool loop、streaming、永続化をまとめ、個人開発者が agentic SaaS を作る際の配管作業を減らす方向だ。

収益化の観点では、今週やるべきことは「最新モデルを触る」だけではない。個人開発者は、既存アプリに小さな agent workflow を 1 つ入れ、実行ログ、再試行、ユーザー承認、課金単位を設計するべきだ。学習アプリなら添削から学習計画作成へ、麻雀・将棋なら局面分析から復習レポート生成へ、業務 SaaS なら問い合わせ処理から実行前確認付き自動処理へ広げられる。モデル名の勝ち負けより、失敗しても課金・信用・安全性が壊れない実装を持つ個人開発者が有利になる局面に入っている。

## AI業界の変化
- 短期: 高性能モデルの提供は、政府・安全評価・限定 preview を挟む段階的リリースが標準化しつつある。
- 短期: Sonnet 5、ADK 2.0、Genkit Agents API の流れから、低コスト agent 実行と workflow 化が実装テーマになった。
- 中期: AI agent は「チャット UI」から「検証可能な作業ログ、HITL、権限制御、再現性」を持つ業務実行基盤へ移る。
- 中期: jailbreak / cyber capability の評価基準が、モデル公開速度と利用可能地域に直接影響する可能性が高い。

## 個人開発への影響
- 今週は既存アプリに「1 回の応答」ではなく「複数手順の agent workflow」を 1 本だけ追加する。
- LLM 任せの自動実行ではなく、購入、送信、削除、外部投稿の直前に HITL 承認を入れる。
- Claude Sonnet 5 の導入価格期間を使い、コーディング、レビュー、リサーチ、レポート生成の原価を再計算する。
- Google ADK / Genkit 型に合わせ、状態管理、実行ログ、再試行、失敗時の復旧を先に設計する。
- Shorts / TikTok 向けには「AI が 1 日で作った小型 SaaS」「Agent workflow の失敗と修正」を動画化しやすい。

## 収益化への影響
- SaaS: agent 実行単位の従量課金、月間実行回数制限、上位プランで長時間 agent を解放する設計が作りやすくなった。
- 配信: Claude / Codex / Gemini agent 比較、ADK 2.0 の workflow 化、HITL 実装は開発者向け動画ネタとして強い。
