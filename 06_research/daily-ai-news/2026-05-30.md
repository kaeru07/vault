## AI業界の変化
1. **短期: エージェント競争が「チャット」から「実行環境」へ移行**
   - Claude Code Dynamic workflows、OpenAI Codex、Google Managed Agents が同じ方向に収束。個人開発でも「生成」より「検証・実行・修正」の自動化が重要になる。

2. **短期: 高額 frontier model の使い分けが必須化**
   - Claude Opus 4.8、GPT-5.5 系、Gemini 3.5 Flash のように、性能・速度・価格の組み合わせが複雑化。全タスクを最上位モデルに投げる運用は収益性を削る。

3. **中期: SDK / MCP / Connector が競争優位に**
   - Anthropic による Stainless 買収、Google の MCP 対応 Chrome Enterprise サーバー、OpenAI Codex の企業環境連携から、モデル単体より「外部ツールへ安全につなぐ層」が価値化している。

4. **中期: AI 収益化は広告・SaaS・業務代行の三方向へ分岐**
   - OpenAI は ChatGPT 広告テスト、Anthropic は企業・SMB 導入、Google は検索・Android・Workspace 起点。個人開発は小規模 SaaS か業務特化ツールが現実的。

## 個人開発への影響
1. **今週からモデルルーティングを前提にする**
   - 企画・設計・難所修正は Claude Opus / GPT 系、反復生成・分類・要約は Flash / mini 系に分ける。コストログを必ず残す。

2. **AI エージェント用の安全な実行環境を標準化する**
   - Docker / ephemeral workspace / read-only 権限 / approval log を用意。Codex、Claude Code、Gemini Managed Agents の比較がしやすくなる。

3. **MCP / API connector 対応を優先する**
   - 個人開発ツールでも、Notion、Google Workspace、GitHub、決済、広告、分析への接続が価値になる。単体チャット UI だけでは差別化しにくい。

4. **ベンチマークより自分のユースケース eval を作る**
   - Terminal-Bench などの公表値は参考に留め、麻雀、将棋、学習アプリ、広告運用、Shorts 台本生成など、自分の収益領域で小型 eval を作る。

5. **AI 生成物の来歴表示を早めに検討する**
   - 画像・音声・動画・広告素材を扱う場合、C2PA / SynthID 的な表示や生成履歴保存が信頼性と審査通過に効く可能性がある。

## 収益化への影響
1. **SaaS: 業務特化エージェントが作りやすくなる**
   - Google Managed Agents、Claude Dynamic workflows、Codex の企業連携により、個人でも「特定業務を最後まで処理する小型 SaaS」を作る土台が増えた。

2. **課金: usage-based pricing の設計が必須**
   - 高性能モデルの token 消費が大きいため、月額固定だけでは赤字化しやすい。無料枠、従量課金、上限設定を最初から入れるべき。

3. **広告: ChatGPT 広告テストは注視対象**
   - OpenAI の広告関連発表は、AI 検索 / AI アシスタント内広告の市場形成につながる。個人開発では「広告収益アプリ」より「広告運用支援ツール」の方が近い。

4. **配信 / 動画化: Gemini Omni と AI 動画生成は Shorts 量産に直結**
   - ただし著作権、来歴表示、プラットフォーム審査が不透明。収益化前提なら、素材管理と出典管理をワークフロー化する必要がある。
