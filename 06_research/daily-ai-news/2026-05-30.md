## AI業界の変化

- 短期: コーディング AI は「補完」から「長時間・並列・検証付き実行」へ移行。Claude Code Dynamic Workflows と Codex のモバイル/デスクトップ連携が同じ方向を示している。
- 短期: Google は Gemini 3.5 Flash を検索、アプリ、API、開発環境に同時配備し、低遅延・広範囲配布で攻めている。
- 中期: Anthropic の大型調達により、モデル性能だけでなく計算資源・企業導入・クラウド配備が競争軸としてさらに重くなる。
- 中期: エージェント利用拡大により、MCP、権限管理、監査ログ、セキュリティ評価が個人開発でも差別化要素になり始めた。

## 個人開発への影響

1. 今週は Claude Code / Codex のどちらでも、単発プロンプトではなく「計画、実装、検証、差分確認」までのワークフローをテンプレ化する。
2. Dynamic Workflows や類似の並列エージェント機能は高コスト化しやすいため、小さなリポジトリ単位で費用対効果を測る。
3. Gemini 3.5 Flash は速度と配布範囲が強いので、低単価・大量処理系のプロトタイプ候補に入れる。
4. MCP や外部ツール連携を作る場合は、最初から権限、ログ、失敗時の停止条件を設計に入れる。
5. AI ニュース、学習、将棋・麻雀、Shorts 台本生成などは「毎朝自動実行する小型エージェント」として商品化しやすい。

## 収益化への影響

- SaaS: エージェント監査、実行ログ、権限管理、MCP 接続管理は小規模 SaaS の狙い目。大企業向けだけでなく、個人開発者・小規模チーム向けに薄く作れる。
- 課金: Claude/Codex/Gemini の高機能化により、月額ツールの価値は「モデル性能」ではなく「業務テンプレ」「保存された文脈」「自動実行履歴」に移る。
- 配信: Opus 4.8、Dynamic Workflows、Gemini Spark は動画化しやすい。比較デモ、失敗例、トークン消費検証が Shorts/TikTok 向き。
- 広告: Gemini の検索・AI Mode 強化により、SEO 依存メディアは流入低下リスクあり。AI 検索で引用されやすい構造化コンテンツが重要。
