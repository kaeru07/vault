## 記事本文
今日の AI ニュースは、単なるモデル性能競争から「仕事を丸ごと処理するエージェント」と「その利用コスト」をめぐる競争へ、焦点がさらに移った一日だった。OpenAI は GPT-5.6 と ChatGPT Work を発表したと報じられ、ChatGPT と Codex を統合して、文書作成、表計算、ブラウザ操作、コード変更までをデスクトップ上で扱う方向を強めている。公式一次情報は取得できていないため細部は未確認だが、方向性としては「チャット画面」ではなく「作業環境そのもの」を取りに来ている。

同時に Meta は Muse Spark 1.1 と Meta Model API を公開し、開発者向け AI モデルで有料 API に踏み込んだ。重要なのは、Meta がオープンモデルの象徴から、低価格の商用 API 事業者へ姿勢を変えつつある点だ。二次情報では 100 万入力トークンあたり 1.25 ドル、出力 4.25 ドルという価格が報じられているが、公式価格表未取得のため未確認とする。それでも、個人開発者にとっては「高性能モデルをどれだけ安く組み込めるか」がアプリの粗利を左右する局面に入った。

一方で、Anthropic 周辺では政府による高性能モデルのレビュー、規制解除、無料提供延長が続き、モデル公開そのものが安全保障と商用戦略の交点になっている。Claude Fable 5 や Mythos 5 の扱いは、単に新モデルが使えるかどうかではなく、海外ユーザー、企業導入、セキュリティ機能、輸出管理が個人開発サービスの可用性にも影響し得ることを示している。

コーディングエージェントについては、Microsoft の大規模調査が実務導入の根拠を補強した。CLI 型エージェント利用者の PR マージが約 24% 増えたという結果は、個人開発でも「1 人で作る」から「複数エージェントを運用する」へ移る合理性を示す。ただし、Claude Code の攻撃実証報道が示すように、未知のリポジトリ、README、セットアップコマンドをそのまま実行させる運用は危険だ。今週取るべき一歩は、新モデルを試すことだけではない。価格、権限、ログ、サンドボックス、収益単価を同時に見直し、AI エージェントを利益が出る範囲で安全に使う仕組みを作ることだ。
