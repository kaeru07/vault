## AI業界の変化

- 短期: AI安全の焦点が「モデルの発言」から「AIで実行可能になる周辺産業の規制」へ広がっている。今回は合成DNA/RNAの注文審査が中心。
- 短期: Gemini 障害により、個人開発でも LLM gateway、fallback model、キューイング、障害時メッセージ設計が必須に近づいた。
- 中期: Claude Code / Codex / Gemini CLI などの coding agent は、単なる生成補助ではなく「再現実験・解析実行・業務自動化」の評価対象になっている。
- 中期: Apple のOS統合AIが進むと、汎用チャット・要約・予定作成アプリは差別化が難しくなり、ドメイン特化やデータ連携が重要になる。

## 個人開発への影響

1. 今週からAI機能に fallback model を入れる。Gemini単独、Claude単独、OpenAI単独の構成は障害時に収益機会を失いやすい。
2. AIアプリは「汎用チャット」ではなく、麻雀、将棋、学習、広告運用、動画台本、業務テンプレなどの縦特化に寄せる。
3. Claude Code / Codex を使う場合、実装完了後に「範囲外変更」「不要なファイル編集」「解釈の飛躍」をチェックするテスト手順を追加する。
4. 研究・分析・レポート生成系SaaSでは、出典URL、再現ログ、入力データ、失敗時の理由を保存する設計にする。
5. iOS向けAIアプリは、Siri AIに吸収されにくい独自DB、専門ワークフロー、課金価値のある成果物生成を前面に出す。
