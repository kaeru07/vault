## AI業界の変化

- 短期: AI coding agent は「単発コード生成」から「永続環境で長時間働く実行者」へ移行中。Codex + Ona報道はこの流れを強める。
- 短期: 価格競争の兆候が強い。個人開発では、OpenAI / Anthropic / Gemini / OSSのルーティング設計が収益性を左右する。
- 中期: frontier model は高性能化と同時に、安全制限・利用制限・高価格化が進む可能性がある。
- 中期: MCP、ローカルファイル連携、クラウドVM型エージェントが、個人開発の自動化商品・業務代行SaaSの基盤になりやすい。

## 個人開発への影響

- 今週はAI API呼び出しを「高性能モデル固定」から「安価モデル + 高性能モデルの段階ルーティング」に変えるべき。
- Codex / Claude Code前提で、リポジトリにテスト、lint、再現手順、仕様メモを整備する。エージェントが長時間作業しやすい構造が差になる。
- SaaSや自動化ツールでは、ユーザーごとのAI利用量上限、ジョブキュー、失敗時の再実行制御を最初から入れる。
- AI生成機能を売る場合、「無制限」よりもクレジット制・従量制・高品質モード課金に寄せる。
- MCP対応や外部ツール連携を、小さな業務自動化アプリの差別化要素として検討する。

## 収益化への影響

- 課金: AIコスト上昇と価格競争が同時進行しているため、月額固定だけでは粗利が読みにくい。クレジット制が有利。
- SaaS: 長時間エージェント基盤が進むと、単なるチャットUIより「完了成果物」を売るSaaSが作りやすい。
- 配信: 「Claude Code vs Codex」「AI料金節約」「OSSモデル併用」は動画化・記事化しやすいテーマ。
- 広告: AIニュース要約・AIツール比較サイトは継続需要があるが、一次情報リンクと未確認ラベルがないと信頼性が落ちる。
