## 記事本文
2026-07-05のAIツール棚卸しでは、個人開発向けの新しいアプリそのものよりも、AI Agentを「運用できる状態」に近づける周辺ツールの更新が目立った。VercelはAgent RunsをMCPとCLIから参照できるようにし、GitHubはCopilot Agentのセッションデータをストリーミング可能にした。これは、AIに作業を任せる段階から、AIの作業を観測し、再現し、改善する段階へ市場が移っていることを示している。

個人開発の観点では、最も実用に近いのはkonsistentである。AIにコードを書かせると、型チェックやESLintでは拾えない「フォルダ構造」「必須export」「命名パターン」のズレが残りやすい。konsistentはそこをCLIで機械的に検出できるため、Claude CodeやCodexで量産する小規模SaaS、テンプレート販売、学習アプリ、広告収益アプリの保守コストを下げやすい。無料でローカル実行できる点も、個人開発の初期費用を抑える意味で大きい。

一方、Vercel AI Gateway Routing RulesやGitHubのCopilot監査機能は、個人がすぐ使って収益化するというより、将来の運用設計を先取りする材料である。モデルの退役、単価上昇、品質低下が起きた時に、アプリコードを変えずにモデルを差し替えられる仕組みは、AI機能を商品に組み込む開発者にとって保険になる。小さなプロダクトでも、要約、翻訳、コード生成、チャット補助などを複数モデルで動かすなら、早い段階で抽象化しておく価値がある。

Kimi K2.7 CodeがGitHub Copilotに入ったことも重要だ。open-weightモデルをCopilotのモデルピッカーから選べる流れは、開発者が「高性能モデルを常時使う」だけでなく、「安いモデルで日常作業を回し、難所だけ高性能モデルに切り替える」運用へ向かう可能性を示す。個人開発では、AI利用料が利益率を圧迫しやすいため、低コストモデルの品質が上がるほど、小さな有料ツールや買い切りテンプレートの採算が取りやすくなる。

次に取るべき一歩は、Agentを増やすことではなく、Agentが壊した時に検知できる最小セットを作ることだ。具体的には、既存のTypeScriptプロジェクトにkonsistentを1ルールだけ導入し、CodexやClaude Codeに修正させ、CIで失敗させる流れを試す。あわせてVercel MCPやCLIでAgent Runsを取得する検証を行えば、将来の自動開発パイプラインを小さく始められる。

## 個人開発で活用できそうなもの
- konsistent: AIが生成したTypeScriptコードの構造ズレをCIで検出し、テンプレート販売や小規模SaaSの保守負荷を下げる。
- Vercel Agent Runs: AI Agentの失敗ログ、token使用量、tool callを見て、開発自動化のコストと品質を改善する。
- AI Gateway Routing Rules: 高額モデルから低額モデルへ切り替えるルールを用意し、AI機能付きアプリの粗利を守る。
- Kimi K2.7 Code in Copilot: 日常的な修正や調査を低コストモデルで回し、個人開発のAI支出を抑える。
- Copilot Agent Session Streaming: 個人用途では過剰だが、受託・チーム開発のAI利用監査テンプレートとして転用余地あり。

## Claude Code/Codex連携可能性
- konsistent: Claude Code/Codexに`konsistent.json`を生成・修正させ、規約違反を自動修正する。
- Vercel MCP Agent Runs: CodexからMCP経由で直近Agent実行を読み、失敗原因やtoken過多を自己診断させる。
- Vercel CLI `agent-runs`: MCPが使えない環境でもCodex/Claude CodeからCLIを叩いてtraceをMarkdownで確認する。
- AI Gateway Routing Rules: Codexにモデル移行ルールを作らせ、コスト上限や退役モデル対策を自動化する。
- GitHub Copilot usage records API: Agent利用履歴を分析し、どの作業をClaude Code/Codexへ移すべきか判断する。
