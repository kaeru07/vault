## 記事本文
今日のAIツール調査では、単体のチャットボットよりも「開発・業務の作業面を丸ごと預けるエージェント環境」への移行がはっきり出ていた。OpenAIはChatGPT WorkとCodexを同じ体験の中に置き、ファイル編集、開発、ブラウザ利用、スケジュール実行を一つの作業場にまとめようとしている。AnthropicもClaude CodeをCLIだけでなく、Web、デスクトップ、モバイル、Slack、Routinesへ広げており、エージェントが端末内の補助ツールではなく、継続的に動く作業主体になる流れが強い。

個人開発者にとって重要なのは、AIの性能差そのものよりも、作業の粒度が「1プロンプト1回答」から「1タスク1成果物」に変わっている点だ。Codexは複数エージェント、クラウド環境、Skills、定期作業を前提にしており、Claude CodeはMCP、hooks、skills、background agents、remote controlを組み合わせて、テスト、レビュー、依存更新、ドキュメント同期まで担当できる。Gemini CLIは無料枠が大きく、Apache-2.0のCLIとして触りやすいため、初期費用を抑えたい個人開発では有力な試験対象になる。

収益化の観点では、今日の変化は「AIツールを作る」よりも「AIエージェントが使いやすい小さな業務導線を作る」方向に意味がある。たとえば、麻雀・将棋・学習アプリのような個人開発テーマでも、問題生成、解説、復習通知、ユーザー別レポートをエージェント化すれば、月額課金の理由を作りやすい。さらにMCPサーバーとして外部データや自作DBを公開すれば、Claude CodeやCodexから運用・分析・改善までつなげられる。

ただし、導入には注意点もある。価格は無料枠だけでは本番運用の予測が難しく、ChatGPT系は公式ページで月額金額が取得できない部分があった。ClaudeはPro $20/月、Max $100/月からと読み取れたが、重い開発用途では上位プラン前提になりやすい。次に取るべき一歩は、Gemini CLIを無料枠で検証し、Claude CodeまたはCodexに小規模リポジトリのテスト追加、PRレビュー、依存更新を任せ、実作業時間と課金の見合いを1週間単位で測ることだ。

## 個人開発で活用できそうなもの
- Gemini CLI: 無料枠が大きく、MCP対応もあるため、個人開発の調査、コード生成、軽い運用スクリプト作成に向く。
- Claude Code: 既存リポジトリのテスト追加、PRレビュー、依存更新、定期メンテナンスの自動化に向く。課金はPro $20/月またはMax $100/月から。
- Codex in ChatGPT: 複数エージェントで機能実装、リファクタ、レビューを並列化する用途に向く。価格は公式ページ上で未取得。
- MCPサーバー: 自作アプリのDB、Notion、Google Drive、FigmaなどをAIエージェントから扱う入口として有用。自前実装ならセルフホスト可。
- ADK Arena: すぐ導入するツールというより、どのAgent SDKを採用するかの比較軸として有用。公式実装・リポジトリは未確認。

## Claude Code/Codex連携可能性
- MCP: Claude Code、ChatGPT/Codex、Gemini CLIの共通接続面として、自作APIやDBをエージェントに渡せる。
- Claude Code Agent SDK: Claude Codeのツール、権限、オーケストレーションを使った独自エージェント構築に使える。
- Codex SDK / MCP Server / GitHub Action: CodexをCLI外の自動ワークフローやGitHub上の開発作業に組み込める可能性がある。
- Gemini CLI MCP Integration: `~/.gemini/settings.json`でMCPサーバーを追加し、外部ツール連携を試せる。
- OpenAI Apps SDK / MCP Apps in ChatGPT: ChatGPT内にUI付きアプリを出す導線があり、個人開発SaaSの配布面として検討余地がある。

## MCP/Agent系情報

- Claude Code MCP: Google Drive、Jira、Slack、自作ツールなどに接続でき、MCP quickstartも公式ドキュメントに掲載。
- Gemini CLI MCP: GitHub READMEでMCP server integration、custom extensions、headless mode、GitHub Actions連携の導線を確認。
- ADK Arena論文: 51のPython ADKを比較し、フレームワーク選定がエージェントの成功率と実装コストに影響することを示す。ただし実運用ツールとしての導入可否は未確認。
