## 記事本文
今日のAIツール調査では、単体のチャットAIよりも「作業環境そのものをAIに接続する」流れが明確だった。OpenAIのCodexはChatGPT、IDE、CLI、クラウドを横断する作業面として整理され、公式ページでもワークツリー、クラウド環境、複数エージェント、スケジュール実行が前面に出ている。さらにCodex SDKは、ローカルCodexをTypeScriptやPythonから制御できるため、個人開発者が自分のCI、レビューBot、リリース準備Botに組み込みやすい形になっている。

一方で、Claude Codeは2026年7月6日の2.1.202で、動的ワークフローサイズ設定、workflow単位のOpenTelemetry属性、Remote ControlやMCP設定まわりの修正が入っていた。これは派手な新機能というより、複数エージェントを日常運用するための観測性と安定性の改善に近い。個人開発では「一発で全部作らせる」より、朝の依存更新、PRレビュー、CI失敗調査、リリースノート生成のような小さな定期作業を任せる価値が高い。

収益化の観点では、v0とDevinの位置づけが分かりやすい。v0は無料枠から始めて、GitHub同期、Vercelデプロイ、Design Mode、テンプレートを使い、検証用LPや小型SaaSを短時間で出せる。Devinは無料枠がある一方、Pro 20 $/月、Max 200 $/月、Teamの座席課金が明示されており、個人開発では本番実装を丸投げするより、面倒な修正・テスト・レビュー補助に限定した方が費用対効果を測りやすい。

MCPまわりでは、GitHub MCP Server 1.5.0、CodexのMCP対応、Claude CodeのMCP連携が揃い、ツール接続は独自プラグインより標準化に寄っている。特にGitHub、Figma、Playwright、Sentry、Context7のようなMCPは、個人開発の収益化に直結しやすい。次に取るべき一歩は、新規アプリのアイデアを増やすことではなく、既存リポジトリにGitHub MCP、Codex SDK、v0のどれか一つを接続し、1日1回の自動レビューやLP生成など、売上に近い反復作業を小さく自動化することだ。

## Claude Code/Codex連携可能性
- GitHub MCP Server: Codex/Claude CodeからIssue、PR、Actionsを扱い、個人開発のレビュー・リリース作業を自動化できる。https://github.com/github/github-mcp-server
- Codex SDK: 自作Node.js/PythonアプリからCodex threadを開始・再開し、開発支援機能として組み込める。https://learn.chatgpt.com/docs/codex-sdk
- Codex GitHub Action: PRレビュー、リリース前チェック、移行作業をGitHub Actions上でCodexに実行させられる。https://learn.chatgpt.com/docs/github-action
- Codex MCP設定: STDIO/HTTP/OAuth対応のMCPサーバーをCodex CLI、IDE extension、ChatGPT desktopで共有できる。https://learn.chatgpt.com/docs/extend/mcp
- Claude Code Agent SDK: Claude Codeのツール・権限・オーケストレーションを使った独自エージェント構築に使える。https://code.claude.com/docs/en/agent-sdk/overview

## MCP/Agent系情報
- Model Context Protocol公式ドキュメント: MCPはAIアプリを外部データ、ツール、ワークフローへ接続するオープン標準として説明され、Claude、ChatGPT、VS Code、Cursorなどの対応が確認できた。https://modelcontextprotocol.io/
- MCP Reference Servers: 公式リファレンス実装としてFilesystem、Git、Memory、Fetch、Sequential Thinkingなどが整理され、MCP Registryへの誘導も確認。https://github.com/modelcontextprotocol/servers
- GitHub MCP Server: GitHub公式のMCPサーバーで、リポジトリ、Issue、PR、Actions、code security等のtoolsetsを選択可能。最新リリースは1.5.0。https://github.com/github/github-mcp-server
- Codex MCP: CodexはSTDIO/Streamable HTTP、OAuth、tool allow/deny、project-scoped configに対応し、Context7、Figma、Playwright、Sentry、GitHubなどの接続例を公式に掲載。https://learn.chatgpt.com/docs/extend/mcp
