## 個人開発で活用できそうなもの
- Claude Fable 5: 高単価だが、複雑なリファクタ・移行・検証タスクを短時間で処理する用途に向く。個人開発では常用よりも難所だけスポット利用。
- GitHub Copilot Agent Mode: Issue から PR 下書きまで任せる運用が現実的。小規模SaaS、学習アプリ、広告収益アプリの保守タスク自動化に使いやすい。
- GitHub Spark: ノーコード寄りに MVP を素早く作れる可能性がある。動画化しやすい「1プロンプトでアプリ化」系ネタにも向く。
- Microsoft Agent Framework 1.0: 複数エージェントの役割分担、評価、ツール連携をまとめたい場合に候補。Azure 依存・価格は要確認。
- MCP Apps: 個人開発の管理画面、分析UI、内部ツールを MCP 経由で Claude/Codex に見せる用途が有望。

## Claude Code/Codex連携可能性
- MCP Apps: Claude Code/Codex から MCP サーバーを呼び、フォームや可視化UI付きの内部ツールを操作する連携が可能そう。
- GitHub Copilot Agent Mode: Codex/Claude Code と並列に Issue 単位で実装案を比較し、最も良い PR を採用する運用が考えられる。
- Microsoft Agent Framework 1.0: Codex をコード生成担当、別エージェントをテスト・調査担当にする多段ワークフローの土台候補。
- Claude Fable 5: Claude Code の難しい移行・長時間推論タスクの上位モデルとして利用できる可能性。ただし Claude Code での具体的な提供形態は未確認。
- OpenAI Codex + Ona: 長時間クラウド実行やジョブ管理が強くなる可能性。買収完了・製品反映は未確認。

## MCP/Agent系情報
- MCP Apps: MCP の拡張として、テキストや構造化データだけでなく対話UIをホスト側へ渡す流れが出ている。公式性は二次情報ベース。
- MCP セキュリティ: OX Security による STDIO 処理由来の RCE リスク報道あり。個人開発でも未知の MCP サーバー導入時は権限・コマンド実行範囲を絞るべき。
- Microsoft Agent Framework 1.0: Build 2026 で本番向けマルチエージェント基盤として紹介されたとの報道。Azure AI Foundry 周辺との連携が中心。
- MCP 利用実態研究: arXiv 論文で 177,436 件の MCP ツール分析が報告され、ソフトウェア開発用途が大半、action tool の比率上昇が示された。

## 導入価値評価
- GitHub Copilot Agent Mode: ★★★★☆ 即試したい。Issue 駆動の個人開発と相性がよく、保守・小改修の自動化価値が高い。
- MCP Apps: ★★★★☆ 即試したい。Claude Code/Codex 向け内部ツールUI化に直結し、差別化しやすい。
- Claude Fable 5: ★★★☆☆ 経過観察。性能は魅力だが高単価で、個人開発では難所限定の利用が現実的。
- GitHub Spark: ★★★☆☆ 経過観察。MVP 量産には有望だが、価格・出力品質・エクスポート性が未確認。
- Microsoft Agent Framework 1.0: ★★★☆☆ 経過観察。収益化プロダクトの基盤候補だが、個人には Azure 依存と運用コストが重い可能性。
- Siri AI: ★★☆☆☆ 長期観察。OS統合は強いが、個人開発者がすぐ収益化へ転用できる範囲は未確認。
