## 記事本文
2026-07-19のAIツール調査では、昨日の「業務SaaSをMCP化する流れ」から一段進んで、個人開発者が実際に小さなプロダクトへ組み込める実行環境、音声UI、運用エージェントが目立った。ChatGPT Workのような大きな業務エージェントは昨日すでに拾っているため、今日はそれを支える周辺の新規性として、GPT-Live、Vercel Agent、Cloudflare Agents、Gemini Docs MCPを重く見た。

観測した変化は、エージェントが「チャット画面の中で答える存在」から、「本番ログを読み、MCPで外部ツールに触れ、音声で継続的に対話し、長い作業を非同期に処理する存在」へ移っている点だ。OpenAIのGPT-Liveは音声を単なる入出力ではなく、割り込みや沈黙を扱う操作面にしている。Vercel Agentは本番障害やPRレビューをダッシュボード上で扱い、CloudflareはWorkersのAgentからMCPを呼び出せるようにしている。GoogleのGemini Docs MCPは、AIコーディングエージェントが古い知識で実装する問題を公式ドキュメント接続で埋めに来ている。

個人開発の収益化に近いのは、巨大な汎用エージェントを作ることではなく、特定領域の作業口をエージェントに渡すことだ。たとえば「障害ログを読んでユーザー向け告知文を作る」「問い合わせからFAQ候補を作る」「GeminiやCloudflareの最新仕様に沿ってテンプレートコードを生成する」「音声で学習アプリの復習を進める」といった単機能なら、小額サブスクや従量課金にしやすい。特にMCPやSkillsは、Claude CodeやCodexとの相性がよく、開発者向けSaaSの差別化要素になりやすい。

一方で、Vercel AgentやCloudflare Agentsは便利だが、プラットフォーム依存が強い。収益化前の個人開発では、最初から重い運用基盤に寄せるより、無料枠で試し、課金が発生する処理をログ解析、PRレビュー、レポート生成など明確に価値が見える部分に絞るべきだ。次の一歩は、既存の自作アプリに「公式ドキュメントMCP + コード生成 + 承認付きデプロイ前チェック」を1本だけ組み込み、動画化しやすいデモとして見せることになる。

## 個人開発で活用できそうなもの
- Gemini Docs MCP + Skills: Gemini APIを使うテンプレート生成、実装レビュー、SDK移行補助に使える。CodexやClaude Codeへ公式仕様を直接渡せる点が実用的。
- Vercel Agent: 個人開発SaaSの障害調査、PRレビュー、デプロイ後の原因分析を半自動化できる。課金箇所がtokens単位なので、小さく試しやすい。
- Cloudflare Agents McpClient: Workers上で軽量な常駐エージェントを作り、外部MCP経由でGitHub、DB、Slack連携を組める。
- GPT-Live: 学習アプリ、語学練習、将棋・麻雀の音声コーチなど、画面を見続けない体験に向く。
- Claude Science: 科学・医療・論文系の個人開発や研究者向けツールの参考になる。一般用途では過剰だが、監査可能な成果物設計は真似しやすい。
