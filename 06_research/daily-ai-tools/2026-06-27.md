## 個人開発で活用できそうなもの
- Copilot SDK: 自作SaaSや開発支援ツールにCopilot連携を組み込む候補。既存GitHub利用者向けの小規模課金ツールにしやすい。
- Arcade MCP: Gmail、GitHub、Stripe、PostHog、Vercel等の外部操作をAgentに渡す認可レイヤーとして有望。収益化アプリでは「AIにどこまで操作させるか」の制御が価値になる。
- GitHub Agentic Workflows: Issue作成から修正PRまでを自動化する個人開発工場向き。Actions課金とAI creditsの消費管理が必要。
- Copilot CLI: CLIベースの調査、修正、レビューを日次運用に組み込みやすい。GitHub Copilot FreeでもCLIが含まれるため試用ハードルが低い。
- MCP Apps: 将来的に「AIチャット内で使える小型UI」を配布できる可能性あり。ただし現時点の収益化導線は未確認。

## Claude Code/Codex連携可能性
- Arcade MCP: Claude Code、GitHub Copilot、OpenAI Agents、Vercel AI SDK向けドキュメントがあり、外部API操作をMCP経由で統制できる。
- MCP公式仕様: Claude CodeやChatGPT/Codex系クライアントが外部ツール・リソース・プロンプトへ接続する標準インターフェースとして利用可能。
- GitHub Agentic Workflows: CodexやClaude Codeで作った修正方針をGitHub Actions側の自動化に接続する運用が考えられる。
- Copilot Agent Tasks REST API: 外部ツールからCopilot agent taskを作成・管理する導線として、Codex管理スクリプトとの連携余地あり。
- GitHub Issues MCP support: Claude Code/CodexからIssue情報をMCPで読み書きし、仕様整理からPR作成までつなげられる可能性あり。

## MCP/Agent系情報
- MCP公式ドキュメントでは、MCPはAIアプリを外部データ・ツール・ワークフローへ接続する標準と説明。Claude、ChatGPT、VS Code、Cursor等の対応も記載あり。https://modelcontextprotocol.io/docs/getting-started/intro
- MCP 2025-06-18仕様では、Tools、Resources、Promptsに加え、Sampling、Roots、Elicitationが整理されている。セキュリティ面ではユーザー同意、データプライバシー、ツール安全性が明記。https://modelcontextprotocol.io/specification/2025-06-18
- GitHubは2026年6月にAgentic Workflows、Agent tasks REST API、Copilot CLI GA、Issue fields MCP supportなどAgent周辺更新を集中投入。https://github.blog/changelog/
- Arcade.devはMCP clientsとしてClaude Code、GitHub Copilot、VS Code等を案内し、OpenAI Agents、LangChain、Google ADK、Vercel AI SDK等のフレームワーク連携も掲載。https://docs.arcade.dev/
