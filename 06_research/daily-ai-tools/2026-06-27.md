## 個人開発で活用できそうなもの
- OpenClaw: ローカル常駐の個人秘書Agentとして、問い合わせ整理、記事下書き、請求・タスク整理を自動化できそう。セルフホスト可だが、外部API権限は最小化が必要。
- Codex: 既存リポジトリの小規模機能追加、テスト修正、リファクタの量産に向く。収益化アプリの短期リリース速度を上げやすい。
- GitHub Agent HQ: Issue単位でClaude Code/Codex/Copilotを使い分ける運用ができれば、個人開発でもPR作成とレビューの分業がしやすい。
- CrewAI: 調査、要約、実装計画、記事化などを複数Agentに分ける自動化に使える。OSSなので小さく検証しやすい。
- TaintAWI: GitHub ActionsにAI Agentを組み込む前のセキュリティチェック観点として有用。製品というより監査手法として参照価値あり。

## Claude Code/Codex連携可能性
- GitHub Agent HQ: IssueやPRからClaude/Codex系Agentを呼び分け、実装・レビュー・修正を分担する。
- MCP: Claude CodeやCodexからファイル、DB、SaaS操作をツール化できるが、RCE・prompt injection対策が必須。
- OpenClaw: ローカルAgentの実行役にし、Codexをコード変更、Claude Codeを設計レビューに回す構成が考えられる。
- CrewAI: 調査Agent、実装Agent、検証Agentを分け、Codex CLIやClaude Codeを外部実行ステップとして呼ぶ。
- GitHub Actions Agent workflows: Codex/Claude生成PRのCI検証に使えるが、未信頼のIssue本文を直接プロンプト化しない設計が必要。

## MCP/Agent系情報
- Codex利用分析: 2026年前半にCodex利用が5倍超へ伸びたという研究報告。個人開発でもAgent型開発が主流化している可能性が高い（https://arxiv.org/abs/2606.26959）
- AI Coding Agent検出研究: 180M超リポジトリ調査で、Claude CodeやCodexの痕跡がPRだけでは捕捉しきれないと報告（https://arxiv.org/abs/2606.24429）
- MCPセキュリティ: MCP SDKのSTDIO設計に起因するRCEリスクが二次情報で報道。公式対応状況は未確認（https://www.tomshardware.com/tech-industry/artificial-intelligence/anthropics-model-context-protocol-has-critical-security-flaw-exposed）
- Agentic Workflow Injection: GitHub Actions上のAI AgentがIssue本文やPR本文から攻撃されるリスクを体系化（https://arxiv.org/abs/2605.07135）
