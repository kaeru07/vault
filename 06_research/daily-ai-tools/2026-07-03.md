## 記事本文
2026-07-03のAIツール棚卸しでは、単体の生成AIアプリよりも、既存のWeb、リポジトリ、メッセージング、開発環境をAIエージェントの作業対象に変える基盤系ツールが目立った。特にNLWebは、Webサイトを自然言語で問い合わせられる口に変え、同時にMCPサーバーとしても機能する点が重要だった。従来の「検索されるサイト」から、「AIエージェントに直接使われるサイト」へ移る流れが見える。

個人開発の観点では、これは収益化導線の作り方を変える。たとえばレシピ、地域情報、学習教材、商品比較、麻雀・将棋の問題集のような構造化コンテンツを持つ小規模サービスは、SEOだけでなく、AI Agentが問い合わせるデータソースとして提供できる。NLWebのような実装を使えば、既存DBやRSS、Schema.orgを入口にして、会話型UIとMCP接続を同時に用意できる可能性がある。これは広告収益だけでなく、API利用、プレミアム検索、業務向けナレッジ提供への展開余地がある。

一方で、OpenClawやClaude Code、Codexの観測からは、個人開発者自身の開発環境もエージェント化が進んでいる。OpenClawはメッセージアプリを操作面にして、ローカル環境のエージェントを常駐させる方向性を示している。Claude CodeはCLI、IDE、デスクトップ、Web、CI/CD、MCP、Agent SDKへ広がり、CodexもMCP、Subagents、Skills、非対話実行、GitHub連携を前面に出している。個人開発者は「AIツールを使う人」から、「AIに運用・修正・調査を任せる小さな開発組織の管理者」に近づいている。

ただし、本日の情報は公式ドキュメントとGitHub README中心で、日付付きリリースノートまで確認できたものは限られる。価格、正式な更新日、昨日との差分は未確認が残る。次に取るべき一歩は、まずNLWebを小さな既存コンテンツに接続し、MCP経由でClaude CodeまたはCodexから問い合わせられる試作を作ることだ。あわせて、OpenClawのような常駐型エージェントはDM入力が攻撃面になるため、ローカル限定、許可リスト、サンドボックス前提で検証するのが現実的である。

## 個人開発で活用できそうなもの
- NLWeb: 構造化データを持つ個人サービスをAI Agent向け問い合わせ口にできる。学習アプリ、地域DB、商品比較、麻雀・将棋問題集に向く。
- Codex Skills・Subagents: テスト追加、リリースノート作成、issue整理、軽微な修正PR作成をテンプレ化し、個人開発の保守コストを下げられる。
- Claude Code Agent SDK: 独自の開発支援AgentやレビューAgentを組める。課金前提だが、少人数開発の自動化には実用性が高い。
- OpenClaw: SlackやDiscordを入口に個人用の常駐AI秘書を構築できる。外部公開せず、ローカル検証から始める価値がある。
- MCP Reference Servers: Git、Filesystem、Fetch、Memoryを組み合わせ、既存ツールをClaude Code/Codexから安全に呼び出す雛形にできる。

## Claude Code/Codex連携可能性
- NLWeb: サイトやDBをMCPサーバー化し、Claude Code/Codexから「自分のサービス内データ」を調査・修正方針に使う。
- MCP Reference Servers: Git・Filesystem・Fetch・Memoryを接続し、開発中の調査、変更、記録をAgentワークフローに組み込む。
- Claude Code Agent SDK: 複数Agentに調査、実装、レビューを分担させる開発パイプラインを構築する。
- Codex MCP・Skills: 定型の調査、パッチ作成、検証、PR説明生成をCodex側の再利用可能ワークフローにする。
- OpenClaw: Slack/DiscordからCodexやClaude Codeのタスク起動をラップする常駐窓口として使う。ただし権限管理が必須。
