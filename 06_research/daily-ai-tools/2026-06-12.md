## 個人開発で活用できそうなもの
- GitHub Agentic Workflows: issue整理、依存更新、CI失敗調査を定型ワークフロー化できる。個人開発でも「毎朝のリポジトリ棚卸しAgent」に転用しやすい。
- Codex Sites: 市場調査結果、売上シミュレーター、学習進捗ダッシュボードなどを軽量Webアプリとして素早く可視化できる可能性。個人利用可否は未確認。
- Copilot CLI `/settings`: CLI環境の設定が見える化され、複数端末・複数プロジェクトでのAgent運用事故を減らせる。
- MCP: Claude/ChatGPT/Codex系から外部データ・ローカルファイル・業務ツールへ接続する標準として継続有望。公式: https://modelcontextprotocol.io/docs/getting-started/intro

## Claude Code/Codex連携可能性
- MCP server: Claude CodeやChatGPT/Codex系クライアントに外部ツールを接続する標準口。個人開発ではNotion、DB、GitHub、ローカル分析ツール接続に向く。
- GitHub Agentic Workflows: Codex/Claude Codeで作った変更を、GitHub Actions上のAgent検査・triage・docs更新に流せる。
- Codex role-specific plugins: Figma、Canva、Tableau、Snowflake等と連携する方向性が示されており、Codexを調査・制作・分析の実行環境にできる。
- Copilot CLI: ターミナルからAgent作業を進める導線。Codex CLIやClaude Codeと役割分担して、実装・レビュー・CI修正を分けられる。
- OpenAI on Oracle Cloud / Codex: OCI契約経由でOpenAI modelsとCodexへアクセスできる予定。個人開発では直接価値は薄いが、法人案件では導入障壁低下の可能性。公式: https://openai.com/index/openai-on-oracle-cloud/
