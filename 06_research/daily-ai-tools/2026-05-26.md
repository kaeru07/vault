## 個人開発で活用できそうなもの
- Google AI Studio Android app build: 学習アプリ、麻雀・将棋の練習アプリ、広告収益系ユーティリティのAndroid MVPを短時間で作る候補。Play Console内部テストまで近い導線がある点が収益化に近い。
- Dyad: ローカル・オープンソースなので、課金を抑えて小型SaaSや管理画面の試作に使いやすい。セルフホスト可のため、量産テンプレ作成にも向く。
- Codex Goal mode: 「成功条件」を明示して長めの実装を任せられるため、個人開発の機能追加、LP改善、テスト追加を並列化しやすい。
- GitHub MCP Server secret scanning: 個人開発でもAPIキー流出防止に直結する。収益化前の小規模プロダクトほど導入コストに対する安全効果が高い。
- Nuxt MCP Toolkit MCP Apps: 自作MCPツールをテキスト応答だけでなくUI付きで返せるため、Claude/ChatGPT内で使える小型業務ツールの販売・配布余地がある。

## Claude Code/Codex連携可能性
- GitHub MCP Server: Claude Code/CodexからIssue、PR、secret scanningを呼び出し、実装前後のリポジトリ操作と安全確認を自動化。
- Nuxt MCP Toolkit MCP Apps: CodexでNuxt側のMCP appを実装し、Claude/ChatGPT内にフォームや結果UIを返す小型ツールとして連携。
- Zendesk MCP Server: カスタマーサポート系SaaSを作る場合、Claude Code/Codexからチケット・ナレッジ情報を参照する検証環境に使える。
- beehiiv MCP: Claude Code/Codexと組み合わせ、ニュースレター本文生成、指標分析、配信改善案の作成に利用可能。公式ヘルプではClaude Code、Cursor、Codex対応が明記。URL: https://beehiivhelp.zendesk.com/hc/en-us/articles/39255979546263-Getting-started-with-the-beehiiv-MCP
- Google AI Studio + Codex: AI StudioでAndroid MVPを起こし、Codexでリポジトリ化、テスト、課金導線、CIを補強する流れが現実的。
