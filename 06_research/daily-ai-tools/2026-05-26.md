## 個人開発で活用できそうなもの
- Agent Toolkit for AWS: CodexやClaude CodeからAWS構築・デプロイ・ログ調査を任せやすい。個人SaaSのMVPをAWSに載せる場合、CloudFormationやIAMのミス削減に使える。
- OpenAI Codex更新: Goal modeとAppshotsは、フロントエンド修正・長時間タスク・UIレビューに直結。個人開発の「作って直す」ループを短縮できる。
- MCP Registry: 既存MCPサーバー探索に使える。Notion、GitHub、DB、ブラウザ操作などを組み合わせた小型自動化アプリの調査起点になる。
- Claude Connectors / Custom MCP: 自作MCPをClaudeに接続して、Vault、Issue管理、収益分析などを会話UIから操作する用途に向く。ただし外部公開URLと権限管理が必要。
- Claude Design: LP、アプリUI案、スライド、動画台本用のビジュアル案作成に使える可能性あり。課金条件が未確認のため本格導入前に料金確認が必要。

## Claude Code/Codex連携可能性
- Agent Toolkit for AWS: Codexは `codex plugin marketplace add aws/agent-toolkit-for-aws`、Claude CodeもPlugin/MCP経由でAWS操作支援に接続可能。
- AWS MCP Server: Claude Code、Codex、Cursor、Kiro、Windsurf、ClineなどMCP対応AgentからAWSドキュメント検索・API操作に使える。
- MCP Registry: Codex/Claude Codeで利用するMCPサーバー候補を探すレジストリとして使える。
- Claude Custom Connectors: 自作remote MCPをClaudeに登録し、個人開発用のGitHub、Notion、分析DB、収益ダッシュボード連携を構成可能。
- Grok Build: MCP、hooks、skills対応が報道されているため、既存のMCP資産を流用できる可能性あり。ただしClaude Code/Codexとの直接連携は未確認。
