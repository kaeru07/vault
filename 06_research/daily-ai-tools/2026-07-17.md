## 記事本文
今日のAIツール調査では、個人開発に使えるAIが「コード補完」から「作業単位の実行基盤」へ寄っていることがはっきり見えた。ChatGPT Work、Gemini Managed Agents、AI SDK 7はいずれも、単発の回答ではなく、ファイル、外部アプリ、MCP、サンドボックス、承認フローを含む長い作業を扱う方向に進んでいる。昨日のLovable MCPやBase44 Superagentsがプロトタイプ生成と業務自動化に近かったのに対し、今日はその裏側にある実行基盤、統制、観測性が主役だった。

OpenAIのChatGPT Workは、個人開発者にとって「小さな外注先」に近い意味を持つ。市場調査、競合表、LP文案、簡易デモ、営業資料、週次レポートのような、収益化前後に必ず発生する雑務を一つのワークフローとして任せやすい。GPT-5.6がCodexとAPIにも展開されたことで、実装だけでなく、調査、設計、検証、資料化まで同じモデル群でつなぐ導線が強くなった。ただし、上位機能はプラン依存で、従量課金や利用制限も絡むため、最初から常用前提にせず、収益に直結する作業だけに使うのが現実的だ。

開発者向けにはAI SDK 7の重要度が高い。MCP Appsでツール結果をUIとして表示でき、WorkflowAgentで中断や承認を含む実行を継続でき、HarnessAgentでClaude CodeやCodexのような既存ハーネスを扱える。これは個人開発で「AI機能付きアプリ」を作るときに、チャット欄だけで終わらない体験を作れるということだ。たとえば麻雀牌譜分析、将棋学習、広告運用レポート、App Storeレビュー分析のようなサービスで、AIが裏で処理し、ユーザーは結果UIを確認して承認する形にしやすい。

一方、Gemini Managed AgentsとJetBrains AI for Teamsは、個人にはやや重いが今後の標準形を示している。Googleはremote MCP、バックグラウンド実行、サンドボックスをAPI化し、JetBrainsは複数AIエージェントの利用状況、コスト、権限をまとめる方向に進んでいる。個人開発でも、ツールが増えるほど「どのエージェントが何をしたか」「いくら使ったか」「本番に触っていないか」の管理が必要になる。小さなSaaSでも、最初から作業ログ、承認、従量費の上限を設ける価値が増している。

次に取るべき一歩は、AIを使う対象を「実装」だけに閉じないことだ。まずAI SDK 7かGemini Managed Agentsで、1つの定例作業をエージェント化する。候補は、競合アプリ調査、課金ページの改善案生成、レビュー要約、セキュリティ確認、短尺動画台本化がよい。そこで処理時間、費用、手戻り、成果物品質を記録し、利益に近い作業だけをChatGPT WorkやClaude Tagのような高機能エージェントに移す。今日の結論は、AIツール選定の軸を「賢さ」から「売上に近い反復作業を安全に任せられるか」へ移すべき、ということだ。

## 個人開発で活用できそうなもの
- AI SDK 7: Next.jsやNode.jsで、自分のSaaSにMCP Apps、承認付きツール実行、長時間ワークフローを組み込む用途に向く。
- ChatGPT Work: 競合調査、営業資料、LP改善案、週次KPIレポートなど、収益化に近い非実装タスクをまとめて処理する用途に向く。
- Gemini Managed Agents: 無料枠で、調査、コード実行、ファイル処理を含むバックグラウンドエージェントを試す用途に向く。従量課金のため上限管理が必要。
- GitHub Copilot in VS Code更新: Webアプリの画面検証、スクリーンショット確認、複数作業セッション管理に使える。既存Copilot課金がある場合は導入摩擦が低い。
- Claude Tag: Slack中心で運用している小規模チームなら、問い合わせ、障害調査、コード調査をチャンネル文脈付きで委任できる。

## Claude Code/Codex連携可能性
- AI SDK 7 HarnessAgent: Claude CodeやCodexを同じAgentインターフェースで呼び、サンドボックス、スキル、ツールを統一管理する。
- AI SDK 7 MCP Apps: CodexでNext.js側のMCP App表示UIを実装し、ツール実行結果をチャット内の操作画面として返す。
- JetBrains Central CLI / MCP・ACP: Claude Code、Codex、Gemini CLIの利用状況を組織側で観測し、個人開発でも将来のチーム化に備える。
- Gemini Managed Agents remote MCP: Codexで作ったMCPサーバーをGemini側の管理エージェントから呼び、調査・変換処理をクラウドで実行する。
- GitHub Copilot VS Code MCP OAuth: GitHub作業中のMCP認証情報をVS Code secret storageに置き、エージェントから外部ツールへ安全に接続する。
