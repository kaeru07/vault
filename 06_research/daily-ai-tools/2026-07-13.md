## 記事本文
2026年7月13日のAIツール棚卸しでは、個人開発に直結する変化として「AIがコードを書く」段階から「AIが作業単位を受け取り、外部ツールを呼び、必要なら課金まで発生させる」段階への移行が目立った。特にGemini CLI、OpenAI Codex CLI、GitHub Copilot cloud agent、Claude Codeは、いずれもターミナル、GitHub、IDE、MCPを前提にしており、単なる補完ツールではなく、小さな開発チームの疑似メンバーとして扱う方向へ寄っている。

観測した中で最も個人開発向きなのはGemini CLIとCodex CLIである。Gemini CLIは無料枠が明記され、MCP、検索、ファイル操作、シェル実行、GitHub Action連携まで入っているため、低コストで「調査、実装、レビュー、Issue整理」を自動化しやすい。Codex CLIはChatGPTプラン依存だが、ローカル端末で動く軽量Agentとして、既存の個人開発リポジトリに導入しやすい。どちらもセルフホスト型のモデル実行基盤ではないが、CLI自体がOSSであるため、運用手順や検証ワークフローは自分の資産として残せる。

収益化の観点では、Cloudflare Agentsのx402 paid MCP toolsが重要だった。これまではAIツールを作っても、SaaS画面、Stripe Checkout、APIキー管理を別途組む必要があった。MCPツール単位で少額課金できるなら、「画像圧縮」「SEO監査」「翻訳チェック」「麻雀牌譜解析」「将棋局面評価」のような小さな機能をAgent向けAPIとして販売する形が見えてくる。Webアプリより先に、Agentが呼ぶ有料ツールとして出す選択肢が現実的になっている。

一方で、MCP周辺はまだ安全性と品質管理が弱い。Cloudflareの公式ドキュメントも少数のよく設計されたツールを推奨しており、研究報告でもMCPサーバーのランタイム障害やアーキテクチャ分類が増えている。個人開発者がすぐ取るべき一歩は、大きな万能Agentを作ることではなく、既存プロダクトの中から1つだけ「Agentから呼ばれると価値が出る処理」を切り出し、Gemini CLIまたはCodexでMCPサーバー化、Cloudflare上で公開、必要ならx402課金を試すことだ。

## 個人開発で活用できそうなもの
- Gemini CLI: 無料枠を使い、個人開発リポジトリのIssue整理、PRレビュー、実装メモ生成、日次リサーチ自動化に使える。
- OpenAI Codex CLI: 既存コードベースの改修、テスト追加、リファクタ、仕様書からの小規模機能実装に向く。
- Cloudflare x402 paid MCP tools: API単体では売りにくい小機能を、Agentが呼ぶ有料MCPツールとして販売する実験に使える。
- GitHub Copilot cloud agent: backlogの軽微Issue、ドキュメント更新、テスト補強を非同期に進める用途に合う。
- Figma MCP: デザインからReact/Next.js実装へ移す個人開発のUI制作時間を圧縮できる可能性がある。

## Claude Code/Codex連携可能性
- Cloudflare MCP Server: Claude Code/Codexから自作MCPを呼び、検索、変換、課金付き処理を外部ツール化する。
- Figma MCP Server: Claude Code/CodexにFigma設計情報を渡し、見た目推測ではなく構造ベースでUI実装する。
- GitHub Copilot CLI ACP: ACP対応ツールからCopilot CLIをAgentとして呼び、Codex/Claude Codeと役割分担する。
- Gemini CLI GitHub Action: PRレビュー、Issueトリアージ、@gemini-cli呼び出しをCIに組み込み、Codexの実装結果を別Agentで検査する。
- MCP Registry: Claude Code/Codex用に導入するMCPサーバーを探索・公開する入口として使える。
