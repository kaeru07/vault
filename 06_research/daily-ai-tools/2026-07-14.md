## 記事本文
2026-07-14 の観測では、個人開発者が注目すべき変化は「単発の生成AIツール」よりも「常駐エージェント」「開発エージェント」「MCP接続」の三方向に寄っている。ChatGPT Work、Claude Cowork、Perplexity Teammate はいずれも、チャットで答えるだけではなく、ファイル、ブラウザ、メール、Slack、コードベースを横断して作業を進める方向を示している。ただし本日は公式一次情報を十分に取得できず、主要な新規情報の多くは二次メディア経由であるため、名称や料金、提供範囲は未確認として扱う必要がある。

特に個人開発の観点では、AIコーディング市場が IDE 補完から「Issue を渡すとPRまで進める」「レビューや監視まで含める」段階へ移っている点が重要だ。Perplexity Teammate は未公開ながら、検索企業が開発エージェント市場に入ろうとしている兆候であり、Cursor、Claude Code、Codex、GitHub Copilot の競争がさらに厳しくなる可能性がある。これは個人開発者にとって、単にコードを書く速度だけでなく、調査、仕様化、実装、レビュー、リリース文生成までを一つの制作ラインとして組めるかどうかが差になるという意味を持つ。

一方で、GhostApproval と呼ばれる AIコーディングツール群の脆弱性報道は、エージェント化の副作用も示している。AIがファイルを書き換え、コマンドを実行し、外部サービスに接続するほど、権限境界、承認フロー、サンドボックス、差分レビューが収益化以前の前提条件になる。個人開発では「速く作る」だけではなく、顧客データ、決済、APIキー、GitHub権限をAIにどこまで渡すかを設計に含める必要がある。

収益化に近いのは、MCPを使った小さな業務特化エージェントである。公式 MCP ドキュメント、OpenAI Agents SDK、GitHub Copilot の MCP 対応を見る限り、ローカルファイル、GitHub、DB、ブラウザ自動化を標準的な接続口で扱う流れは強い。個人開発者は、巨大な汎用AIアプリを作るよりも、既存業務の「毎日繰り返す1作業」をMCPサーバー化し、Claude Code や Codex から呼び出せる小さな有料ツールにする方が短期リリースしやすい。

次に取るべき一歩は、今日出た未確認の大型ツールを追いかけることではなく、自分の開発ワークフローで一番時間を食う処理を一つ選び、AGENTS.md、MCP、CLI、テスト実行、PR作成を連結した再利用可能なテンプレートにすることだ。AIエージェント市場は派手な発表が多いが、収益につながるのは、請求書処理、調査メモ生成、レビュー自動化、学習コンテンツ化、動画台本化のような狭い反復業務を確実に短縮する部分である。

## MCP/Agent系情報
- Model Context Protocol 公式: MCP はAIアプリを外部システムへ接続するオープン標準として、Claude、ChatGPT、VS Code、Cursor などの対応が案内されている。https://modelcontextprotocol.io/docs/getting-started/intro
- OpenAI Agents SDK MCP: Hosted MCP、Streamable HTTP、SSE、stdio、承認フロー、ツールフィルタリングが公式ドキュメント化されている。https://openai.github.io/openai-agents-python/mcp/
- GitHub Copilot MCP: IDE、Copilot CLI、GitHub Copilot app、cloud agent、code review でMCP利用が可能と公式ドキュメントに記載。https://docs.github.com/en/copilot/concepts/context/mcp
- MCP reference servers: MCP steering group 管理の参照実装として Fetch、Filesystem、Git などが掲載され、本番利用には独自の安全対策が必要と明記。https://github.com/modelcontextprotocol/servers
