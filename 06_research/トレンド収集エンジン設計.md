## 情報源（すべて無料・課金なし）

| 源 | 取得対象 | 取得手段の想定 | 注意 |
|---|---|---|---|
| Google Trends | 急上昇ワード | 公開ページ/RSS 相当 | レート/規約順守 |
| Google Play ランキング | カテゴリ上位アプリ | 公開ストアページ | 既存 scrape-lab-v2 流用候補 |
| App Store ランキング | 同上 | 公開ページ | 同上 |
| Reddit | 関連 subreddit 人気投稿 | 公開 JSON | UA/レート順守 |
| YouTube 検索 | キーワード上位/再生数 | 公開検索 | 規約順守 |
| X トレンド | トレンドワード | 公開トレンド | 取得可否は要確認（未確認） |
| Product Hunt | 新着プロダクト | 公開ページ | — |
| AI ニュース | 主要 AI ニュース見出し | 既存 ny01/ai-trend 系流用候補 | — |

> 取得可否・規約適合は**未確認**項目を「未確認」と明記（確認済みと書かない）。実装前に各源の利用規約確認が必須。

## 取得形式の統一スキーマ（NDJSON 1行=1トレンド）

```
{
  "collectedAt": "YYYY-MM-DDTHH:MM:SS+09:00",
  "source": "google-trends|play-ranking|reddit|...",
  "term": "トレンド語/タイトル",
  "metric": { "rank": 0, "score": 0, "volume": null },   // 取得不能は null（推測値を入れない）
  "url": "公開URL",
  "category": "任意",
  "raw": "要約（機密・個人情報除外）"
}
```

- 機密/個人情報/トークンは保存しない（[[../03_prompts/GitHub反映ルール]] 機密スキャン準拠）
- 取得不能な数値は `null`（「未確認」を数値化しない）

## 保存先構成

```
06_research/daily/
  YYYY-MM-DD/
    trends.ndjson        # 全源の統一レコード（追記）
    summary.md           # その日の上位N件サマリー（人間/案生成用）
  _README.md             # 本構成と運用の説明
```

- 1日1ディレクトリ。既存ファイルは削除しない（追記/新規のみ）
- summary.md は [[トレンド案自動生成設計]]（Issue #26）の入力になる想定

## cron 運用方針（設計・実登録はしない）

- 頻度: 1日1回（例 06:00 JST）。[[../05_monetization/定期調査運用]] の第1→第2段階思想に合わせ、まず手動/半自動から
- 実行主体: 実装後は cron or GitHub Actions（**承認後に別タスクで実装**）。本設計では登録しない
- 失敗時: その源をスキップしログに残す（全体停止しない）。機密混入時は保存せず停止
- GitHub 反映: 収集後 daily/ を commit/push（[[../03_prompts/GitHub反映ルール]]）

## 段階導入（いきなり全自動にしない）
1. **設計（本ファイル・完了）**
2. 手動 1 源で試行（scrape-lab-v2 流用・承認後）
3. 複数源＋日次手動 → 安定後に cron/Actions 化
4. [[トレンド案自動生成設計]]（#26）/[[案プール自動昇格ルール]]（#27）と接続
