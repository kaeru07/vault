# market-research-hub 設計

## 一言要約
ClaudeでもCodexでも同じ調査を回せる、DBなし・JSON/NDJSON蓄積の市場調査ハブ構成案。

## 決定事項 / 結論
- 構成: `UI / runner（Claude/Codex実行スクリプト）/ data（JSON蓄積）/ prompts（共通）/ exports（progress用ToDo JSON）`
- フロー: keywords.json登録 → runner実行 → App Store/Google Play/Trends/Web取得 → `data/runs/日付/` 生データ → `data/apps/` 履歴 → `data/insights/` AI分析 → UIでランキング/差分 → progress用ToDo出力
- **重要制約: 他社アプリ情報/レビューの広範取得は公式APIで強く制限**（Google Play Developer APIは自社アプリ管理向け）。倫理・規約ガード必須
- DBなし運用ではJSON/NDJSONファイル蓄積が最も安全
