# Opus 4.7 プロンプト方針

## 一言要約
Opus 4.7 では手順を過剰に固定せず、**「管理ルールは厳密 / 実装方法は自律裁量」** に寄せると安定する、という方針。

## 決定事項 / 結論
- 思考手順・hooks構成・useMemo・ファイル構成などを機械的に固定しない（Opus4.7は自分で最適化する方が強い）
- ただし必須は守る: 変更範囲厳守 / 既存機能非破壊 / build・lint・typecheck通過 / スマホUI非崩壊 / 推測修正禁止 / 変更理由を説明可能に
- 共通プロンプト（Opus4.7固定・高品質モード）の下に、案件固有情報だけ足す形が最強:
  - 今回の目的 / 対象アプリ / 変更OK範囲 / 禁止範囲 / 完了条件
- 避けるべき: role過剰増殖 / 思考順序固定 / 「まず〜次に〜」 / 過剰DDD化
