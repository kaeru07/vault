# アプリ案工場 Phase1構想

## 一言要約
APIなし・追加課金なしで、progress 内に収益化狙いの小型アプリ案を整理・評価・ToDo化する画面を作る構想。

## 決定事項 / 結論
- **Phase1: API/課金なし** — progressに「アプリ案工場」画面。手入力でアプリ案 → 収益化評価軸でスコア → モック作成ToDo自動登録（ローカルJSON保存）
- **Phase2: 半自動** — Codex/Claudeが市場調査Markdownを読み候補作成→Obsidian蓄積→progressで順位付け
- 外部AI API / 外部DB / Supabase は使わない。既存 ToDo/queue/ExecutionRun/Obsidian連携を壊さない
- 評価軸: 収益化方法 / API不要可否 / 著作権リスク / 実装難易度 / 広告収益見込み / 拡散性 / 継続性 / 総合スコア

## 次に活かせるポイント
- `/app-factory` 等の画面を既存progress構成に自然に追加（[[../05_次にやるToDo候補]] 高優先1番）
- 収益化インパクトを最優先評価軸にする
