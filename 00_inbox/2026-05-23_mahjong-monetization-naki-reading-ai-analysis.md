## 位置づけ

これは単独の開発作業ではなく、収益化テーマ内の作業として扱う。

目的は、麻雀アプリを単なる何切る問題集ではなく、鳴き読み・副露読み・点数状況・押し引き判断を扱う実戦読みクイズへ発展させ、公開アプリ・広告収益・Shorts導線に接続すること。

## 背景

/root/company/apps/mahjong/inputdata/ 配下に、麻雀問題集の写真を追加済み。

この画像群をそのまま公開用データに使うのではなく、AIに十分理解させたうえで、以下を抽出する。

- 問題構造
- 読み思想
- 判断ロジック
- 危険牌評価
- 安全牌評価
- 点数状況込みの押し引き
- 副露読みのパターン
- 類題生成に使える抽象ルール

## 収益化との関連

想定する収益化導線:

1. 麻雀実戦読みクイズアプリとして公開
2. 広告収益化
3. Shorts動画化による流入
4. 中級者向けの差別化コンテンツ化
5. 将来的に問題パック・追加問題・有料版への展開余地を作る

通常の牌効率アプリではなく、「鳴き読み」「副露読み」「オーラス判断」「満貫回避」などの実戦判断に寄せることで差別化する。

## 著作権安全方針

禁止:

- 書籍画像を公開用アプリにそのまま使う
- 原文解説をコピーする
- 同一牌姿をそのまま複製する
- 同一点数状況・同一捨て牌進行をそのまま再現する
- 正答だけを流用する

許可:

- 問題構造の抽象化
- 判断思想の分類
- 読み筋のパターン化
- 独自牌姿への変換
- 独自解説文の生成
- 公開用の新規類題生成

## 初回作業範囲

まずは類題生成まで行かず、認識確認で止める。

Claude Code / Codex に実施させる初回作業:

- inputdata 配下の画像一覧確認
- 問題ページと解説ページのペアリング方針作成
- 画像から読み取るべき項目の整理
- 牌姿、局、点数、親、ドラ、副露、捨て牌、打牌順、正答、解説意図の抽出方針作成
- 原文コピーではなく判断ロジックを抽象化する方針作成
- 問題集が重視している読み筋・判断傾向の初回レポート作成
- 解析用JSONスキーマ案作成
